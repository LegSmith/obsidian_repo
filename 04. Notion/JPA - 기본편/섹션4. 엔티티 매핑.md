## 객체와 테이블 매핑

### `@Entity`

- @Entity가 붙은 클래스는 ==**JPA가 관리**==한다.
- ==**기본 생성자 필수**==
- final 클래스, enum, interface, inner 클래스 사용 x
- 저장할 필드에 final 사용 x

### `@Table`

- name 속성 : 매핑할 테이블 이름 지정.

## 데이터베이스 스키마 자동생성

- ==**DDL을 애플리케이션 실행 시점에 자동 생성**==
- 데이터베이스 방언을 활용하여 데이터베이스에 맞는 DDL 생성
- 이렇게 ==**생성된 DDL은 개발 장비에서만 사용**==

### 자동생성 - 속성

persistence.xml 파일`<property name="hibernate.hbm2ddl.auto" value="create" />`

- create : 기존에 있던 테이블을 지우고 다시 생성
- create-drop : create와 같으나 마지막에 drop함(테스트에서 활용하기 좋음)
- update : 변경부분만 변경(추가부분만 반영, 지우는건안됨!)
- validate : 엔티티와 테이블이 정상 매핑되었는지만 확인

==**절대절대절대**== _==**운영 장비**==_ ==**에는 create, create-drop, update 사용하면 안된다 !!!**==

## 필드와 컬럼 매핑

  

### 매핑 어노테이션 정리

- `@Column` : 컬럼(속성) 매핑
- `@Temporal` : 날짜 타입 매핑
    - `TemporalType.TIMESTAMP` : **날짜와 시간**
    - `TemporalType.TIME` : 시간
    - `TemporalType.DATE` : 날짜
- `@Enumerated` : **enum 타입** 매핑
- `@Lob` : BLOB, CLOB 매핑(큰 컨텐츠)
- `@Transient` : **특정 필드를 컬럼에 매핑X**

### `@Column` 속성

- `insertable` , `updatable` : 애플리케이션에서 컬럼을 등록, 변경 가능 여부(디폴트 값은 true)
- `nullable` : **==null값의 허용 여부==**를 설정( 디폴트값은 true)
- `unique` : ==**유니크 제약조건**==을 걸 때 사용한다.(자주 사용하지는 않음, 단순할 때 사용)
- `columnDefinition` : 컬럼 정보를 **==직접 쿼리문을 작성==**하여 줄 수 있다.
- `length` : 문자 길이 제약조건, **==String 타입에만 사용.==**

  

### enum 타입 사용시 주의 ( 중요 !! )

- EnumType 은 `ORDINAL` , `STRING` 이 있다.
    
    - `ORDINAL` : **==enum 순서==**를 DB에 저장
    - `STRING` : **==enum 이름==**을 DB에 저장.
    
    ==**무조건 STRING을 쓰자 !!**==
    

### `@Temporal`

- 날짜 타입(java.util.Date , java.util.Calender) 매핑할 때 사용

> **자바8버전부터는** `**LocalDate**`**(date) ,** `**LocalDateTime**`**(timestamp)를 사용하면 어노테이션 생략 가능**

- `TemporalType.DATE` : **날짜**, DB date타입
- `TemporalType.TIME` : **시간**, DB time 타입
- `TemporalType.TIMESTAMP` : **날짜와 시간,** DB timestamp 타입

### `@Lob`

- 매핑하는 필드 타입이 문자면 CLOB , 나머지는 BLOB

## 기본 키 매핑

### @ID

- 직접 할당할 때 사용한다.

### @GeneratedValue

- 자동으로 생성할 때 사용
- strategy
    
    - `GenerationType.``_IDENTITY_` _:_ 데이터베이스에 위임한다.
        
        ```sql
        //h2 
        Hibernate: 
            create table Member (
               id varchar(255) generated by default as identity,
                name varchar(255) not null,
                primary key (id)
            )
        //mysql
        Hibernate: 
            create table Member (
               id varchar(255) not null auto_increment,
                name varchar(255) not null,
                primary key (id)
            ) engine=InnoDB
        ```
        
        - IDENTITY 의 전략에 특징으로는 DB에 insert문이 나가기전까지는 PK값을 모른다.
        - 그래서 IDENTITY전략은 commit()시점에 쿼리문이 나가는게 아니고 **==persist()시점에 쿼리문이 나간다.==**
    - `GenerationType.``_SEQUENCE_` : 시퀀스를 생성한다.
        
        ```sql
        Hibernate: create sequence hibernate_sequence start with 1 increment by 1
        Hibernate: 
            
            create table Member (
               id varchar(255) not null,
                name varchar(255) not null,
                primary key (id)
            )
        ```
        
        - SEQUENCE 전략은 시퀀스를 따로 만든다. 즉, 시퀀스를 만들기 전까지는 pk값을 모른다.
        - SEQUENCE 전략은 ==**persist()시점에 DB에 있는 SEQUENCE를 call하여 pk값을 애플리케이션으로 가져와 영속성 컨텍스트에 저장한다.**==
        - allocationSize 라는 속성을 이용하면 성능을 향상시킬 수 있다.
        - ==**allocationSize의 기본값은 50이고 50으로 설정하면 미리 50개만큼의 sequence를 가져와서 애플리케이션에서 사용한다.(DB 의 입출력이 적어져 성능향상 !! )**==
