- **방향(Direction)** : ==**단방향, 양방향**==
- **다중성(Multiplicity)** : **==다대일(N:1) , 일대다(1:N), 일대일(1:1), 다대다(N:M)==**

# Section 2. 양방향 연관관계와 연관관계의 주인

---

**테이블 연관관계는 FK 하나로 양방향 연관관계가 성립된다!!( 객체에서는 다르다 !! )**

## 객체와 테이블이 관계를 맺는 차이 !

- 객체 연관관계 = 2개
    - ==**회원 → 팀 연관관계 (단방향)**==
    - ==**팀 → 회원 연관관계 (단방향)**==
- 테이블 연관관계 = 1개
    - 회원 ↔ 팀의 연관관계(양방향) //==**FK(외래키)로 양방향 가능.**==

**객체에서는 양방향 관계가** ==**사실 서로 다른 단방향 관계 2개다 !!**==

## 연관관계의 주인(Owner)

- 객체 연관관계에서는 **두개의 당방향 연관관계가 있으므로** ==**어느 객체를 외래키(FK)로 관리**==**할 지 정해야 한다.**
- ==**객체의 두 관계중 하나를 연관관계의 주인(Owner)**==으로 지정
- ==**주인이 아닌쪽은 읽기만 가능**==
- ==**주인은 mappedBy 속성 사용불가**==
- 주인이 아니면 mappedBy속성을 지정해야 한다.

### Owner(주인)을 정하는 기준!!

- ==**외래키가 있는 곳을 주인으로 정해라 !!!**==


### Member.java (Owner)

- 연관관계의 주인이기 때문에 mappedBy속성을 사용하지 않았다.

```java
package hellojpa;

import javax.persistence.*;

@Entity
public class Member {

    @Id
    @GeneratedValue
    @Column(name = "MEMBER_ID")
    private Long id;

    @Column(name = "USERNAME")
    private String username;

    @ManyToOne
    @JoinColumn(name = "TEAM_ID")
    private Team team;
		//getter,setter...


}
```

### Team.java

- 주인이 아니기 때문에 mappedBy속성을 사용

```java
package hellojpa;

import javax.persistence.*;
import java.util.ArrayList;
import java.util.List;

@Entity
public class Team {

    @Id @GeneratedValue
    @Column(name = "TEAM_ID")
    private Long id;
    private String name;

    //양방향 연관관계 설정
    @OneToMany(mappedBy = "team")//Member객체의 변수명과 같이해준다.
    private List<Member> members = new ArrayList<>(); //add시에 nullpointer가 안뜬다.(관례임)

		//getter,setter...

}
```
