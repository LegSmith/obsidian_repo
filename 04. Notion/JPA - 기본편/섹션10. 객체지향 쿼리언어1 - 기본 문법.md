### JPA의 다양한 쿼리

- ==**JPQL**==
- JPA Criteria

- 네이티브 SQL
- JDBC API 직접 사용

### JPQL

- 가장 단순한 조회 방법
    - EntityManager.find();
    - 객체 그래프 탐색(a.getB().getC())
- **나이가 18살 이상인 회원을 모두 검색하고 싶다면 ? → 이때 JPQL을 쓴다.**
- JPA를 사용하면 엔티티 객체를 중심으로 개발
- 검색을 할 때도 **테이블이 아닌 엔티티 객체를 대상으로 검색**

### Criteria

- 문자가 아닌 자바코드로 JPQL을 작성
- JPQL 빌더 역할
- ==**너무 복잡하고 실용성이 없다 → 비추**==
- Criteria 대신에 **==QueryDSL 추천!!==**

### QueryDSL

- 문자가 아닌 자바코드로 JQPL 작성
- ==**컴파일 시점에 문법 오류를 찾을 수 있음**==
- 동적쿼리 작성 편리함
- Criteria 보다 단순하고 쉽다.
- ==**실무 사용 권장**==

### 네이티브 SQL 소개

- JPQL로 해결할 수 없는 특정 DB에 의존적인 기능

### JPQL 문법

- SELECT
    - `select m from Member as m where m.age > 18`
    - 엔티티와 속성은 대소문자 구분 해야한다, 키워드(select, from 등등)는 대소문자를 구분하지 않아도 된다.
    - ==**별칭(m)은 필수**==(a
- 집합과 정렬
    - gruop by, order by 도 기본 SQL과 똑같다.
        
        ```sql
        select
        		COUNT(m),   //회원수
        		SUM(m.age), //나이 합
        		AVG(m.age), //평균 나이
        		MAX(m.age), //최대 나이
        		MIN(m.age)  //최소 나이
        from Member m
        ```
        
- TypeQuery, Query
    - TypeQuery : 반환 타입이 명확할 때 사용한다.
        - `TypedQuery<Member> query = em.createQuery("select m from Member m", Member.class);`
        - 이런 경우는 Member가 반환되기 때문에 타입을 알 수 있다.
    - Query : 반환 타입이 명확하지 않을때 사용
        - `Query query2 = em.createQuery("select m.username, m.age from Member m");`
        - username(String), age(int) 가 반환되기 때문에 String인지 int형인지 구분이 어렵다.

### 결과 조회 API

- getResultList()
    - `List<Member> member1 = em.createQuery("SELECT m FROM Member m", Member.class).getResultList();`
    - 결과가 없으면 빈 리스트 반환한다( null포인트 인셉션 에러를 잡지못함 !!)
- getSingleResult()
    - ==**결과가 정확히 하나여야만 한다**==, 단일 객체 반환
    - 결과가 없으면 : **javax.persistence.NoResultException**
    - 둘 이상이면 : **javax.persistence.NonUniqueResultException**
    - getResultList()와는 다르게 결과가 없어도 오류가 난다 !!

### 파라미터 바인딩 - 이름 기준, 위치 기준

- 이름기준으로 하는게 나중에 오류를 방지할 수 있고 직관적이다
    
    ```java
    Member query = em.createQuery("SELECT m FROM Member m WHERE m.username = :key", Member.class)
            .setParameter("key", "Cristiano Ronaldo")
            .getSingleResult();
    
    System.out.println("query.getId() = " + query.getId());
    ```
    
- 위치기준은 쓰지말자 !!

### 프로젝션

- SELECT 절에 조회할 대상을 지정
- 프로젝션 대상 : 엔티티, 임베디드 타입, 스칼라 타입(숫자, 문자등)

### 프로젝션 - 여러 값 조회

`select m.username,m.age from Member m`

1. Query 타입으로 조회
2. Object[] 타입으로 조회
    
    ```java
    List<Object[]> resultList = em.createQuery("select m.username,m.age from Member m").getResultList();
    
    Object[] result = resultList.get(0);
    System.out.println("username = "+result[0]);
    System.out.println("age = " + result[1]);
    ```
    
3. **new 명령어로 조회 → 가장 깔끔하다.**
    - 단점은 패키지명이 길어지면 코드가 길어진다
    - 순서와 타입이 일치하는 생성자가 필요하다.
    - 별도의 DTO를 생성하여 new 명령어를 통해 쿼리문안에 기입한다.
        
        ```java
        package hellojpa.jpql;
        
        public class MemberDTO {
        
            private String username;
            private int age;
        
            public MemberDTO(String username, int age) {
                this.username = username;
                this.age = age;
            }
        
            public String getUsername() {
                return username;
            }
        
            public void setUsername(String username) {
                this.username = username;
            }
        
            public int getAge() {
                return age;
            }
        
            public void setAge(int age) {
                this.age = age;
            }
        }
        ```
        
        ```java
        List<MemberDTO> result = em.createQuery("select new hellojpa.jpql.MemberDTO(m.username,m.age) from Member m", MemberDTO.class)
                .getResultList();
        
        MemberDTO memberDTO = result.get(0);
        System.out.println("memberDTO.getUsername() = " + memberDTO.getUsername());
        System.out.println("memberDTO.getId() = " + memberDTO.getAge());
        ```

# 조인

- 내부 조인
    - `SELECT m FROM Member m [INNER] JOIN m.team t`
- 외부 조인
    - `SELECT m FROM Member m LEFT [OUTER] JOIN m.team t`
- 세타 조인
    - `SELECT count(m) from Member m, Team t WHERE m.username = t.name`

## 조인 - ON 절

> JPA 2.1 부터 지원

1. 조인 대상 필터링
2. 연관관계 없는 엔티티 외부 조인(Hibernate 5.1 버전 부터 지원)

### 조인 대상 필터링

- 예) 회원(Member)과 팀(Team)을 조인하면서, 팀 이름이 A인 팀만 조인
- JPQL
    - `SELECT m, t FROM Member m LEFT JOIN m.team t` **`ON`** `t.name = 'A'`
- SQL
    - `SELECT m.* , t.* FROM Member m LEFT JOIN Team t` **`ON`** `m.TEAM_ID = t.id and t.name='A'`

### 서브쿼리 지원 함수

- **[NOT] EXISTS (subquery)** : **==서브쿼리에 결과가 존재하면 참==**
    - **{ALL | ANY | SOME} (subquery)**
    - **ALL** : 모두 만족하면 참
    - **ANY, SOME** : 조건을 하나라도 만족하면 참
- **[NOT] IN (subquery)** : **==서브쿼리의 결과 중 하나라도 같은 것이 있으면 참==**

### JPA 서브 쿼리 한계

- JPA는 WHERE , HAVING 절에서만 서브 쿼리 사용 가능
- FROM 절의 서브 쿼리는 현재 JPQL에서 불가능
    - 조인으로 풀 수 있으면 풀어서 해결
