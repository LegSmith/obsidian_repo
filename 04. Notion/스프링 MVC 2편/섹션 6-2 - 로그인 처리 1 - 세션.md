## 세션 동작 방식

---

==_**쿠키에 중요한 정보를 담으면 보안이 취약하므로 중요한 정보는 서버에 저장하고 임의의 세션ID를 서버에서 지정하여 세션ID를 쿠키에 담아서 식별한다.**_==

### 세션 ID 생성

- 클라이언트가 로그인 요청을 하면 서버에서는 ID 와 Password 를 DB에서 조회하여 **맞는 사용자면 세션 저장소에 _==임의의 세션ID 를 생성해 쿠키에 담아 응답==_해준다.**
- 이때 세션 ID는 추적이 불가능하게 `UUID` 를 이용한다.
- _==**세션 ID 와 중요한 정보는 서버에 세션 저장소에서 관리**==_하게 된다.
    
    ![[Untitled 8.png|Untitled 8.png]]
    
- 클라이언트는 다음 요청부터 세션ID 를 쿠키에 담은 채 요청하게 된다.
- _==**서버에서는 클라이언트가 전달한 세션ID 를 세션 저장소에 있는지 조회하여 사용자를 식별**==_한다.

## 순수 Session 구현

---

- Session 생성,조회,삭제를 담당하는 SessionManager 클래스를 생성 후 `@Component` 로 스프링 빈으로 등록
- ==_**세션 저장소는 HashMap 을 사용해도 되지만 동시성 문제를 해결**_==하기 위해 `ConcurrentHashMap` 사용
    
    ```java
    @Component
    public class SessionManager {
    
        public static final String SESSION_COOKIE_NAME = "mySessionId";
        private Map<String,Object> sessionStore = new ConcurrentHashMap<>(); // 동시성 문제 떄문에 ConcurrentHashMap 을 사용
    
    		// 생략 ...
    }
    ```
    

### 세션 생성

- `UUID` 를 사용하여 sessionId 를 생성
- 넘어온 ==_**Value 와 sessionId 를 세션 저장소에 저장**_==
- 응답헤더에 쿠키를 생성한다.
    
    ```java
    /**
     * 세션 생성
     * <li>sessionId 생성 (UUID 사용)</li>
     * <li>세션 저장소에 sessionId와 보관할 값 저장</li>
     * <li>sessionId로 응답 쿠키를 생성해서 클라이언트에 전달</li>
     */
    public void createSession(Object value, HttpServletResponse response){
    
        //sessionId 생성 (UUID 사용) 후 값을 세션에 저장
        String sessionId = UUID.randomUUID().toString();
        sessionStore.put(sessionId,value);
    
        // 쿠키 생성
        response.addCookie(new Cookie(SESSION_COOKIE_NAME, sessionId));
    }
    ```
    

### 세션 조회

- HTTP 응답 메시지에서 쿠키를 가져온다.
    - `findCookie()` 메서드에서는 _==**쿠키를 조회하면 배열 타입이기 때문에 루프를 돌려서 쿠키를 반환해준다.**==_
- 쿠키가 비어있지 않다면 세션저장소에서 쿠키값을 조회하여 리턴해준다.
    
    ```java
    /**
     * 세션 조회
     */
    public Object getSession(HttpServletRequest request){
        Cookie sessionCookie = findCookie(request, SESSION_COOKIE_NAME);
        if(sessionCookie == null)
            return null;
    
        return sessionStore.get(sessionCookie.getValue());
    
    }
    public Cookie findCookie(HttpServletRequest request, String cookieName){
        if(request.getCookies() == null){
            return null;
        }
        return Arrays.stream(request.getCookies())
                .filter(cookie -> cookie.getName().equals(cookieName))
                .findAny()
                .orElse(null);
    }
    ```
    

### 세션 만료

- HTTP 요청헤더에서 쿠키를 가져와 세션 저장소에 있는 쿠키를 제거해주면 된다.
    
    ```java
    /**
     * 세션 만료
     */
    public void expire(HttpServletRequest request){
        Cookie cookie = findCookie(request, SESSION_COOKIE_NAME);
        if(cookie != null){
            sessionStore.remove(cookie.getValue());
        }
    }
    
    
    public Cookie findCookie(HttpServletRequest request, String cookieName){
        if(request.getCookies() == null){
            return null;
        }
        return Arrays.stream(request.getCookies())
                .filter(cookie -> cookie.getName().equals(cookieName))
                .findAny()
                .orElse(null);
    }
    ```
    

### 세션 테스트 코드

==_**HttpServletRequest 나 HttpServletResponse 는 인터페이스이기 때문에 구현체를 사용해야한다. 하지만 테스트를 용이하게 해주는 Mock 을 이용하면 코드가 간편해진다.**_==

```java
class SessionManagerTest {

    SessionManager sessionManager = new SessionManager();

    @Test
    void sessionTest(){

        // 세션 생성을 생성하고 응답헤더에 쿠키담음
        MockHttpServletResponse response = new MockHttpServletResponse();
        Member member = new Member();
        sessionManager.createSession(member,response);

        // 요청에 응답 쿠키 저장
        MockHttpServletRequest request = new MockHttpServletRequest();
        request.setCookies(response.getCookies()); // 응답을 했다고 가정하고 쿠키 설정

        // 세션 조회
        Object result = sessionManager.getSession(request);

        assertThat(result).isEqualTo(member);

        // 세션 만료
        sessionManager.expire(request);
        Object expiredSession = sessionManager.getSession(request);
        assertThat(expiredSession).isNull();
    }
}
```

## 서블릿 HTTP 세션

---

**Session 은 웹 어플리케이션에서 꼭 필요한 것이다. 그렇기 때문에 Servlet 에서는 `HttpSession` 기능을 제공한다.**

- `HttpSession` 에서는 쿠기 이름을 `JSESSIONID` 로 지정 되어 있고 값은 추정 불가능한 랜덤 값이다.

### 로그인/로그아웃 구현

- `HttpSession` 에서 `getSession()` 메서드에서는 _==**파라미터로 boolean 이 들어온다.**==_
    - `true` : 디폴트 값이며, **_==session 이 없으면 session 을 생성==_**한다.
    - `false` : session 이 없으면 session 을 생성하지 않는다.
    - `SessionConst.LOGIN_MEMBER` 는 임의로 지정한 상수이다.
- `session.setAttrubte()` 메서드는 하나에 세션에 여러 값을 지정한다.
- `session.invalidate()` 메서드는 세션을 제거한다.
    
    ```java
    @PostMapping("/login")
    public String loginV3(@Valid @ModelAttribute LoginForm form, BindingResult bindingResult, HttpServletRequest request){
        // Field Error
        if(bindingResult.hasErrors()){
            return "login/loginForm";
        }
    
        Member loginMember = loginService.login(form.getLoginId(), form.getPassword());
    
        // Global Error
        if(loginMember == null){
            bindingResult.reject("loginFail","아이디 또는 비밀번호가 맞지 않습니다.");
            return "login/loginForm";
        }
    
        // HttpSession 이용
        // 세션이 있으면 있는 세션 반환, 없으면 신규 세션을 생성해서 반환
        HttpSession session = request.getSession();
        // 세션에 로그인 회원 정보를 보관
        session.setAttribute(SessionConst.LOGIN_MEMBER,loginMember);
    
        return "redirect:/";
    }
    
    @PostMapping("/logout")
    public String logoutV3(HttpServletRequest request){
        HttpSession session = request.getSession(false);
        if(session != null)
            session.invalidate();
    
        return "redirect:/";
    }
    ```
    

### @SessionAttribute 사용

- 스프링에서 개발자에게 더 간편하게 해줄려고 `@SessionAttribute` 를 제공한다.
- name 속성에는 session 에 저장된 키 값을 받는다.
- `HttpSession` 을 사용하면 ==_**반환 객체로 타입 캐스팅을 해줘야 하는데 @SessionAttribute 로 미리 반환된 객체를 받아서 코드가 줄어든다.**_==
    
    ```java
    @GetMapping("/")
    public String homeLoginV3Spring(
            @SessionAttribute(name =SessionConst.LOGIN_MEMBER, required = false) Member loginMember, Model model){
    
        // Session 에 회원 데이터가 없으면 home
        if(loginMember == null)
            return "home";
    
        model.addAttribute("member",loginMember);
        return "loginHome";
    }
    ```
    

## 세션 정보와 타임아웃 설정

---

개발 코드에서는 사용자가 로그아웃 버튼을 클릭했을 때 `session.invalidate()` 가 동작하면서 세션이 삭제됐다. 하지만 _==**대부분 사용자는 웹브라우저를 종료한다.**==_

- 사용자가 로그아웃 버튼을 누르지 않고 브라우저만 종료했을 때 ==_**서버 입장에서는 HTTP 비 연결성(ConnectionLess) 때문에 사용자가 껏는지 알지 못한다.**_==
- 즉, ==**세션의 종료 시점을 사용자가 최근에 요청한 시간을 기준**==으로 잡아야한다.
    
    예시)
    
    - 사용자가 첫 로그인을 하고 1시간동안 계속 웹 서핑을 한다고 치자. 그렇게 되면 30분마다 세션이 종료되기 때문에 2번을 로그인 해야한다.
    - 하지만 사용자는 한 페이지에만 머무는게 아니라 이곳 저곳을 클릭하면서 페이지 이동이 있기 떄문에 계속 요청이 들어온다.
    - 세션 종료시점을 새 요청 기준으로 잡으면 사용자 입장에서는 매끄러운 웹서핑이 가능해진다.
    

### 타임아웃 설정 - 글로벌, 특정 세션

- `application.yml`파일에 타임아웃을 지정하면 글로벌하게 적용된다.
    
    - 기본은 1800(30분)이다.
    
    ```xml
    server.servlet.session.timeout=60 \#1초
    ```
    
- 특정 세션에서 적용할려면 `setMaxInactiveInterval()` 메서드를 사용하면 된다.
    
    ```java
    session.setMaxInactiveInterval(1800); //1800초
    ```
    

> 실무에서는 ==_**세션에 최소한의 데이터만 보관**_== 해야 한다. `보관한 데이터 용량*사용자 수` 로 인하여 세션의 메모리 사용량이 급격하게 늘어나 장애로 이어진다.
