# 입력 폼 처리

---

입력 폼(Input Form) 을 보면 id , name ,value 등의 속성을 각각 지정하다보면 귀찮을 때 가 있다. 타임리프에서는 저 속성들을 한꺼번에 지정해주는 `th:object` 가 있다

### Controller 부분

- 단순히 Get 메서드로 뷰만 리턴해주는 메서드에 빈 객체 하나를 model 에 담아서 뷰에 뿌려준다.

    - 빈 객체는 비용이 거의 들지 않기 때문에 괜찮다.

    ```java
    @GetMapping("/add")
    public String addForm(Model model){
        model.addAttribute("item",new Item());
        return "form/addForm";
    }
    ```


### View 부분

- 기존에 코드를 보면 id 속성, name 속성 , value 속성을 지정해주고 서버가 동작할때 렌더링될때 데이터를 갖고오는 th:value 도 설정되었다.

    ```html
    <form>
    	<div>
    	  <label for="id">상품 ID</label>
    	  <input type="text" id="id" name="id" class="form-control" value="1" th:value="${item.id}" readonly>
    	</div>
    <!-- 생략 .. -->
    </form>
    ```

- `th:object="${item}"` 은 form 태그 안에서 사용할 객체(커맨드 객체) 를 지정해준다.
- `th:field=*{itemName}` 을 지정해주면 커맨드 객체인 item 엔티티에서 itemName 이라는 필드를 가져와서 바인딩 해준다.( `${item.itemName}` 과 같다.)

    - id 속성을 제거해도 th:field 가 지정해주지만 <label> 태그 때문에 그냥 적었다.

    ```html
    <form>
    	<div>
    	    <label for="itemName">상품명</label>
    	    <input type="text" id="itemName" th:field="*{itemName}" class="form-control" placeholder="이름을 입력하세요">
    	</div>
    <!-- 생략 .. -->
    </form>
    ```


# 체크박스 - 단일

---

_**체크박스를 사용할 때에는 체크 사용여부에 따라 어떤 값이 서버로 넘어오는지 확인을 잘 해야 한다!!**_

- 체크박스 name을 open이라고 설정하고 클라이언트가 체크박스를 선택하고 서버로 요청하면 open에는 on 이라는 값이 담겨서 전송된다.

    ```html
    <div>판매 여부</div>
    <div>
        <div class="form-check">
            <input type="checkbox" id="open" name="open" class="form-check-input">
            <label for="open" class="form-check-label">판매 오픈</label>
        </div>
    </div>
    ```

    ![[Untitled 5.png|Untitled 5.png]]

- 하지만 체크박스를 체크안하고 전송하면 필드가 아예 서버로 전송도 안된다.
- 로그를 찍어 확인해보면 null 이 찍힌다. 개발자 입장에서는 그냥 null 일 경우 분기를 찍어서 로직을 짜면 되긴한데 필드가 아예 안넘어오면 실수인지 고의인지 서버에서는 구분이 힘들다.

    ```java
    @PostMapping("/add")
    public String addItem(@ModelAttribute Item item, RedirectAttributes redirectAttributes){
        log.info("item.open={}",item.getOpen());
        Item savedItem = itemRepository.save(item);
        redirectAttributes.addAttribute("itemId",savedItem.getId());
        redirectAttributes.addAttribute("status", true);

        return "redirect:/form/items/{itemId}";
    }
    ```

    ![[Untitled 1 4.png|Untitled 1 4.png]]

- 그래서 스프링은 히든 태그를 넣어 구분할 수 있다.
- type 은 hidden 으로 하고 name을 위에 체크박스 네임과 같게 하고 앞에 _ 만 붙히면 된다.

    ```html
    <div>판매 여부</div>
    <div>
        <div class="form-check">
            <input type="checkbox" id="open" name="open" class="form-check-input">
            <input type="hidden" name="_open" value="on"/> <!-- 히든 필드 추가 -->
            <label for="open" class="form-check-label">판매 오픈</label>
        </div>
    </div>
    ```

- open 필드는 없지만 _open 필드는 무조건 서버로 요청이간다.

![[Untitled 2 4.png|Untitled 2 4.png]]

- ==**그럼 스프링은 _open 을 보고 체크를 해제했다고 판단한다 . 그래서 null 이 아닌 false 로 찍힌다.**==

# 체크 박스 - 단일2

---

_**체크박스 하나하나 히든필드를 넣어서 하면 너무 번거롭기 때문에 타임리프가 제공하는 편리한 기능을 쓰면 된다 !!**_

- 타임리프에서 제공하는 th:field 를 통해서 check 박스를 쉽게 구현해준다.
- `<form>` 태그에서 `th:object` 로 Item 엔터티를 커멘트객체로 선언했기 떄문에 선택 변수식인 `*{}` 를 사용해서 필드명을 바로 사용할 수 있다.

    ```html
    <!-- single checkbox -->
    <div>판매 여부</div>
    <div>
        <div class="form-check">
            <input type="checkbox" id="open" th:field="*{open}" class="form-check-input">
            <label for="open" class="form-check-label">판매 오픈</label>
        </div>
    </div>
    ```

- th:field 가 checked 옵션까지 알아서 랜더링 해주기 때문에 개발자는 매우 편하게 된다 !!

# 체크 박스 - 멀티

---

### @ModelAttribute 의 다른기능

- 보통 메서드 파라미터 부분에 `@ModelAttribute`를 사용하면 HTTP 파라미터들을 Java 객체에 바인딩해주는 역할을 한다.
- **==클래스 내부에 선언하게 되면==** `model.addAttribute()` **==역할을 해주기도한다.==**

    ```java
    @ModelAttribute("regions")
    public Map<String, String> regions(){
        Map<String, String> regions = new LinkedHashMap<>(); // 순서 보장
        regions.put("SEOUL", "서울");
        regions.put("BUSAN", "부산");
        regions.put("JEJU","제주");

        return regions;
    }

    // 위 코드는 아래 로직을 컨트롤러 내부에 있는 모든 메서드에 적용시켜준다.

    @GetMapping("/test")
    public void test(Model model){
    	Map<String, String> regions = new LinkedHashMap<>(); // 순서 보장
      regions.put("SEOUL", "서울");
      regions.put("BUSAN", "부산");
      regions.put("JEJU","제주");

    	model.addAttribute("regions",regions");
    	return "/testForm";
    }
    ```

- ==**즉, 해당 컨트롤러를 요청할 때 regions() 메서드에서 반환한 값이 자동으로 모델에 담기게 된다!**==

### Multi Checkbox

- 체크 박스를 여러개 출력해야하기 때문에 th:each 에 regions가 키값인 Map 컬렉션을 담는다.
- 중요한건 체크박스가 여러개이기 때문에 label 로 구분을 해야한다.

    - `th:for="${#ids.prev('regions')}"` : 이 코드가 각각의 input 태그의 id값을 다르게 지정해준다.

    ```html
    <!-- multi checkbox -->
    <div>
        <div>등록 지역</div>
        <div th:each="region : ${regions}" class="form-check form-check-inline">
            <input type="checkbox" th:field="*{regions}" th:value="${region.key}"
                   class="form-check-input">
            <label th:for="${\#ids.prev('regions')}"
                   th:text="${region.value}" class="form-check-label">서울</label>
        </div>
    </div>
    ```

    - 이런식으로 regions1,2,3 으로 id를 구분해준다.

    ```html
    <input type="checkbox" value="SEOUL" class="form-check-input" id="regions1"
    name="regions">
    <input type="checkbox" value="BUSAN" class="form-check-input" id="regions2"
    name="regions">
    <input type="checkbox" value="JEJU" class="form-check-input" id="regions3"
    name="regions">
    ```

- 코드에 th:field 덕분에 아무것도 체크안해도 _regions 를 타임리프가 자동으로 생성해서 null 이 아닌 false 를 리턴해준다.
- checked = "checked" 도 타임리프가 th:field 에 지정한 값과 th:value 값을 비교해서 자동으로 처리해준다.

# 라디오 버튼

---

_**라디오버튼은 체크박스와 다르게 여러 선택지 중에 하나만 선택할 수 있따 !!**_

- 체크박스는 체크를 하지 않으면 null 이 넘어가기 때문에 타임리프가 별도의 히든필드를 만들어줬지만, **==라디오버튼은 이미 선택이 되면 수정시에도 항상 하나를 선택하도록 되어있기 때문에 별도의 히든필드가 없다.==**

    ```html
    <!-- radio button -->
    <div>
        <div>상품 종류</div>
        <div th:each="type : ${itemTypes}" class="form-check form-check-inline">
            <input type="radio" th:field="*{itemType}" th:value="${type.name()}"
                   class="form-check-input">
            <label th:for="${\#ids.prev('itemType')}" th:text="${type.description}"
                   class="form-check-label">
                BOOK
            </label>
        </div>
    </div>
    ```
