## 스프링 타입 컨버터 소개

---

**==HTTP 요청 파라미터가 컨트롤러로 넘어올 때 스프링타입 컨버터가 중간에서 타입을 변환해준다.==**

- HTTP 쿼리 스트링으로 넘어오는 data 는 웹 브라우저에서 문자타입이지만 **==컨트롤러로 넘어오면서 Integer 로 변환된채 받을 수 있다.==**

    ```java
    @RestController
    public class HelloController {

        @GetMapping("/hello-v2")
        public String helloV2(@RequestParam Integer data){
            System.out.println("data = " + data);
            return "ok";
        }
    }
    ```

- `@RequestParam` 뿐 아니라 `@ModelAttribute` , `@PathVariable` 등 다양하다.

### 컨버터 인터페이스

- 스프링은 확장 가능한 컨버터 인터페이스를 제공한다.
- 개발자가 추가적인 타입 변환이 필요하면 인터페이스를 구현해서 스프링에 등록하면 된다.

    ```java
    package org.springframework.core.convert.converter;

    public interface Converter<S, T> {
      T convert(S source);
    }
    ```


## 타입 컨버터 - Converter

---

**==스프링 등록 없이 직접 Converter 인터페이스를 구현해서 테스트코드 까지 구현==**

### String → Integer , Integer → String

- ==**Converter 인터페이스를 받아 직접 구현**==하고 `convert()` 메서드만 오버라이딩 해주고 로직을 짜면 된다.

    ```java
    @Slf4j
    public class IntegerToStringConverter implements Converter<Integer, String> {
        @Override
        public String convert(Integer source) {
            log.info("convert source={}",source);
            return String.valueOf(source);
        }
    }

    @Slf4j
    public class StringToIntegerConverter implements Converter<String, Integer> {
        @Override
        public Integer convert(String source) {
            log.info("convert source={}", source);
            return Integer.valueOf(source);
        }
    }
    ```

- 스프링에 등록하지 않았기 때문에 테스트코드에서는 직접 객체를 불러와서 쓰면 된다.

    ```java
    @Test
    @DisplayName("String -> Integer 컨버트")
    void stringToInteger(){
        StringToIntegerConverter converter = new StringToIntegerConverter();
        Integer result = converter.convert("10");
        assertThat(result).isEqualTo(10);
    }

    @Test
    @DisplayName("Integer -> String 컨버트")
    void integerToString(){
        IntegerToStringConverter converter = new IntegerToStringConverter();
        String result = converter.convert(10);
        assertThat(result).isEqualTo("10");
    }
    ```


### IpPort→String , String→IpPort

- 개발자가 직접 만든 객체(`IpPort`)도 가능하다.

    `@EqualsAndHashCode` : ==**두 객체의 내용이 같은지 비교하는 Equals**== 와 ==**두 객체가 같은 객체인지 동일성을 비교하는 hashCode 를 구현**==해주는 `Lombok` _**이 제공하는 어노테이션**_이다.

    ```java
    @Getter
    @EqualsAndHashCode
    public class IpPort {
        private String ip;
        private int port;

        public IpPort(String ip, int port) {
            this.ip = ip;
            this.port = port;
        }
    }
    ```

- String 타입은 `"127.0.0.1:8000"` 형태로 오기 때문에 `split()` 메서드로 `:` 를 기준으로 끊어서 **==String 을 나눈후 IpPort 객체 필드에 각각 담아줘서 반환==**하면 된다.

    ```java
    @Slf4j
    public class StringToIpPortConverter implements Converter<String, IpPort> {
        @Override
        public IpPort convert(String source) {
            log.info("convert source = {}",source);
            // 127.0.0.1:8000"
            String[] split = source.split(":");
            String ip = split[0];
            int port = Integer.parseInt(split[1]);

            return new IpPort(ip,port);
        }
    }

    @Slf4j
    public class IpPortToStringConverter implements Converter<IpPort, String> {
        @Override
        public String convert(IpPort source) {
            log.info("convert source = {}",source);
            return source.getIp()+":"+source.getPort();
        }
    }
    ```


## **컨버전 서비스 - ConversionService**

---

**==개별 컨버터를 등록 해서 편리하게 사용할 수 있는 기능이다.==**

- `ConversionService` 인터페이스는 단순히 `canConvert()` , `convert()` 메서드만 제공한다.
- 그래서 개발자는 ==**ConversionService 에 만든 컨버터들을 등록해두고 다른곳에 두고 스프링에 등록하면 필요한 클래스에서 주입받아서 convert() 메서드로 사용**==만 하면 된다.
- 등록과 사용이 분리되있어서 ==**인터페이스 분리 법칙(ISP)**== 가 잘 지켜져 있다.

    ```java
    @Test
    void conversionService(){
        //등록
        DefaultConversionService conversionService = new DefaultConversionService();
        conversionService.addConverter(new StringToIpPortConverter());
        conversionService.addConverter(new IntegerToStringConverter());
        conversionService.addConverter(new StringToIntegerConverter());
        conversionService.addConverter(new IpPortToStringConverter());

        //사용
        assertThat(conversionService.convert("10", Integer.class)).isEqualTo(10);
        assertThat(conversionService.convert(10, String.class)).isEqualTo("10");

        IpPort ipPort = conversionService.convert("127.0.0.1:8080", IpPort.class);
        assertThat(ipPort).isEqualTo(new IpPort("127.0.0.1",8080));

        String result = conversionService.convert(new IpPort("127.0.0.1", 8080), String.class);
        assertThat(result).isEqualTo("127.0.0.1:8080");
    }
    ```


## 스프링에 Convert 등록하기

---

**==WebMvcConfigurer 인터페이스를 구현화시킨 클래스에 Convert를 등록할 수 있다.==**

- `addFormatters()` 메서드를 오버라이딩해서 Convert를 등록하면 된다.

    ```java
    @Configuration
    public class WebConfig implements WebMvcConfigurer {

        @Override
        public void addFormatters(FormatterRegistry registry) {
            registry.addConverter(new StringToIntegerConverter());
            registry.addConverter(new IntegerToStringConverter());
            registry.addConverter(new StringToIpPortConverter());
            registry.addConverter(new IpPortToStringConverter());
        }
    }
    ```

- 아까 만든 IpPort 객체 컨버터를 등록했기 때문에 파라미터로 받을 수 있다.

    ```java
    @GetMapping("/hello-v3")
    public String ipPort(@ModelAttribute IpPort ipPort){
        System.out.println("ipPort.getIp() = " + ipPort.getIp());
        System.out.println("ipPort.getPort() = " + ipPort.getPort());
        return "ok";
    }
    ```


## 포맷터 - Formatter

---

`Converter` 는 ==**입력,출력 타입에 제한이 없는 범용 타입 변환 기능**==을 제공한다. `Formatter` 는 **==문자에 특화된 기능이고 현지정보(Locale)을 참고하여 특별하게 변환이 가능한 Converter 의 특별한 버전==**이다.

- 예를 들어 ==**1000 이란 숫자형 타입이 들어올 때 한국 기준(Locale) 으로 "1,000" 으로 변환**==할 때 `Formatter` 를 사용하면 편하게 변환이 가능하다. 물론 그 반대도 가능하다.
- `Formatter` 는 인터페이스이기 때문에 구현화해서 `print()` , `parse()` 메서드를 오버라이딩 하면된다.

    - `print()` : 객체 → 문자
    - `parse()` : 문자 → 객체

    ```java
    public interface Printer<T> {
        String print(T object, Locale locale);
    }

    public interface Parser<T> {
      T parse(String text, Locale locale) throws ParseException;
    }
    public interface Formatter<T> extends Printer<T>, Parser<T> {

    }
    ```

- 실제로 구현할 때는 개발자가 직접 "1,000" 이 들어올때 Parse 해서 1000으로 바꿀 수 있지만 매우 복잡하다. 하지만 자바가 제공하는 `NumberFormat` 객체를 사용하면 `Locale` 정보를 활용하여 ==**나라별로 다른 숫자 포맷을 만들어준다.**==

    - `Number` 타입은 `Integer` , `Long` 타입의 부모 클래스이다.
    - `getInstance()` 로 Locale 의 정보를 가져온 뒤 `format()` 메서드로 변환하면 된다.

    ```java
    @Slf4j
    public class MyNumberFormatter implements Formatter<Number> {
        @Override
        public Number parse(String text, Locale locale) throws ParseException {
            log.info("text={}, locale={}",text, locale);
            return NumberFormat.getInstance(locale).parse(text); // "1,000" -> 1000
        }

        @Override
        public String print(Number object, Locale locale) {
            log.info("object={}, locale={}",object,locale);
            return NumberFormat.getInstance(locale).format(object); // 1000 -> "1,000"
        }
    }
    ```


## 포맷터를 지원하는 컨버전 서비스

---

컨버전 서비스에는 컨버터만 등록할 수 있고, 포맷터는 등록할 수 없다. 하지만 `FormattingConversionService` 를 사용하면 ==**컨버터와 포맷터를 다 등록**==할 수 있다.

- `DefaultFormattingConversionService` 는 `FormattingConversionService` 에 기본적인 통화,숫자 관련 몇가지 기본 포맷터를 추가해서 제공한다.

    - 컨버터를 등록할 때는 addConverter() , 포맷터를 등록할 때는 addFormatter() 로 등록하면 된다.
    - 사용할 때는 `convert()` 메서드에 **==변환할 객체와 변환할 객체 타입만 파라미터==**로 넘겨주면 된다.

    ```java
    @Test
    @DisplayName("포맷터를 지원하는 컨버전 서비스 테스트")
    void formattingConversionService(){
        DefaultFormattingConversionService conversionService = new DefaultFormattingConversionService();

        // 컨버터 등록
        conversionService.addConverter(new StringToIpPortConverter());
        conversionService.addConverter(new IpPortToStringConverter());

        // 포맷터 등록
        conversionService.addFormatter(new MyNumberFormatter());

        // 컨버터 사용
        IpPort ipPort = conversionService.convert("127.0.0.1:8080", IpPort.class);
        assertThat(ipPort).isEqualTo(new IpPort("127.0.0.1",8080));

        // 포맷터 사용
        assertThat(conversionService.convert("1,000", Integer.class)).isEqualTo(1000);
        assertThat(conversionService.convert(1240124, String.class)).isEqualTo("1,240,124");
    }
    ```


## 웹 애플리케이션에 포맷터 적용하기

---

- `WebConfig` 파일에서 `addFormatter()` 메서드로 포맷터를 추가해주면 된다.
- String ↔ Integer 변환하는 컨버터를 주석을 한 이유는 `MyNumberFormatter()` 도 **==문자를 숫자로 바꾸는 특별한 버전의 컨버터이므로 우선순위에 밀릴 수 있다==**. 그래서 주석을 하고 포맷터를 추가해준 것 .

    ```java
    @Configuration
    public class WebConfig implements WebMvcConfigurer {
        @Override
        public void addFormatters(FormatterRegistry registry) {
            // 우선순위 때문에 주석
    //        registry.addConverter(new StringToIntegerConverter());
    //        registry.addConverter(new IntegerToStringConverter());

            registry.addConverter(new StringToIpPortConverter());
            registry.addConverter(new IpPortToStringConverter());

            // 포맷터 추가
            registry.addFormatter(new MyNumberFormatter());
        }
    }
    ```


## 스프링이 제공하는 기본 포맷터

---

**==스프링에서는 어노테이션 기반으로 원하는 형식을 지정하여 포맷터를 지원한다.==**

### @NumberFormat

- ==**숫자 관련 형식 지정 포맷터**==이다.
- patter 옵션을 이용하여 형식을 지정해서 받을 수 있다.

    ```java
    @NumberFormat(pattern = "###,###")
    private Integer number;
    ```


### @DateTimeFormat

- 날짜 관련 형식 지정 포맷터이다.
- pattern 옵션을 이용하여 날짜 형식을 지정할 수 있다.

    ```java
    @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss")
    private LocalDateTime localDateTime;
    ```
