# 타임리프 소개

---

### 타임리프 특징

- SSR(Server Side Rendering) : 서버사이드랜더링
- 네츄럴 템플릿
- **==스프링 통합 지원==**

### 네츄럴 템플릿(Natural Templates)

- 타임리프는 ==**순수 HTML을 최대한 유지**==하는 특징이 있다.
- 타임리프로 작성한 파일은 **HTML 을 유지**하기 때문에 ==**서버를 가동하지 않고 웹 브라우저에서 파일을 직접 열어도 내용을 확인할 수 있다.**==
    - JSP 는 그냥 열면 깨진다 ;;;

### 타임리프 사용 선언

- 일반적인 HTML 을 시작하면 <html> 로 시작한다.
- 타임리프는 아래와 같이 태그를 바꿔주면 사용할 수 있다.
    
    `<html xmlns:th="http://www.thymeleaf.org">`
    

  

# 텍스트 - text, utext

---

타임리프는 기본적으로 HTML 태그 속성에 기능을 정의해서 동작한다.

### th:text

- 콘텐츠 안에 데이터를 출력할려면 th:text 를 태그 속성안에 선언해주면 된다.
- 그리고 데이터는 ${ param } 로 감싸주면된다.
    
    ```java
    @GetMapping("/text-basic")
    public String textBasic(Model model){
        model.addAttribute("dat "Hello Cristiano!");
        return "basic/text-basic";
    }
    ```
    
- 기본은 태그 속성안에서 정의하지만 태그안이 아닌 컨텐츠 안에 직접 출력할 때에는 [[param]] 으로 넣으면 된다.
    
    ```html
    <!doctype html>
    <html lang="ko"
          xmlns:th="http://www.thymeleaf.org">
    <head>
        <meta charset="UTF-8">
        <title>Document</title>
    </head>
    <body>
    <h1>컨텐츠에 데이터 출력하기</h1>
    <ul>
        <li>th : text 사용 <span th:text="${data}"></span></li>
        <li>컨텐츠 안에서 직접 출력하기 = [[${data}]]</li>
    </ul>
    </body>
    </html>
    ```
    

### th:utext

- 만약 데이터를 진하게 해서 출력하고 싶다고 하면 컨트롤러에서 보내는 데이터를 <b></b> 태그를 같이 적어서 뷰에 뿌려줄 수 있다.
    
    ```java
    @GetMapping("/text-unescaped")
    public String textUnescaped(Model model){
        model.addAttribute("data", "Hello <b>Cristiano!</b>");
        return "basic/text-unescaped";
    }
    ```
    
- 하지만 타임리프는 ==**기본적으로 이스케이프(escape)를 제공**==하기 때문에
    - < → &lt;
    - > → &gt;
- 위와 같이 이스케이프 처리가 된다. 이를 막을때 ==**utext**== 를 사용한다.
- 이때 ==**[[param]] → [(param)] 으로 바뀐다.**==
    
    ```html
    <!DOCTYPE html>
    <html xmlns:th="http://www.thymeleaf.org">
    <head>
        <meta charset="UTF-8">
        <title>Title</title>
    </head>
    <body>
    <h1>text vs utext</h1>
    <ul>
        <li>th:text = <span th:text="${data}"></span></li>
        <li>th:utext = <span th:utext="${data}"></span></li>
    </ul>
    <h1><span th:inline="none">[[...]] vs [(...)]</span></h1>
    <ul>
        <li><span th:inline="none">[[...]] = </span>[[${data}]]</li>
        <li><span th:inline="none">[(...)] = </span>[(${data})]</li>
    </ul>
    </body>
    </html>
    ```
    

# 변수 - SpringEL

---

타임리프에서 변수를 표현할 때는 `${...}` 로 표현을 한다.

## 스프링 EL

- Object , List, Map 등등을 다양하게 표시할 수 있는 스프링이 제공하는 기능이다.
- 아래와 같이 Object 는 `user` 라는 키로 View에 뿌리고 List는 `users`, Map은 `userMap` 으로 뿌려준다.
    
    ```java
    @GetMapping("/variable")
    public String variable(Model model){
        User userA = new User("Ronaldo", 37);
        User userB = new User("Messi", 35);
    
        List<User> list = new ArrayList<>();
        list.add(userA);
        list.add(userB);
    
        Map<String,User> map = new HashMap<>();
        map.put("userA",userA);
        map.put("userB",userB);
    
        model.addAttribute("user",userA);
        model.addAttribute("users",list);
        model.addAttribute("userMap",map);
    
        return "basic/variable";
    }
    @Data
    static class User{
        private String username;
        private int age;
    
        public User(String username, int age) {
            this.username = username;
            this.age = age;
        }
    }
    ```
    

### Object

- `user.username` : user의 usermae 필드를 프로퍼티 접근법으로 해서 가져온다.
- `user[’username’]` : 위와 같다.
- `user.getUsername()` : user 의 getUsername() 메서드를 직접호출

### List

- `users[0].username` = 리스트 0번째 인덱스의 객체를 프로퍼티 접근법으로 username가져온다.
- `users[0][’username’]` : Object와 동일
- `users[0].getUsername()`: Object와 동일

### Map

- List는 인덱스 위치로 객체를 가져오지만 Map은 Key 값을 이용하여 접근한다.

# 기본 객체들

### 타임리프가 제공하는 기본 객체들(==스프링 부트 3.0 미만 기준==)

> 스프링부트 3.0 이상이면 컨트롤러에서 직접 Model에 담아서 보내줘야 한다.

- `${#request}`
- `${#response}`
- `${#session}`
- `${#servletContext}`
- `${#locale}`

## 쿼리스트링, 세션, 스프링 빈 접근

- 세션은 sessionData 라는 Key로 값을 저장하면 된다.
- 스프링 빈에 등록된걸 접근하기 위해 `@Component` 어노테이션으로 스프링 빈에 등록한다.
    
    ```java
    @GetMapping("/basic-objects")
    public String basicObjects(HttpSession session){
        session.setAttribute("sessionData","Cristiano Session");
        return "basic/basic-objects";
    }
    
    @Component("helloBean")
    static class HelloBean{
        public String hello(String data){
            return "Hello "+data;
        }
    }
    ```
    
- param은 타임리프에서 제공하는 HTTP 요청 파라미터에 바로 접근할 수 있는 key 이름이다.
    
    ```html
    <!DOCTYPE html>
    <html xmlns:th="http://www.thymeleaf.org">
    <head>
        <meta charset="UTF-8">
        <title>Title</title>
    </head>
    <body>
    <h1>식 기본 객체 (Expression Basic Objects)</h1> <ul>
        <li>request = <span th:text="${\#request}"></span></li>
        <li>response = <span th:text="${\#response}"></span></li>
        <li>session = <span th:text="${\#session}"></span></li>
        <li>servletContext = <span th:text="${\#servletContext}"></span></li>
        <li>locale = <span th:text="${\#locale}"></span></li>
    </ul>
    <h1>편의 객체</h1> <ul>
        <li>Request Parameter = <span th:text="${param.paramData}"></span></li>
        <li>session = <span th:text="${session.sessionData}"></span></li>
        <li>spring bean = <span th:text="${@helloBean.hello('Spring!')}"></span></li> </ul>
    </body>
    </html>
    ```

# URL 링크

---

==**타입리프에서 URL을 생성할 때는** `@{URL}` **문법을 사용한다.**==

- param1 , param2 라는 키값을 통해 data1, data2라는 문자열을 VIEW에 뿌린다고 해보자.
    
    ```java
    @GetMapping("/link")
    public String link(Model model){
        model.addAttribute("param1","data1");
        model.addAttribute("param2","data2");
        return "basic/link";
    }
    ```
    

### 단순한 URL

- 그냥 URL 주소 그대로 `@{URL주소}` 로 표현해주면 된다.

### 쿼리 파라미터

- 예를들어 naver.com?name=ronaldo&age=37 이라는 주소를 타임리프로 바꾸면
    - `@{/naver.com(param1=${param1}, param2=${param2})`
    - 쿼리파라미터의 변수들은 ()로 묶어주면 된다.

### 경로변수

- [naver.com/news/busan](http://naver.com/news/busan) 이라는 주소가 있다면
    - `@{/naver.com/{param1}/{param2}(param1=${param1},param2=${param2})`
    - / 를 이용해 경로에 PathVariable 처럼 넣어주고 뒤에 () 로 model 로 넘어온 키를 넣어주면 된다.

### 경로변수 + 쿼리 파라미터

- [naver.com/news?title=busan](http://naver.com/news?title=busan) 이라는 주소가 있다면
    - `@{/naver.com/{param1}(param1=${param1}, param2=${param2})`
    - 경로는 / 로 적어주고 쿼리파라미터는() 안에 적어주면 된다.

# 리터럴

---

==**타임리프에서 리터럴이란 고정된 값을 의미한다.**==

- 아래 코드에서는 “hello” 라는 문자열이 문자 리터럴
- 10, 20이 숫자 리터럴이다.
    
    ```java
    String a = "hello";
    int a = 10 * 20;
    ```
    
- 타임리프에서는 문자 리터럴을 반드시 ‘(작은 따옴표)로 감싸야 한다.
    - `<span th:text=”’hello’”>`
- 하지만 띄어쓰기가 없이 한번에 쭉 이어진 문자열은 ‘ 로 안 감싸도 하나의 토큰으로 인식해준다.
- ==**단, 공백이 있는 문자열은 반드시 ‘ 로 묶어야 한다.**==

### 리터럴 대체(Literal substitutions)

- 리터럴 대체 문법인 || 로 묶어주면 마치 템플릿을 사용하는 것 처럼 편하다.
    
    `<span th:text="|hello ${data}|">`
    
- 리터럴 대체 문법을 안썻다면 아래와 같이 + 연산자를 사용하여 보기도 불편하고 유지보수가 어려울 수 있다.
    
    `<span th:text="'hello ' + ${data}">`
    

# 연산

---

==**타임리프의 연산은 자바의 연산과 크게 다르지 않다. >< 같은 기호들의 이스케이프만 조심해서 쓰면 된다.**==

- > 연산자는 &gt; 같이 바꿔서 쓰면 된다.
- 조건식은 자바의 조건식과 유사하다.
- Elvis 연산자
    - 조건식의 편의 버전으로 만약 데이터가 null 이 아니면 데이터를 출력하고 null 이면 : 다음에 오는 데이터를 출력한다.
- No-Operation
    
    - _ 인 경우는 마치 타임리프가 실행되지 않는 것 처럼 동작한다.
    - `${nullData}? : _` 이 코드에서는 nullData의 값이 없으면 타임리프 태그들은 무시하고 순수 HTML태그를 실행한다.
    
    ```html
    <ul>
        <li>산술 연산 <ul>
            <li>10 + 2 = <span th:text="10 + 2"></span></li>
            <li>10 % 2 == 0 = <span th:text="10 % 2 == 0"></span></li>
        </ul>
        </li> <li>비교 연산
        <ul>
            <li>1 > 10 = <span th:text="1 &gt; 10"></span></li>
            <li>1 gt 10 = <span th:text="1 gt 10"></span></li>
            <li>1 >= 10 = <span th:text="1 >= 10"></span></li>
            <li>1 ge 10 = <span th:text="1 ge 10"></span></li>
            <li>1 == 10 = <span th:text="1 == 10"></span></li>
            <li>1 != 10 = <span th:text="1 != 10"></span></li>
        </ul> </li>
        <li>조건식 <ul>
            <li>(10 % 2 == 0)? '짝수':'홀수' = <span th:text="(10 % 2 == 0)? '짝수':'홀수'"></span></li>
        </ul> </li>
        <li>Elvis 연산자
            <ul>
                <li>${data}?: '데이터가 없습니다.' = <span th:text="${data}?: '데이터가 없습니다.'"></span></li>
                <li>${nullData}?: '데이터가 없습니다.' = <span th:text="${nullData}?: '데이터가 없습니다.'"></span></li>
            </ul>
        </li>
        <li>No-Operation
            <ul>
                <li>${data}?: _ = <span th:text="${data}?: _">데이터가 없습니다.</span></li>
                <li>${nullData}?: _ = <span th:text="${nullData}?: _">데이터가 없습니다.</span></li>
            </ul>
        </li>
    </ul>
    ```
    

# 속성 값 설정

---

`==th:*==` ==속성을 지정하면 타임리프는 기존의 * 속성을`th:*` 속성으로 대체한다.==  
ex) `<span name=”sar” th:name=”cos”/span>` → cos로 랜더링이 된다.

### 속성 추가

`th:attrappend` : ==**속성 값의 뒤**==에 값을 추가한다.

`th:attrprepend` : ==**속상 값의 앞**==에 값을 추가한다.

`th:classappend` : ==**class 속성에 자연스럽게 추가**==한다.

### check 속성

- HTML 에서는 checked 속성에 true, false 중 아무거나 와도 checked 처리가 된다.
    - `<input type="checkbox" name="active" checked="false"/>`
    - `<input type="checkbox" name="active" checked="true"/>`
    - 위 2개의 코드는 동일한 결과로 나온다.
- 개발자 입장에서는 매우 불편하다 …
- ==**타임리프는**== `==th:checked=”false”==` ==**이 올 경우 그냥 checked 속성을 지워 버린다.**==

# 반복

---

**타임리프에서는** `th:each` **속성으로 반복문을 사용한다.**

- 컨트롤러에서 model 에 key와 value를 뷰로 넘겨서 뷰에서는 key를 통해 반복문을 실행한다.
    
    ```java
    @GetMapping("/each")
    public String each(Model model){
        addUsers(model);
        return "basic/each";
    }
    private void addUsers(Model model){
        List<User> list = new ArrayList<>();
        list.add(new User("Ronaldo",37));
        list.add(new User("Messi",35));
        list.add(new User("Park",40));
    
        model.addAttribute("users",list);
    }
    ```
    
- 뷰에서는 th:each 속성을 통해 받은 컬렉션을 뿌려준다.
- `<tr th:each="user : ${users}">` → ==**user 라는 변수에 컬렉션의 있는 데이터를 차례로 넣어서 반복한다.**==
    
    ```html
    <!DOCTYPE html>
    <html xmlns:th="http://www.thymeleaf.org">
    <head>
        <meta charset="UTF-8">
        <title>Title</title>
    </head>
    <body>
    <h1>기본 테이블</h1>
    <table border="1">
        <tr>
            <th>username</th>
            <th>age</th>
        </tr>
        <tr th:each="user : ${users}">
            <td th:text="${user.username}">username</td>
            <td th:text="${user.age}">0</td>
        </tr>
    </table>
    <h1>반복 상태 유지</h1>
    <table border="1">
        <tr>
            <th>count</th>
            <th>username</th>
            <th>age</th>
            <th>etc</th>
        </tr>
        <tr th:each="user, userStat : ${users}">
            <td th:text="${userStat.count}">username</td>
            <td th:text="${user.username}">username</td>
            <td th:text="${user.age}">0</td>
            <td>
                index = <span th:text="${userStat.index}"></span>
                count = <span th:text="${userStat.count}"></span>
                size = <span th:text="${userStat.size}"></span>
                even? = <span th:text="${userStat.even}"></span>
                odd? = <span th:text="${userStat.odd}"></span>
                first? = <span th:text="${userStat.first}"></span>
                last? = <span th:text="${userStat.last}"></span>
                current = <span th:text="${userStat.current}"></span>
            </td>
        </tr>
    </table>
    </body>
    </html>
    ```
    

### 반복 상태 유지

- `<tr th:each="user, userStat : ${users}">` → 반복의 상태를 확인 할 수 있다.
- 두 번째 파라미터인 userStat 은 생략이 가능하다. 왜냐하면 ==**타임리프에서 user라는 변수 뒤에 Stat을 알아서 붙여줘서 사용하기 때문이다.**==

### 반복 상태 유지 기능

- `index` : 0부터 시작하는 값
- `count` : 1부터 시작하는 값
- `size` : 전체 사이즈
- `even` : 짝수 여부(boolean)
- `odd` : 홀수 여부(boolean)
- `first, last` : 처음 마지막 여부(boolean)
- `current` : 현재 겍체

# 조건부 평가

---

**타임리프에서 조건부평가(if, unless) 는** `th:if=”조건”` **,** `th:unless=”조건”` **속성을 사용한다.**

### if, unless

- ==**타임리프에서 if 조건도 아니고, unless 조건도 아니면 태그 자체를 랜더링하지 않는다.**==

### switch

- 자바에서 사용하는 switch와 똑같고 ==**디폴트부분만 ***== 로 표현한다.
    
    ```html
    <tr th:each="user, userStat : ${users}">
        <td th:text="${userStat.count}">1</td>
        <td th:text="${user.username}">username</td>
        <td th:switch="${user.age}">
    			<span th:case="10">10살</span> 
    			<span th:case="20">20살</span> 
    			<span th:case="*">기타</span>
    		</td> 
    </tr>
    ```
    

# 주석

---

**타임리프에서 사용하는 주석은** `<!—/* …. */—!>` **로 사용한다.**

### 기본 HTML 주석

- 태그 자체가 랜더링 되지 않는다.
- ==**페이지 소스 보기를 하면 주석 부분이 그대로 노출이 된다.**==
    
    `<!— <span th:text="${data}">html data</span> !—>`
    

### 타임리프 파서 주석

- ==**타임리프의 주석이고 페이지 랜더링시 랜더링 되지 않고, 페이지 소스 보기를 해도 주석 부분이 보이지 않는다.**==
    
    ```html
    <!--/*-->
    <span th:text="${data}">html data</span>
    <!--*/-->
    ```
    

### 타임리프 프로토타입 주석

- 타임리프 파서 주석과 조금 다르다.
- ==**그냥 HTML 파일만 열었을 때는 렌더링 되지 않고, 서버를 통해 열었을 때 렌더링이 된다.**==
    
    ```html
    <!--/*/
    	<span th:text="${data}">html data</span>
    /*/-->
    ```

# 자바스크립트 인라인

---

**타임리프에서 자바스크립트를 지원할때** `th:inline="javascript"` **를 사용하면 타임리프에 데이터도 자바스크립트에서 편하게 쓸 수 있다.**

- 타임리프가 아닌 일반 HTML 에서 자바스크립트를 썻을 때와 아닐때를 보자
    
    ```html
    !-- 자바스크립트 인라인 사용 전 -->
    <script>
        var username = [[${user.username}]];
        var age = [[${user.age}]];
        //자바스크립트 내추럴 템플릿
        var username2 = /*[[${user.username}]]*/ "test username";
        //객체
        var user = [[${user}]];
    </script>
    ```
    
- username은 제대로 나왔고 user변수만 조금 다르게 표현이 됐다.
    
    ![[Untitled 4.png|Untitled 4.png]]
    
- 하지만 F12를 눌러 console 창을 보면 이런 오류가 있다. 왜냐하면 타임리프에서 값이 그대로 넘어갔기 때문에 “”(쌍따옴표)가 없어서 String이 아닌 변수로 인식해서 저렇게 된다.
    
    ![[Untitled 1 3.png|Untitled 1 3.png]]
    
- 타임리프에서 제공하는 자바스크립트 인라인을 사용하면
    
    ```html
    <!-- 자바스크립트 인라인 사용 후 -->
    <script th:inline="javascript">
        var username = [[${user.username}]];
        var age = [[${user.age}]];
        //자바스크립트 내추럴 템플릿
        var username2 = /*[[${user.username}]]*/ "test username";
        //객체
        var user = [[${user}]];
    </script>
    ```
    
- 달라진점은 3가지가 있다.
- username 의 값에 “” 처리가 됐다.
- 서버랜더링을 하면 주석처리된 값인 Son 이 출력되고 HTML 파일을 그냥 열었을때는 “test username” 이 출력된다. 즉, ==**네츄럴 템플릿**==으로 바뀐다는 것이다.
- 그리고 ==**객체를 JSON 형식**==으로 출력해준다.
    
    ![[Untitled 2 3.png|Untitled 2 3.png]]
    

## 자바스크립트 인라인 each

- each를 통해 반복문을 사용해야 할 때가 있다.
- 잘 사용하지는 않지만 아래와 같이 사용한다.
    
    ```html
    <!-- 자바스크립트 인라인 each -->
    <script th:inline="javascript">
        [# th:each="user, stat : ${users}"]
        var user[[${stat.count}]] = [[${user}]];
        [/]
    </script>
    ```
    

# 템플릿 조각

---

**웹 페이지를 개발할때 공통으로 사용되는 부분들이 많다. 이를 복사하면서 사용하면 추후에 유지보수할 때 힘들다. 타임리프는 템플릿조각 기능을 지원한다.**

- `th:fragment=”copy”` 을 통해서 나중에 공통부분을 사용할 태그에 copy와 같이 템플릿조각 이름을 적으면 된다.
- `th:fragment="copyParam (param1, param2)` 와 같이 데이터를 넘겨 받을 수 있다. (copyParam은 이름이다)
    
    ```html
    <!DOCTYPE html>
    <html xmlns:th="http://www.thymeleaf.org">
    <body>
    	<footer th:fragment="copy"> 
    		푸터 자리 입니다.
    	</footer>
    	<footer th:fragment="copyParam (param1, param2)">
    		<p>파라미터 자리 입니다.</p>
    	<p th:text="${param1}"></p> <p th:text="${param2}"></p>
    	</footer>
    </body>
    </html>
    ```
    
- 공통부분을 사용할 뷰에서는 insert 가 잇고 replace 가 있다.
- `th:insert=”~{경로 :: 템플릿조각이름}”` → 현재 태그 안에 템플릿조각 태그를 넣는다.
    - ex) `<div><p>템플릿조각</p></div>`
- `th:replace=”~{경로 :: 템플릿조각이름}”` → 현재 태그를 템플릿조각 태그로 교체한다.
    - ex) `<p>템플릿조각</p>`
- 경로를 간단히 표현할 때에는 ~{} 를 제외하고 쓸 수 있다.
- 파라미터도 템플릿조각 이름과 템플릿조각에 보낼 데이터를 같이 적어주면 된다.
    
    ```html
    <h1>부분 포함</h1>
    <h2>부분 포함 insert</h2>
    <div th:insert="~{template/fragment/footer :: copy}"></div>
    <!--div 태그안에 가져온걸 넣는다.-->
    
    <h2>부분 포함 replace</h2>
    <div th:replace="~{template/fragment/footer :: copy}"></div>
    <!--div 가져온 태그로 대체한다.-->
    
    <h2>부분 포함 단순 표현식</h2>
    <div th:replace="template/fragment/footer :: copy"></div>
    
    <h1>파라미터 사용</h1>
    <div th:replace="~{template/fragment/footer :: copyParam ('데이터1', '데이터 2')}"></div>
    ```
    

# 템플릿 레이아웃

---

**템플릿 레이아웃은 <head>나 <footer> 같이 공통을 소아요하는 부분을 한 곳에 모아두고, 페이지마다 필요한 정보를 더 추가해서 사용이 가능하다.**

- layoutMain.html 파일에 haed 부분에 title과 link 2개를 파라미터로 base.html 템플릿 레이아웃으로 보내고 replace 한다고 해보자
- `layoutMain.html` → <head> 부분을 common_header라는 이름을 가진 base.html 로 교체하고, title 과 link 2개를 base.html 로 전송한다.
    
    ```html
    <!DOCTYPE html>
    <html xmlns:th="http://www.thymeleaf.org">
    <head th:replace="template/layout/base :: common_header(~{::title},~{::link})">
        <title>메인 타이틀</title>
        <link rel="stylesheet" th:href="@{/css/bootstrap.min.css}">
        <link rel="stylesheet" th:href="@{/themes/smoothness/jquery-ui.css}">
    </head>
    <body>
    메인 컨텐츠
    </body>
    </html>
    ```
    
- `base.html`
    
    → `layoutMain.html` 에서 보낸 title이 replace되서 레이아웃 타이틀이 메인 타이틀로 바뀌고
    
    → 공통인 부분은 그대로 출력이 된다.
    
    → `<th:block th:replce=”${links}”/>` → 이부분에 `layoutMain.html` 에서 보낸 link 2개가 replace되서 들어가진다.
    
    ```html
    <html xmlns:th="http://www.thymeleaf.org">
    <head th:fragment="common_header(title,links)">
    
    	<title th:replace="${title}">레이아웃 타이틀</title>
    
    	<!-- 공통 -->
    	<link rel="stylesheet" type="text/css" media="all" th:href="@{/css/awesomeapp.css}">
    	<link rel="shortcut icon" th:href="@{/images/favicon.ico}">
    	<script type="text/javascript" th:src="@{/sh/scripts/codebase.js}"></script>
    	
    	<!-- 추가 -->
    	<th:block th:replace="${links}" />
    
    </head>
    ```
    
- 즉, 결과는 이렇게 나온다.
    
    ```html
    <!DOCTYPE html>
    <html>
    <head>
    
        <title>메인 타이틀</title>
    
        <!-- 공통 -->
        <link rel="stylesheet" type="text/css" media="all" href="/css/awesomeapp.css">
        <link rel="shortcut icon" href="/images/favicon.ico">
        <script type="text/javascript" src="/sh/scripts/codebase.js"></script>
    
        <!-- 추가 -->
        <link rel="stylesheet" href="/css/bootstrap.min.css">
    		<link rel="stylesheet" href="/themes/smoothness/jquery-ui.css">
    
        </head>
    <body>
    메인 컨텐츠
    </body>
    </html>
    ```
    

## 템플릿 레이아웃 확장

- 템플릿의 <head> 나 <footer> 같이 큰 부분을 가져올 수 있지만. html 파일 자체를 통째로 바꿀 수 있다.
- `layoutFile.html` 이라는 껍데기 파일을 만든다.
    
    ```html
    <!DOCTYPE html>
    <html th:fragment="layout (title, content)" xmlns:th="http://www.thymeleaf.org">
    <head>
        <title th:replace="${title}">레이아웃 타이틀</title> </head>
    <body>
    <h1>레이아웃 H1</h1>
    
    <div th:replace="${content}">
        <p>레이아웃 컨텐츠</p>
    </div>
    
    <footer>
        레이아웃 푸터
    </footer>
    </body>
    </html>
    ```
    
- <html> 태그에 그냥 replace 로 템플릿 레이아웃으로 바꾸게 되면 layoutFile.html 이 통째로 들어가게 된다.
- title과 section을 파라미터로 보낸다.
    
    ```html
    <!DOCTYPE html>
    <html th:replace="~{template/layoutExtend/layoutFile :: layout(~{::title},~{::section})}"
          xmlns:th="http://www.thymeleaf.org">
    <head>
        <title>메인 페이지 타이틀</title> </head>
    <body>
    <section>
        <p>메인 페이지 컨텐츠</p>
        <div>메인 페이지 포함 내용</div>
    </section>
    </body>
    </html>
    ```
    
- 결과는 아래와 같다.
    
    ```html
    <!DOCTYPE html>
    <html>
    <head>
        <title>메인 페이지 타이틀</title> </head>
    <body>
    <h1>레이아웃 H1</h1>
    
    <section>
        <p>메인 페이지 컨텐츠</p>
        <div>메인 페이지 포함 내용</div>
    </section>
    
    <footer>
        레이아웃 푸터
    </footer>
    </body>
    </html>
    ```
