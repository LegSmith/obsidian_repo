## 1. Java 메모리 구조 복습

Java 에서 메모리는 크게 `메서드 영역(Method Area)`, `스택 영역(Stack Area)`, `힙 영역(Heap Area)` 로 나뉜다.

![[image 18.png|image 18.png]]

- `메서드 영역(Method Area)` : 프로그램을 실행하는데 필요한 공통 데이터를 관리
    - **클래스 정보** : 클래스 _==**실행 코드**==_(바이트 코드), 필드, 메서드와 생성자 코드등 모든 실행 코드가 존재
    - **static 영역** : _==**static 변수**==_들을 보관
    - **런타임 상수 풀** : 프로그램을 실행하는데 필요한 _==**공통 리터럴 상수**==_를 보관
- `스택 영역(Stack Area)` : Java 실행 시 하나의 실행 스택(main 메서드)가 생성
    - 스텍 프레임은 지역변수, 중간 연산 결과, 메서드 호출 정보 등을 포함.
- `힙 영역(Heap Area)` : 객체(인스턴스)와 배열이 생성되는 영역.
    - _==**가비지 컬렉션(GC)이 이루어지는 주요 영역**==_, 더 이상 _참조되지 않는 객체는 CG에 의해 제거_된다.

  

> 참고 : 스택 영역은 더 정확히는 ==**각 스레드별로 하나의 실행 스택이 생성**==된다. 즉, 스레드 수 만큼 스택이 생성된다.

## 2. 스레드 생성

스레드를 생성하는 방법은 다음과 같다.

1. `Thread` 클래스를 상속 받는 방법
2. `Runnable` 인터페이스의 구현체를 생성하는 방법

### 2.1. 스레드 생성 - Thread 상속

```java
package thread.start;

public class HelloThread extends Thread {

    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName() + " : run()");
    }
}
```

- `Thread` 클래스를 상속하고, 스레드가 실행할 코드를 `run()` 메서드에 `재정의(Override)`한다.
- `Thread.currentThread()` 를 호출하면 _==해당 코드를 실행하는 스레드 객체를 조회==_
- `Thread.currentThread().getName()` 은 실행 중인 _==스레드의 이름을 조회==_

  

### 2.2. 스레드 실행

```java
public class HelloThreadMain {

    public static void main(String[] args) {

        System.out.println(Thread.currentThread().getName() + ": main() start");

        HelloThread helloThread = new HelloThread();

        System.out.println(Thread.currentThread().getName() + ": start() 호출 전");
        helloThread.start();
        System.out.println(Thread.currentThread().getName() + ": start() 호출 후");

        System.out.println(Thread.currentThread().getName() + ": main() end");
    }
}
```

- `Thread` 를 상속받은 객체를 실행할 때는 `run()` 메서드가 아닌 `start()` 메서드를 호출해야 한다
- `start()` 를 호출하면 `run()` 메서드를 호출해준다.
    
    ![[image 1 9.png|image 1 9.png]]
    
    > _==참고로 실행결과는 제각각일 수 있다.==_
    
- 위 실행결과를 보면 `main()` 메서드를 실행하는 스레드의 이름은 main 이다
- 그리고 `HelloThread` 스레드 객체를 생성한 다음 `start()` 를 호출하면 별도의 스레드를 하나 더 생성하고 `run()` 스택이 생긴다.
    
    - 스택이 생기는 시점은 `HelloThread` 객체를 생성할 떄가 아닌 `start()`를 호출할 때이다.
    
    ![[image 2 7.png|image 2 7.png]]
    
- 스레드에 이름을 주지 않으면 Java는 임의의 이름을 부여한다.
- ==**Thread-0 스레드는 run() 메서드의 스택 프레임을 스택에 올리면서 run() 메서드를 실행**==한다.

### 2.3. 흐름 분석

![[image 3 6.png|image 3 6.png]]

- 핵심은 _==**main 스레드는 Thread-0 스레드를 기다리지 않는 것**==_ 이다.
- 쉽게 말해 main 스레드는 본인의 실행코드를 실행시키고, Thread-0 의 실행코드에 간섭하지 않는다.
- _==**스레드 간 실행 순서는 보장하지 않는다.**==_
