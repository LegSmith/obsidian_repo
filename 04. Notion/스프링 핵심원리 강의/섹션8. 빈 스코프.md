## 빈 스코프란 ?

==**스코프란 빈이 존재할 수 있는 범위**==를 말한다.

### 스프링에서 지원하는 스코프

1. ==**싱글톤**==
    - 기본 스코프이며, 스프링 컨테이너의 시작과 종료까지 유지되는 스코프이다.
2. 프로토 타입
    - 스프링 컨테이너는 프로토타입 빈의 ==**생성과 의존관계 주입까지만 관여**==하고 더는 관리하지 않는 ==**매우 짧은 범위의 스코프**==이다.
3. 웹 관련 스코프
    - **request** : ==**웹 요청**==이 들어오고 나갈때까지
    - **session** : **==웹 세션==**이 생성되고 종료될 때까지
    - **application** : **==웹의 서블릿 컨텍스==**트와 같은 범위유지

## 프로토타입 스코프

- 기존 싱글톤 스코프는 ==**스프링 DI 컨테이너에서 빈을 관리**==하고 여러 사용자가 같은 빈을 요청해도 하나의 객체를 공유하였다.
- 프로토타입 스코프는 매번 같은 **==요청이 들어와도 그 시점에 프로토타입 빈을 생성하고, 의존관계 주입, 초기화까지만 처리==**한다. 그 다음부터는 스프링 DI 컨테이너가 관리하지 않고, Client가 관리해야 한다.

==**프로토타입 스코프는 생성,주입,초기화까지만 책임**==지고 관리는 Client가 한다.

  

### SingletonTest.java

```java
package hello.core.scope;

import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import org.assertj.core.api.Assertions;
import org.junit.jupiter.api.Test;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;
import org.springframework.context.annotation.Scope;

import static org.assertj.core.api.Assertions.*;

public class SingletonTest {

    @Test
    void singletonBeanFind(){
        AnnotationConfigApplicationContext ac = new AnnotationConfigApplicationContext(SingletonBean.class);

        SingletonBean singletonBean1 = ac.getBean(SingletonBean.class);
        SingletonBean singletonBean2 = ac.getBean(SingletonBean.class);

        System.out.println("singletonBean1 = " + singletonBean1);
        System.out.println("singletonBean2 = " + singletonBean2);

        assertThat(singletonBean1).isSameAs(singletonBean2);

        ac.close();
    }

    @Scope("singleton")
    static class SingletonBean{
        @PostConstruct
        public void init(){
            System.out.println("SingletonBean.init");
        }

        @PreDestroy
        public void destroy(){
            System.out.println("SingletonBean.destroy");
        }
    }
}
```

- 우리가 기존에 사용하던 싱글톤 스코프는 싱글톤 방식으로 객체를 공유하기 때문에 객==**체이름도 같고, 스프링 컨테이너가 종료되면 종료메서드도 호출**==해준다.

## 프로토타입 스코르프 - 싱글톤 빈과 함께 사용시 문제점

==**프로토타입 스코프의 빈과 싱글톤 빈을 함께 사용할 때는 의도한 대로 잘 동작되지 않으므로 주의**==해야 한다!!

```java
package hello.core.scope;

import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import org.assertj.core.api.Assertions;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;
import org.springframework.context.annotation.Scope;

import static org.assertj.core.api.Assertions.*;

public class SingletonWithPrototypeTest1 {

    @Test
    void singletonClientUsePrototype(){
        AnnotationConfigApplicationContext ac =
                new AnnotationConfigApplicationContext(ClientBean.class, PrototypeBean.class);

        ClientBean clientBean1 = ac.getBean(ClientBean.class);
        int count1 = clientBean1.logic();
        assertThat(count1).isEqualTo(1);

        ClientBean clientBean2 = ac.getBean(ClientBean.class);
        int count2 = clientBean2.logic();
        assertThat(count2).isEqualTo(2);

    }

    @Scope("singleton")
    static class ClientBean{
        private final PrototypeBean prototypeBean; //생성시점에 주입
        //싱글톤에서 관리가 되기 때문에 1개의 prototypeBeand을 공유한다.

        public ClientBean(PrototypeBean prototypeBean){
            this.prototypeBean = prototypeBean;
        }

        public int logic(){
            prototypeBean.addCount();
            return prototypeBean.getCount();
        }

    }

    @Scope("prototype")
    static class PrototypeBean{
        private int count = 0;

        public void addCount(){
            count++;
        }
        public int getCount(){
            return count;
        }

        @PostConstruct
        public void init(){
            System.out.println("PrototypeBean.init : "+this);
        }

        @PreDestroy
        public void destroy(){
            System.out.println("PrototypeBean.destroy");
        }
    }
}
```

- ㅅClientBean은 싱그톤스코프이고 ClientBean에서 PrototypeBean을 생성자 주입으로 주입을 받으면 ClientBean 객체 안에서 사용하게 된다.
- 이렇게 되면 여러 요청이 들어와도 PrototypeBean이 요청마다 생성되는게 아닌 ClientBean에서 관리하고 있는 PrototypeBean 1개로 로직을 처리하기 때문에 개발자가 의도한 Prototype스코프의 기능을 쓰지 못한다.
    
    ![[Untitled 2 21.png|Untitled 2 21.png]]
    
- 위와 같이 초기화시점에 객체타입을 반환하는 코드를 적었음에도 2번의 요청이들어와도 1번의 초기화 작업밖에 하지 못한것을 알 수 있다.

즉, 싱글톤 스코프와 클라이언트 스코프를 같이 사용하게 되면 개발자가 의도한대로 로직을 처리할 수 없다.

## 프로토타입 스코프 - 싱글톤 빈과 함께 사용시 `Provider` 로 문제해결

  

### ObjectFactory 사용

- ApplicationContext를 주입받으면 스프링코드에 의존적이고 단위테스트 코드작성시 어려움이 있다.
- 우리가 필요한 스프링빈을 조회해서 찾는것을 ==**Dependency Lookup(DL) 의존관계 조회(탐색)**==이라 한다.
- 이 때 스프링에서는 `ObjectFactory` 또는 `ObjectProvider` 를 제공하여 DL 기능을 사용한다.

```java
package hello.core.scope;

import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import org.assertj.core.api.Assertions;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.ObjectProvider;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;
import org.springframework.context.annotation.Scope;

import static org.assertj.core.api.Assertions.*;

public class SingletonWithPrototypeTest1 {

    @Test
    void singletonClintUsePrototype(){
        AnnotationConfigApplicationContext ac =
                new AnnotationConfigApplicationContext(ClientBean.class, PrototypeBean.class);

        ClientBean clientBean1 = ac.getBean(ClientBean.class);
        int count1 = clientBean1.logic();
        assertThat(count1).isEqualTo(1);

        ClientBean clientBean2 = ac.getBean(ClientBean.class);
        int count2 = clientBean2.logic();
        assertThat(count2).isEqualTo(1);

    }

    @Scope("singleton")
    static class ClientBean{
        @Autowired
        private ObjectProvider<PrototypeBean> prototypeBeanProvider;

        public ClientBean(ObjectProvider<PrototypeBean> prototypeBeanProvider) {
            this.prototypeBeanProvider = prototypeBeanProvider;
        }

        public int logic(){
            PrototypeBean prototypeBean = prototypeBeanProvider.getObject();
            prototypeBean.addCount();
            int count = prototypeBean.getCount();
            return count;
        }

    }

    @Scope("prototype")
    static class PrototypeBean{
        private int count = 0;

        public void addCount(){
            count++;
        }
        public int getCount(){
            return count;
        }

        @PostConstruct
        public void init(){
            System.out.println("PrototypeBean.init : "+this);
        }

        @PreDestroy
        public void destroy(){
            System.out.println("PrototypeBean.destroy");
        }
    }
}ㅇ
```

- 생성자 주입시 ObjectProvider<T>가 T타입의 스프링빈을 찾아 주입시켜준다. 즉 logic() 요청이 들어올때마다 getObject()로 새로운 프로토타입 빈이 생성되는것을 알 수 있다.
- ObjectProvider 의 ==**getObject()는 내부에서 스프링 컨테이너를 통해 해당 빈을 찾아서 반환하는 기능이다.**==

![[Untitled 3 13.png|Untitled 3 13.png]]

> ObjectProvider는 프로토타입 빈을 찾는 기능이 아닌 DL의 기능을 구현하기 위한 기능이다.

### 특징

- **ObjectFactory** : **기능이 단순**, 별도의 라이브러리 필요없음. ==**스프링에 의존**==
- **ObjectProvider** : ObjectFactory의 상속, 옵션, 스트림 처리등 편의 기능이 많고, 별도의 라이브러리 필요없음, ==**스프링에 의존**==

ObjectFactory , ObjectProvider 둘 다 **스프링에 의존적**이다.

### JSR-330 Provider

JSR-330 은 자바 표준이다. 이 방법을 사용하려면 javax.inject:javax.inject:1 를 gradle에 추가해줘야 한다.

- **스프링부트 3.0 미만**
    - `javax.inject:javax.inject:1` 라이브러리를 gradle에 추가해야 한다.
- **스프링부트 3.0 이상**
    - `jakarta.inject:jakarta.inject-api:2.0.1` 라이브러리를 gradle에 추가해야 한다.

```java
package hello.core.scope;

import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import jakarta.inject.Provider;
import org.assertj.core.api.Assertions;
import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.ObjectProvider;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;
import org.springframework.context.annotation.Scope;

import static org.assertj.core.api.Assertions.*;

public class SingletonWithPrototypeTest1 {

    
    @Test
    void singletonClientUsePrototype(){
        AnnotationConfigApplicationContext ac =
                new AnnotationConfigApplicationContext(ClientBean.class, PrototypeBean.class);

        ClientBean clientBean1 = ac.getBean(ClientBean.class);
        int count1 = clientBean1.logic();
        assertThat(count1).isEqualTo(1);

        ClientBean clientBean2 = ac.getBean(ClientBean.class);
        int count2 = clientBean2.logic();
        assertThat(count2).isEqualTo(1);

    }

    @Scope("singleton")
    static class ClientBean{

        @Autowired
        private Provider<PrototypeBean> prototypeBeanProvider;

        public int logic(){
            PrototypeBean prototypeBean = prototypeBeanProvider.get();
            prototypeBean.addCount();
            int count = prototypeBean.getCount();
            return count;
        }

    }

    @Scope("prototype")
    static class PrototypeBean{
        private int count = 0;

        public void addCount(){
            count++;
        }
        public int getCount(){
            return count;
        }

        @PostConstruct
        public void init(){
            System.out.println("PrototypeBean.init : "+this);
        }

        @PreDestroy
        public void destroy(){
            System.out.println("PrototypeBean.destroy");
        }
    }
}
```

- get() 메서드로 DL 을 하는거다. 코드가 단순하다.
- 단, 별도의 라이브러리가 필요하다
- 자바 표준이므로 스프링이 아닌 곳에서도 사용가능하다.
- Provider 는 딱 필요한 DL 정도의 기능만 제공한다.

## 웹 스코프

### 웹 스코프의 특징

- 웹 스코프는 ==**웹 환경에서만 동작**==한다.
- 해당 스코프의 종료시점까지 관리한다. 따라서 ==**종료 메서드**==가 호출된다.

### 웹 스코프의 종류

- **request** : ==**HTTP 요청 하나**==가 들어오고 나갈 때 까지 유지. **==HTTP 요청마다 별도의 빈 인스턴스 생성,관리==**된다.
- **session** : HTTP Session과 동일한 생명주기
- **application** : 서블릿 컨텍스트(`ServletContext`)와 동일한 생명주기
- **websocket** : 웹 소켓과 동일한 생명주기

  

### request 스코프

동시에 여러 HTTP 요청이 오면 정확히 어떤 요청이 남긴 로그인기 구분하기 어렵다.  
request 스코프를 사용하면 구분이 쉬워진다.

  

### MyLogger.java

- 클라이언트에 요청이 들어올때 request 스코프를 이용한 로그를 찍기위한 클래스

```java
package hello.core.common;

import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Component;

import java.util.UUID;

@Component
@Scope(value = "request")
public class MyLogger {

    private String uuid;
    private String requestURL;

    public void setRequestURL(String requestURL) {
        this.requestURL = requestURL;
    }

    public void log(String message){
        System.out.println("["+uuid+"]" + "["+requestURL+"]" + "["+message+"]");
    }

    @PostConstruct
    public void init(){
        uuid = UUID.randomUUID().toString(); //절대 겹치지않는 고유한 정보
        System.out.println("[" + uuid + "] request scope bean create:" + this);
    }
    @PreDestroy
    public void close(){
        System.out.println("[" + uuid + "] request scope bean close:" + this);
    }
}
```

## 프록시를 이용하여 코드 간결화

- `@Scope(”request”)` ← 이렇게 기입하면 proxyMode를 사용할 수 없기에 value 옵션을 적어준거다.
- proxyMode는 적용 대상이 클래스면 `TARGET_CLASS` 이고 인터페이스면 `_INTERFACES_` _를 입력하면 된다._

```java
package hello.core.common;

import jakarta.annotation.PostConstruct;
import jakarta.annotation.PreDestroy;
import org.springframework.context.annotation.Scope;
import org.springframework.context.annotation.ScopedProxyMode;
import org.springframework.stereotype.Component;

import java.util.UUID;

@Component
@Scope(value = "request", proxyMode = ScopedProxyMode.TARGET_CLASS)
public class MyLogger {

    private String uuid;
    private String requestURL;

    public void setRequestURL(String requestURL) {
        this.requestURL = requestURL;
    }

    public void log(String message){
        System.out.println("["+uuid+"]" + "["+requestURL+"]" + "["+message+"]");
    }

    @PostConstruct
    public void init(){
        uuid = UUID.randomUUID().toString(); //절대 겹치지않는 고유한 정보
        System.out.println("[" + uuid + "] request scope bean create:" + this);
    }
    @PreDestroy
    public void close(){
        System.out.println("[" + uuid + "] request scope bean close:" + this);
    }
}
```

  

- 프록시는 가짜 클래스를 만들어 요청이 들어오면 진짜 클래스를 호출해준다.
- 가짜 프록시 객체는 원본 클래스를 상속 받아서 만들어졌기 때문에 이 객체를 사용하는 클라이언트 입장에서는 원본인지 아닌지도 모르게 사용한다.(다형성)
