로그를 system.out.println() 출력문을 이용해 해도 되지만 ==**운영서버에서 println()문으로 로그를 남기면 서버과부하가 일어날 수 있기 때문에 로깅객체를 이용하여 로그를 남겨야한다.**== 혼자서 개발공부를 할때에는 print()문을 이용하여 로그를 남겨도 무방. 실무에서는 절대 그렇게 하면 안된다!

## 로그 단계

- 로그레벨 : ==**trace > debeg > info > warn > error**==
- 로깅레벨 설정법 : `application.properties` 파일에 적어둔다

    ```java
    logging.level.hello.springmvc = 로깅레벨
    ```

- 개발서버에서는 로깅레벨을 debug나 trace로 해두는게 좋다.
- 운영서버에서는 로깅레벨을 info 부터 해놓는게 좋다.

```java
				// trace, debug는 개발서버에서 적용하는게 좋다
        log.trace(" trace log={}",name);
        log.debug(" debug log={}", name);
        // info level 로 설정하면 info, warn, error 로그만 남는(운영서버에서만 적용)
        log.info(" info log={}", name);
        log.warn(" warn log={}", name);
        log.error(" error log={}", name);
```
