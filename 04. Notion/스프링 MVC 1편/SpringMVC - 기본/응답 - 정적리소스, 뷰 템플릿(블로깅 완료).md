  

스프링에서 응답데이터를 다루는 방법은 3가지가 있다  
1. 정적 리소스  
2. 뷰 템플릿 사용  
3. HTTP 메시지 사용

  

## 정적 리소스

- 스프링 부트는 /resources 경로안에서 정적 리소스를 관리한다.
- /static , /public 등의 클래스패스에서 정적 리소스를 제공하고 모든 시작 경로는 /resources 이다
- 정적 리소스는 해당파일 변경없이 그대로 사용하는 것이다.

## 뷰 템플릿

- 뷰 템플릿은 HTML 을 동적으로 생성하는 용도이다.
- 뷰 템플릿을 거쳐서 HTML이 생성되고, 뷰가 응답을 만들어서 전달한다.
- 스트링 부트는 뷰 템플릿 경로를 /resources/templates 로 지정된다.
- ModelAndView 방식은 코드가 길고 불편하다
    
    ```java
    @RequestMapping("/response-view-v1")
        public ModelAndView responseViewV1(){
            ModelAndView mav = new ModelAndView("response/hello").addObject("data", "hello!");
    
            return mav;
        }
    ```
    
- Model 객체를 사용한다
- 스프링부트에서 타임리프를 사용하면 뷰리졸버 경로를 자동으로 설정해준다. 즉 리턴값에 경로만 적어주면 된다.

```java
@RequestMapping("/response-view-v2")
    public String responseViewV2(Model model) {
        model.addAttribute("data","Cristiano Ronaldo");

        return "response/hello";
    }
```
