  

### 매핑 정보

- `@RestController`
    - `@Controller`는 반환 값이 String이면 View 이름으로 인신된다. 그래서 뷰를 찾고 뷰가 렌더링이 된다. → 뷰 리졸버
    - `@RestController`는 반환 값으로 뷰를 찾는게 아닌 HTTP 메시지 바디에 바로 입력한다
        - @ResponseBody 어노테이션을 이용하면 @Controller에서도 @RestController와 같이 사용이 가능하다
    - URL 은 여러개도 호출이 가능하다!!
        - `@RequestMapping({"/hello-basic","hello-go", ..."})`
        - 중괄호({})로 묶어서 콤마(,)로 구분하여 여러 URL을 지정할 수 있다.

  

### HTTP 메서드를 지정하지 않으면

- 모든 메서드를 허용하게 된다.
    - `@RequestMapping("/hello-basic")`
- Postman 을 통해서 get , post , patch , delete 로 실행해도 모두 응답한다.
- 따로 메서드를 지정할 수 있다.
    - `@RequestMapping(value = "/hello-basic" ,method = RequestMethod.``_GET_``)`

## PathVariable

- URL 경로에 들어가는 변수를 사용할 떄 쓰는 방법이다
- PathVariable 키값과 파라미터 변수명이 같으면 어노테이션만 붙여도 된다.
    
    ```java
    /**
    	* PathVariable 사용
    	* 변수명이 같으면 생략 가능
    	* @PathVariable("userId") String userId -> @PathVariable userId
    	* String mappingPath(@PathVariable("userId") String data) === String mappingPath(@PathVariable String userId)
    */
    @GetMapping("/mapping/{userId}")
    public String mappingPath(@PathVariable String userId){
        log.info("mappingPath userId={}",userId);
        return "ok";
    }
    ```
    
- 다중으로 PathVariable을 받을려면 아래와 같이 하면된다.
    
    ```java
    /**
    	 * PathVariable 다중사용
     */
    @GetMapping("/mapping/users/{userId}/orders/{orderId}")
    public String mappingPath(@PathVariable String userId, @PathVariable String orderId){
        log.info("mapping Path userId={}, orderId={}",userId,orderId);
        return "ok";
    }
    ```
    

  

### 특정 헤더 조건 매핑

- 헤더에 조건을 걸어 매핑할 수 있다.
    
    ```java
    /**
    	 *특정 헤더로 추가 매핑
    	 * headers="mode",
    	 * headers="!mode"
    	 * headers="mode=debug"
    	 * headers="mode!=debug" (! = )
     */
    @GetMapping(value = "/mapping-header", headers = "mode=debug")
    public String mappingHeader() {
        log.info("mappingHeader");
        return "ok";
    }
    ```

## 미디어 타입 조건 매핑 - Content-Type, consume

- consumes은 요청 Header에 Content-Type을 정할때 사용한다.
    
    ```java
    /**
    	 * Content-Type 헤더 기반 추가 매핑 Media Type
    	 * consumes="application/json"
    	 * consumes="!application/json"
    	 * consumes="application/*"
    	 * consumes="*\/*"
     * MediaType.APPLICATION_JSON_VALUE
     */
    @PostMapping(value = "/mapping-consume", consumes = "application/json")
    public String mappingConsumes() {
        log.info("mappingConsumes");
        return "ok";
    }
    ```
    

## **미디어 타입 조건 매핑 - HTTP 요청 Accept, produce**  

- produce는 요청 헤더에 Accept 이다.
    
    ```java
    /**
    	 * Accept 헤더 기반 Media Type * produces = "text/html"
    	 * produces = "!text/html"
    	
    	 * produces = "text/*"
    	 * produces = "*\/*"
     */
    @PostMapping(value = "/mapping-produce", produces = "text/html")
    public String mappingProduces() {
        log.info("mappingProduces");
        return "ok";
    }
    ```
    
- produces를 text/html로 지정했는데 요청헤더에 json 타입을 정하면 404 오류가 난다.
