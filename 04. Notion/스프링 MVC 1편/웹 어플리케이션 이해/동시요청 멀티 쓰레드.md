# 쓰레드(Thread)

- 애플리케이션 코드를 하나하나 순차적으로 실행하는 것
- 동시 처리가 필요하면 쓰레드를 추가로 생성
- 단일 쓰레드를 사용하면 아래와 같이 처리지연발생( 쓰레드를 추가생성 ! )

    ![[Untitled 1 55.png|Untitled 1 55.png]]

    ![[Untitled 2 39.png|Untitled 2 39.png]]

    ### 요청마다 쓰레드 생성시 장단점

    - 장점
        - 동시 요청을 처리
        - 리소스(CPU,메모리)가 허용할 때 까지 처리가능
    - 단점
        - ==**생성비용이 매우 비싸다!!( 쓰레드 생성하면 응답 속도가 늦어진다)**==
        - 쓰레드는 컨텍스트 스위칭 비용이 발생한다.

# 쓰레드 풀

- ==**쓰레드를 빌려쓰고 다시 반환 하는것**==

![[Untitled 3 29.png|Untitled 3 29.png]]

- 특징
    - 필요한 쓰레드를 쓰레드 풀에 보관하고 관리.
    - 최대 생성 가능한 쓰레드의 최대치를 관리( 톰캣은 최대 200개, 변경가능함 )
- 쓰레드가 필요하면 쓰레드 풀에서 꺼내고 사용종료하면 쓰레드 풀에 반납
- 쓰레드가 모두 사용중이면 요청을 거절하거나 특정 숫자 만큼 대기하도록 설정 가능


## 실무 팁
