## 1. IPv4 기본구조

- `IPv4` 의 기본구조는 ==**8bit의 데이터가 총 4개, 즉 32bit**==로 이루어져 있다.
- `.(Dot)`로 구분되며 앞에 _**3개의 8bit 데이터**_는 `Network ID` 이고, _**마지막 1개의 8bit 데이터**_는 `Host ID` 이다.

## 2. L3 IP Packet

- 이전 L2 레이어에서는 데이터 단위를 Frame 이라 불렀다.
- L3 레이어에서는 `L3 IP Packet` 이라 부른다. L2에서와 똑같이 **데이터단위**이다.
- L2,L3 뿐만 아니라 모든 데이터들은 `Header` 와 `Payload` 로 구분되어 있다.
- 이 때, 데이터의 최대 길이를 `MTU(Maximun Transmission Unit)` 이라 하는데 보통 `1500byte(1.4 kb)`정도의 크기를 가진다.

    ![[image 1 12.png|image 1 12.png]]

## 3. Encapsulation / Decapsulation

- L4 Segment 부터 L3 Packet , L2 Frame 순으로 데이터를 감싸는 것을 `Encapsulation` 라 한다.
- 반대로 L2 Frame 부터 벗기고, L3, L2 순으로 데이터를 벗겨내는 컷을 `Decapsulation` 라 한다.

## 4. 패킷의 생성, 전달, 소멸

### 4.1. 패킷의 흐름 비유

- 패킷은 네트워크 상에서 데이터인데, 쉽게 비유하면 택배와 같다.
    1. ==**철수는 영희**==에게 택배로 보낼 ==**물건을 상자에 담는다.**==
    2. 받는사람,보내는사람 주소, 보내는사람 이름, 받는사람 이름이 담긴 ==**송장**==을 상자에 붙힌다.
    3. 철수네 ==**현관에서 택배기사님에게 상자를 주면, 택배기사님은 택배차에 물건을 싣는다.**==
    4. 택배물류체계에 따라 송장에 적힌 영희네로 간다.
    5. 영희네에 영희대신 엄마나 동생이 받으면 ==**받는이를 보고 영희에게 전달**==한다.
    6. 영희는 ==**물건을 받고 상자를 연다.**==
- 위 과정들을 네트워크 용어로 풀면 다음과 같다.
    1. `데이터(물건)`를 `패킷(상자)`에 담는다.
    2. 목적지주소, 출발지주소 등의 정보를 헤더에 저장한다.(송장)
    3. `Host(철수네)` 와 `Gateway(택배기사님)` 가 만나는 접점에서 현관이라는 `인터페이스`를 통해 패킷이 나간다.
    4. `Gateway` 는 목적지주소를 보고 `라우팅`을 한 후 ==**목적지주소로 패킷을 보낸다.**==
    5. 목적지주소로 온 후 `Port(받는이)`를 보고 최종 전달한다.
    6. 최종적으로 패킷을 연 후 데이터를 확인한다.

### 4.2. 패킷의 구체적름

`Process` 가 `Internet` 을 통해 정보를 전달할려고 할때 전체 흐름을 보면 다음과 같다.

- `Data` 는 `Socket` 이라는 인터페이스를 통해 `Ketnel mode` 에 접근하여 `Data` 를 전달하는데, 이 때 ==**TCP 계층을 만날 때 Data 앞에 TCP 헤더를 붙힌다.**== → `Segment`
    - `Socket` 은 쉽게말해 **==Kernel 에 있는 TCP/IP 를 추상화한 File==** 이다.
- IP 계층을 만나면 Segment 앞에 IP 헤더가 붙는다. → 이를 `Packet` 이라 부름.
- 그리고 Driver 를 통해 나갈려고 할때 Packet 앞에 Ethernet Frame 헤더가 붙는다. → 이를 `Frame` 이라 부름.
- 이제 LAN 카드를 통해 `L2 Access Switch` 를 만나고, `Router Gateway` 를 만나서 Internet 으로 나간다.

## 5. 계층별 데이터 단위

### 5.1. 계층별 데이터단위

OSI 7 계층의 각 계층별 구체적인 단위가 아닌 ==**H/W 와 S/W(User mode 와 Kernel mode)을 기준**==으로 쉽게 나타낸다.

![[image 5 7.png|image 5 7.png]]

- L1 ~ L2 계층 : `Frame`
- IP 계층 : `Packet`
- TCP 계층 : `Segment`
- Socekt 계층 : `Stream`

### 5.2. Stream 이란?

- `Stream` 은 쉽게 말해 ==**연속적으로 이어진 크기를 정확히 알 수 없는 큰 데이터**==이다.
- `Stream` 은 시작은 있지만 끝이 언제인지 즉, Stream 의 사이즈를 정확히 알 수 없다.
- 왜냐하면 Stream 의 끝은 Application 수준에서 정의하기 때문이다.
- 다른 계층인 Segment 에서는 `MSS(Maximum Segment Size)`가 있고, Packet 에서는 `MTU(Maximum Transmission Unit)`을 통해 길이를 알 수 있다.
    - MTU 는 보통 1500Byte 이고, MSS 는 보통 1460 Byte이다.
- 만약 ==**Stream 이 MTU와 MSS 보다 큰 4MB 라면 Segment 가 만들어질때 Segmentation(분할)**== 하게 된다.
    - 이 때 일정단위(MSS)로 Stream 을 잘라진다.

## 6. TCP/IP 송수신구조

### 6.1. 송수신 흐름

File 을 전송할 때 어떻게 메모리에 올라가고 송/수신하는지 정리한다.

- 우선 파일을 인터넷을 통해 전송할 때는 **==Process(Application 레이어)에서 Socket 을 통해 전송==**한다.
- 이 떄, Process 에서도 Buffer 가 있고, Socket 에서 `I/O(Send/Receive) Buffer` 가 있다. → 네트워크 통신시에는 `Send/Receive Buffer` 라 한다.
    - **출발지(Client)**에서 **==저장장치(HDD or SSD) 에 있는 1.4MB 용량을 가진 A.bmp(비트맵 파일)을 전송==**한다고 하자.
    - 이 때 Process 에서는 1.4 MB 통째로 Buffer 에 올릴 수 있지만 **==보통 쪼개서 Buffer 에 넣는다.(정확히는 Copy 해서 메모리에 적재한다.==**
    - 그리고 ==**Process Buffer 에서 Socket Send Buffer 로 Copy**== 하여 적재한다. → 이 과정까지는 데이터 단위가 `Stream` 이다.
    - 그리고 Socket 을 통해 Kernel 로 넘어가서 L4(TCP) 로 가게 되는데, 이 때 `분할(Segmentation)`이 일어난다. 그리고 **분할된 데이터들은 순번이 정해진다**. → 이 과정에서 데이터 단위는 `Segment` 이다.
    - 그리고 L3(IP) 로 가게 될 때 각 순번이 정해진 데이터들은 `Packet` 으로 데이터가 감싸진다. → 이 과정에서 데이터 단위는 `Packet` 이다.
    - 그리고 Driver 를 통해 L2 로 가게 되는데 이 때 데이터단위는 `Frame` 인데, 이 Frame 은 목적지주소로 한번에 가는게 아닌 수시로 목적지가 바뀌어서 최종적으로 목적지주소로 전달이 된다. → 이렇게 ==**Frame 까지 데이터가 감싸지는 과정**==을 `Encapsulation` 이라 한다.
    - 목적지(Server) 로 도착하면 Frame 을 뜯어내서 Packet 을 꺼내게 된다.
    - 그리고 위 과정에서 반대로 데이터를 뜯어내면서 Segment 를 꺼내고, Socekt을 통해 Socket Receive Buffer에 데이터를 담아서 Process Buffer에 담기게 된다. → 이렇게 반대로 ==**데이터를 벗겨내는 과정**==을 `Decapsulation` 이라 한다.
    - 위와 같은 Network 통신은 계속 일어나는데 Buffer 에서는 데이터가 채워지고, 데이터를 퍼와서 읽는 과정들이 동시에 일어난다.
    - 하지만 **==TCP 통신은 데이터를 보내면 잘 보냇다는 응답을 보내주기 때문에 순서대로 통신이 가능==**하다.

### 6.2. Network 장애

네트워크는 생각보다 불안정하기 때문에 장애가 자주 일어난다. 다음 장애 예제들은 TCP 장애들이다.

1. `Loss(유실)` : TCP 통신 과정 중 데이터가 유실 되는 경우이다. → ==**보통 네트워크 상 오류**==
2. `Retransmission` : TCP 통신 중 잘 받았다는 `응답(ACK)`을 주는데, 만약 응답이 오지않으면 데이터를 다시 보내는 경우가 있다. → ==**네트워크 이거나 Ent-Point 오류**==
    - 종종 아주 간발의 차이고 ACK 를 보냈는데 다시 데이터가 오는 경우가 있다.
3. `Out of order` : `Segment` 가 순번대로 오지않는 경우 → ==**네트워크 오류**==
    - 예: 1→2→4→3 순서대로 Segment 가 도착
4. `Zero window` : `Window Size` 가 꽉차서 0(Zero)가 되면 Full 이 나서 수신자가 수신을 못함. → ==**Ent-Point 오류**==
    - **==네트워크 송수신 속도가 Buffer 를 비워내는 속도보다 빨라서==** 장애가 일어난다.
    - 보통 Buffer 에 Segment 가 쌓일 때는 여유공간이 있다. ==**수신측 Buffer 에 여유공간의 메모리 크기**==를 `Window Size` 라 한다.
