## 1. NIC + (L2)Frame + LAN card + MAC

### 1.1. NIC

- `NIC(Network Interface Card)`는 흔히 `LAN(Local Area Network)` 카드이다.
    - `LAN` < `MAN` < `WAN` 이 네트워크 규모 순이다.
- NIC도 유/무선이 있지만 굳이 구별하지는 않는다.
- `NIC`은 ==**H/W 이며 MAC 주소**==를 갖는다.

### 1.2. L2 Frame

- L2 Frame 은 데이터 단위이다.
    - 속도가 1Gbps 는 1초에 1bit 이다.

## 2. L2 Switch

### 2.1. L2 Access switch

- End-point 와 직접 연결되는 스위치
- **==MAC 주소를 근거로 스위칭==**
- 아래 그림의 각각 구멍들은 Port 이고 해당 Port 에 LAN 카드를 꽂아서 PC와 연결한다.

    ![[image 20.png|image 20.png]]

- PC가 있고 `L2 Access switch`에 연결이 되면 `Link-up` 이라 한다. 반대로 연결이 끊기면 `Link-down` 이라 한다.
- 그리고 연결된 선이 Router 같은 상위 L3로 연결이 되면 `UpLink` 라 한다.

    ![[image 1 11.png|image 1 11.png]]


### 2.2. L2 Distribution switch

- 보통 PC 에서 LAN 카드가 L2 Access 스위치로 가고 , L2 Access 스위치에서 어떤 스위치를 거쳐서 상위의 Router 로 연결된다.
- ==**L2 Access 스위치를 위한 스위치**==이다.
    - VLAN(Virtual LAN)기능을 제공
- **==L2 Access 스위치들을 모아서 Switching 을 하는 장치==**이다.
- 1,2,3번 PC 끼리 통신할 떄는 Access 스위치만 거쳐가기 때문에 문제 없다.
- 하지만 ==**2번 PC와 5번 PC가 통신을 할려면 Access 스위치를 거치고 Distribution 스위치를 거쳐서 다시 4,5,6에 연결되어 있는 Access 스위치를 거쳐야 통신**==이 가능하다.

    - 그래서 Distribution 스위치는 Access 스위치보다 비싸다.

## 3. LAN과 WAN의 경계 그리고 Broadcast

### 3.1. Broadcast란?

- roadcast 는 직역하면 방송이다.
- **==Broadcast는 효율을 떨어뜨린다.==** `Broadcast` 와 반대되는 것은 `Unicast` 이다.
- ==**Broadcast 의 범위를 최소화 시키는 것이 중요**==하다 !
- Broadcast 주소라는 매우 특별한 주소가 존재한다.(MAC, IP 모두 존재)
    - 2진수로 봤을 때 전부 1로 되어 있으면 Broadcast 주소이다.
- 아래 그림을 예시로 1번 PC가 Broadcast 를 하면 모든 장치에게 알리게 되고, Broadcast 통신이 끝날 때까지 다른 PC들은 통신을 할 수 없다.

- 위와 같은 이유 떄문에 ==**Broadcast 를 최소화**== 해야 한다 !

### 3.2. OSI 7 Layer 관점

- 보통 OSI 7계층에서 그게 S/W 와 H/W 로 나뉘는데, 이 때 ==**H/W 를 영어로 Physical이라 하고 S/W 를 Logical**== 이라 한다.
- L2 계층에서는 MAC 주소로 식별하고, L3 계층에서는 IPv4 주소로 식별한다.
- L1~L2는 물리적이다. 그리고 L3 부터는 논리적(Logical)이다. 즉, 실체가 없다.
- 그리고 IT 쪽에 빗대면 Logical 은 `Virtual` 과 같다.
- 그리고 H/W 를 S/W로 구현하는 기술을 `Virtual Machine` 라 한다.
