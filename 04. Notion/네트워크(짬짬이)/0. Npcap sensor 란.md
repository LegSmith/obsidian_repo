컴퓨터의 세계는 크게 하드웨어(H/W)와 소프트웨어(S/W)로 나눌 수 있습니다.  
소프트웨어는 다시 `User Mode`와 `Kernel Mode` 로 구분됩니다. 즉, 컴퓨터 시스템은 `하드웨어`, `User Mode`, `**Kernel Mode**`라는 세 가지 계층으로 구성됩니다.

### 1. 기본적인 구조

- `TCP/IP` 는 `Kernel Mode`에 구현되어 있다.
- 하드웨어에는 `네트워크 인터페이스 카드(NIC, Network Interface Card)`가 구현되어 있다.
- `NIC`는 _==**소프트웨어의 Driver에 의해서 구동**==_된다.
- `User Mode` 에는 어플리케이션 `Process` 가 구현되어 있다.
- **어플리케이션 Processs는 Kernel 에 진입할 수 가 없다.** ==**Kernel 이 지정해준 인터페이스**==로만 진입이 가능한데 이 때 인터페이스가 `Socket` 이다.
- TCP 는 `L4` 이고, IP 는 `L3` 라 부른다.
- `Socekt` 에서 데이터 단위는 `Socket Stream` 이다.
- Socket 은 한꺼번에 보내지 못하기 때문에 쪼개는데 이 단위 하나하나를 Segment 라 한다.
- `Segment` 는 L4 에서 데이터의 단위 이다. 최대크기는 `MSS(Max Segment Size)` 라 한다.
- L3 에서는 데이터 단위가 `Packet` 이다.
- 그리고 L2 에서 데이터 단위는 `Frame` 이다.

### 2. Npcap(Sensor)의 역할

- L3에서 Packet 이 L2로 넘어갈 때 Frame 으로 변하는데 이 떄 `Npcap(Sensor)`가 _==**Frame을 감지해서 사용자에게 알려준다.**==_
- 실제 ==**NIC를 구동해서 정보(데이터)가 전달되기 바로 직전에 Frame 을 전부 캡쳐해서 사용자에게 보여주는 것**==이다.
- 이 때, `Wireshark` 프로그램은 ==**Npcap이 캡쳐한 Frame들을 사용자에게 보여준다. 사용자는 이를 통해 데이터를 해석하고 분석**==한다.
