# IP(Internet Protocol)

---

### IP 의 역할

- ==**지정한 IP 주소(IP Address)에 데이터 전달**==
- ==**패킷(Packet)이라는 통신 단위**==로 데이터 전달

### 패킷

- 패킷에는 출발 IP 주소와, 목적지 IP 주소, 데이터를 담아서 **==노드들을 통해 목적지까지 이동==**한다.

### IP 프로토콜의 한계

- **비연결성**
    - 패킷을 받을 대상이 없거나 서비스 불능 상태여도 패킷 전송
- **비신뢰성**
    - 패킷이 중간에 사라지거나 패킷이 순서대로 안온다면 ?
- **프로그램** 구분
    - 같은 IP 를 사용하는 서버에서 통신하는 애플리케이션이 둘 이상이라면 ?

# TCP, UDP

---

![[Untitled 34.png|Untitled 34.png]]

- IP 패킷 정보 : 출발지 IP, 목적지 IP, 기타, 전송데이터 가 들어있다.
- TCP 세그먼트 : 출발지 PORT, 목적지 PORT, 전송 제어, 순서, 검증정보 등등

## TCP : **전송 제어 프로토콜(Transmission Control Protocol)**

- **연결지향** - ==**TCP 3 way handshacke(가상연결)**==
- **데이터 전달 보증** : **==응답을 해주기 떄문에 전송한 측에서는 전달됐다는 것을 인지==**한다. → TCP 세그먼트
- **순서 보장** : 중간에 패킷을 유실하면 **==유실한 패킷부터 다시 보내준다.==** —> TCP 세그먼트
- **==신뢰할 수 있는 프로토콜==**

## TCP 3 way handshake

**_물리적인 연결이 아닌 가상연결이다_**

- **SYN** : 접속 요청
- **ACK** : 요청 수락

## UDP(User Datagram Protocol)

- 기능이 거의 없다 !!
- 순서보장, 데이터 전달보증, 3 way handshake → 전부 안된다.
- ==**빠르다, IP 와 거의 같지만 PORT 정보, 체크섬 정도만 추가됐다.**==

# PORT

---

**==같은 IP 를 사용할 때 여러 애플리케이션이 있다면 PORT 로 구분해서 통신한다==**

- 0 ~ 65535 : 할당 가능
- 0 ~ 1023 : 잘 알려진 포트, 사용하지 않는 것이 좋다.
    - `FTP` : 20, 21
    - `TELNET` - 23
    - `HTTP` - 80
    - `HTTPS` - 443
