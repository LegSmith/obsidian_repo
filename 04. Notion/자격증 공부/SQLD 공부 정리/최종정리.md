## SQL 명령문 개괄

---

- **연산순서**
    - From → Where → Group by → Having → Select → orderBy
- **DML** : `SELECT` , `INSERT` , `DELETE` , `UPDATE`
- **DCL** : `GRANT` , `REVOKE`
- **DDL** : `ALTER` , `CREATE` , `DROP` , `MODIFY` , `TRUNCATE`...
- **TCL** : `COMMIT` , `ROLLBACK`

## Select

---

- `Distinct`(중복제외) : 10,10,20,20,20 → 10,20

    `Distinct deptno,mgr` = `Distinct (deptno,mgr)` → 실제론 괄호를 치지 않는다.

- ==As==
    - `SELECT` : _**as 생략가능**_ , 컬럼명에 띄어쓰기
    - `FROM` : _**as 사용불가**_
- `concat()` : concat 은 ==**인수가 반드시 2개**==
    - `+` : SQL Server
    - `||` : Oracle

## 논리연산자

---

- `AND` : A and B → A 와 B 를 모두 만족
- `OR` : A or B → A 이거나 B
- `NOT` : A, B 싫엉

### ==연산순위 → NAO==

1. `NOT`
2. `AND`
3. `OR`

    ```sql
    NOT <조건> -- 1번
    AND <조건> -- 2번
    NOT <조건> -- 3번
    OR <조건> -- 4번

    -- 순서
    (1번 AND 2번 AND 3번) OR 4번
    ```


## SQL 연산자

---

- `BETWEEN` : `A BETWEEN 1 AND 2` → 1≤A≤2
- `IN` : `A IN (1,2,3)` → A=1 OR A=2 OR A=3

### ==**무조건나온다 !!**==

- `LIKE`
    - _ : 한글자
    - % : 0 이상 글자
- `ESCAPE`
    - ==**와일드카드( _ % ) 를 문자로 취급**==

        ```sql
        -- ename LIKE 'A_A' 를 찾는다
        ename LIKE 'A@_A' escape '@'; -- @자리에는 아무문자 가능 !!
        ```

- `ROWNUM` (Oracle)
    - where 조건절에서 Rownum = 1 포함
    - ==**order by 가 가장 마지막에 실행되므로 rownum≤3 을 뽑은 후에 정렬을 한다.**==

        ```sql
        SELECT empno, sal
        FROM emp
        WHERE rownum <=3
        ORDER BY sal DESC;

        -- 출력결과(sal 정렬전) : 정렬전에 rownum 이 먼저 실행
        empno sal
        			1900
        			1400
        			1300
        			...

        -- 출력결과(sal 정렬후) : rownum 으로 3개를 뽑고 
        empno sal
        			1900
        			1400
        			1300
        			...
        ```

- `TOP` (SQL Server)
    - SELECT 절에 쓴다.
    - `TOP(n) 컬럼명` → 컬럼명을 출력할 때 상위 n개

## ★★★★==NULL==★★★★

---

- **NULL 의 정의**
    - 부재 , 모르는 값
    - null + 2 = null , null * 2 = null → 산술연산
    - null = null , null = 2 → 알수없음(Unknown)
    - where(조건절) 에 들어가면 False 와 같다.
    - 정렬 상 의미
        - Orcle : 오름차순일 때 가장 마지막에 나온다
        - SQL Server : 오름차순일 때 가장 처음에 나온다.
- `NVL` : (값1 , 값2) → ==**값1 IS NULL 값2 아니면 값1**==
- `NVL2` : (값1, 값2, 값3) → **==값1 IS NULL 값3, IS NOT NULL 값2==**
- `ISNULL` : (값1, 값2) → NVL 과 같다.
- `NULLIF` : (값1, 값2) → ==**값1 = 값2 null , 다르면 값1**==
- `COALESCE` : ==**Null 이 아닌 첫 번째 값**==

## 정렬

---

- **정렬의 특성**
    - ==**가장 마지막 실행**==
    - 성능이 느려질 수 있다.
    - **NULL 값 과의 관계(Oracle 에서는 무한대기 때문에 가장 큰값)**
- **컬럼 번호로 정렬**
    - ORDER BY 1 → 1번째 컬럼 기준
    - **==출력되는 컬럼의 수보다 큰값이 안된다 !==**
- **인수 2개 정렬**
    - `ORDER BY sal DESC, ename ASC` : sal 이 같으면 ename을 오름차순으로 정렬
- _**SELECT 에 없는 컬럼으로 정렬 가능 !!**_
