## Section 1. 리눅스의 개요

### 1. 리눅스의 특징

- 오픈 소스 운영체제
- 멀티유저, 멀티테스킹 운영체제
- 다중스레드를 지원
- 여러 종류의 파일시스템을 지원

### 2. 리눅스의 장단점

- 리눅스는 유닉스(Unix)와 호환 가능
- PC용 OS보다 안정적
- 하드웨어 기능을 효과적으로 사용
- 오픈소스 OS
- 무료기 때문에 문제 발생 시 기술지원을 받기 어렵다.
- 한글 지원 미흡

### 3. 리눅스 디렉터리

`커널(Kernel)`이란 컴퓨터 하드웨어아 프로세스를 잇는 인터페이스 역할

|**디렉터리**|**저장 내용**||
|---|---|---|
|`/`|==**최상위 디렉터리(루트 디렉터리)**==, 모든 디렉터리의 출발점||
|`/boot`|==**부트 디렉터리**==, 부팅 시 _==**커널 이미지의 부팅 정보 저장**==_ 파일||
|`/proc`|==**시스템 정보 디렉터리**==, 커널 기능을 제어_==**현재 실행되는 프로세스와 실제로 사용되는 장치, 하드웨어 정보 저장**==_||
|`/lib`|==**공유 라이브러리 디렉터리**==||
|`/bin`|==**기본적인 명령어**==가 저장된 디렉터리_==**root 사용자와 일반 사용자가 함께 사용할 수 있는 명령의 디렉터리**==_||
|`/dev`|==**시스템 디바이스 파일**==들을 저장하는 디렉터리||
|`/etc`|==**시스템 환경 설정 파일**== 저장 디렉터리||
|`/root`|==**시스템 관리자용**== 홈 디렉터리||
|`/sbin`|==**관리자용 시스템 표준 명령 및 시스템 관리**==와 관련된 실행 명령어 저장||
|`/usr`|==**사용자 디렉터리**==로 사용자 데이터나 애플리케이션 저장||
|`/home`|사용자 계정 디렉터리로 계정들의 홈 디렉터리가 위치==**일반 사용자들이 로그인 시 처음으로 위치하게 되는 디렉터리**==||
|`/var`|가변 자료 저장 디렉터리==**로그 파일이나 메일 데이터 저장**==||
|`/tmp`|각종 프로그램이나 프로세스 작업 시 ==**임시로 생성되는 파일 저장**==  <br>모든 사용자에 대해서 읽기와 쓰기가 허용||
|`/mnt`|파일 시스템을 일시적으로 마운트 할 때 사용  <br>`마운트(Mount)`란 _==**리눅스에서 다른 파일 시스템을 현재 디렉토리에 임시로 연결**==_하는 작업||
|`/lost+found`|결함이 있는 파일에 대한 정보가 저장되는 디렉터리||

### 4. 리눅스 배포판

- **슬랙웨어 리눅스** : 배포판 가운데 가장 먼저 대중화
- **데비안** : GNU의 공식적인 후원을 받고있는 유일한 배포판
- **우분투** : 데비안 GNU/리눅스에 기초한 운영체제
- **레드햇** : 미국의 레드햇사가 개발하던 리눅스 배포판
- **RHEL** : 레드햇이 개발하여 판매하고 있는 상용 리눅스 배포판
- **페도라** : 리눅스 커널에 기반한 OS와 레드햇의 후원가 개발 공동체의 지원 아래 개발된 배포판
- **CentOS** : 업스트림 소스인 레드햇 엔터프라이즈 리눅스와 완벽하게 호환되는 무료 기업용 컴퓨팅 OS
- **수세** : 독일에서 출시된 배포판으로 유럽에서 인기를 누리고 있다.

## Section 2. 리눅스 라이센스

### 종류

- **GNU** : `GNU’s Not UNIX` 의 약자
- **자유 소프트웨어 재단** : 1985년 리처드 스톨만이 설립한 재단
- **오픈 소스 소프트웨어** : 1998년 _==**일부 커뮤니티에서 ‘자유 소프트웨어 재단’ 대신 ‘오픈 소스 소프트웨어’라는 용어를 사용**==_하기 시작
- **GNU GPL** : `GPL(General Public LIcense)`은 **자유 소프트웨어 재단에서 만든 Free 소프트웨어 라이센스**이다.
- **GNU LGPL** : `LGPL(Lesser General Public License)`은 GPL보다 훨씬 완화된 조건의 공개 소프트웨어 라이센스, _==**LGPL이 적용된 라이브러리를 이용하여 개발하였을 경우 프로그램 소스코드는 공개하지 않아도 된다.**==_
- **BSD 라이센스** : `BSD(Berkeley Software Distribution)`는 캘리포니아 대학의 자유소프트웨어 저작권의 한 가지
- **아파치(Apache) 라이센스** : 아파치 소프트웨어 재단에서 자체적으로 만든 소프트웨어 라이센스 규정
- **MIT 라이센스** : MIT(Massachusetts Institute of Technology)는 BSD 라이센스를 기초로 작성된 BSD 계열 라이센스 중 하나
- **MPL(Mozilla Public License)** : _==**소스코드와 실행파일의 저작권을 분리**==_

  

## Section 3. 기본 설치 및 유형

### 1. 리눅스 설치의 개요

- 리눅스는 단 하나의 제품 또는 한 종류의 제품군만 있는 것이 아니다.
- ==**리눅스 배포판마다 설치 환경과 설치 과정이 다르다.**==
- 패키지에 따라 **데스크탑형**, **서버형**, **사용자 정의형**으로 구분한다.
- 설치 전에 시스템이 있는 모든 파일을 백업
- 부팅 드라이브를 다시 파티션 하는 경우라면, OS의 부트로드를 다시 설치할 수도 있고, 더 많은 경우에 OS 전체를 해당 파티션에 다시 설치

### 2. 리눅스 설치를 위한 하드웨어 정보 파악

- 하드웨어 정보
- 하드웨어 호환성
- 네트워크 설정

  

  

## Section 4. 파티션

### 1. 파티션 특징과 종류

`파티션(partition)`이란, _**하나의 물리적 디스크를 여러 개의 논리적인 디스크로 분할**_하는 것

|종류|설명|
|---|---|
|**주 파티션**|==**부팅이 가능한 기본 파티션**====**하나의 하드디스크에 최대 4개의 주 파티션 분할**== 가능  <br>하드디스크를 4개 이상의 파티션으로 사용해야 할 때 하나의 확장 파티션을 설정하여 확장 파티션 안에 여러 개의 논리 파티션을 분할하여 데이터 저장|
|**확장 파티션**|주 파티션 내에 생성, ==**하나의 물리적 디스크에 1개만 생성**==  <br>파티션 번호는 1~4번이 할당==**데이터 저장 영역을 위한 것이 아니라 논리 파티션을 생성**==|
|**논리 파티션**|==**확장 파티션 안에 생성**==  <br>논리 파티션은 ==**12개 이상 생성하지 않는 것을 권고**==하며 5번 이후의 번호가 붙여짐|
|**스왑 파티션**|==**하드디스크의 일부를 메모리처럼 사용하는 영역**==  <br>주 파티션 또는 논리 파티션에 생성  <br>프로그램 실행 시 부족한 메모리 용량을 하드디스크로 대신 리눅스 설치시에 반드시 설치되어야 하는 영역  <br>스왑 영역의 크기는 메모리의 2배를 설정하도록 권고|

### 2. 파일 시스템

- **파일 시스템**은 _==**OS가 파일을 시스템의 디스크 파티션상에 구성하는 방식**==_
- 일정한 규칙을 가지고 파일을 저장하도록 규칙 방식을 제시
- ==**파티션에 파일 시스템이 없으면, 파일 시스템 생성을 거쳐야 사용이 가능**==하다.
- 리눅스는 고유의 파일 시스템뿐만 아니라 다양한 파일 시스템을 지원하고 있다.

### 3. LVM(Logical Volumn Manager)

- ==**여러 개의 하드디스크를 합쳐서 사용하는 기술**==로 한 개의 파일 시스템을 사용
- 작은 용량의 하드디스크 여러 개를 큰 용량의 하나의 하드디스크처럼 사용
- 서버를 운영하면서 대용량의 별도 저장 공간이 필요할 때 활용

### 4. RAID(Redundant Array of Independent Disk)

- ==**여러 개의 물리적 디스크를 하나의 논리적 디스크로 인식하여 작동**==하게 하는 기술
- ==**하드웨어 RAID**== 와 ==**소프트웨어 RAID**== 로 나뉜다.
- _==**데이터를 저장하는 다양한 방법이 존재하며 이 방법들을 레벨**==_이라 한다.

### 5. 파티션 분할

- `fdisk` 는 **파티션 테이블을 관리하는 명령**으로 리눅스의 _==**디스크 파티션을 생성, 수정, 삭제 할 수 있는 유틸리티**==_
- fdisk 명령어 : `a`, `l`, `n`, `t`, `w`, `p`, `q`

  

## Section 5. 부트 매니저

### 1. 부트로드

- 부트스크랩 로더의 준말
- 컴퓨터를 사용자가 사용할 수 있도록 ==**디스크나 플래시에 저장된 OS를 읽어 주기억장치에 적재**==해 주는 프로그램이다.
- 부트로더는 ==**OS가 시동되기 이전에 미리 실행**==되면서 ==**커널이 올바르게 시동되기 위해 필요한 모든 작업을 마무리**==하고 최종적으로 OS를 시동시키기 위한 프로그램이다.
- OS 실행에 필요한 환경을 설정하고 OS 이미지를 메모리에 복사
- 부트매니저라고도 부르며 ==**크기가 512바이트로 하드디스크의 첫번째 섹터인 MBR(Master Boot Record)에 위치**==
- ==**하나의 컴퓨터에 다수의 OS가 설치되어 있는 경우라면 작업 OS를 선택하여 부팅**==할 수 있게 한다.

### 2. 런레벨

- 리눅스 부팅 시 작동하는 서비스들이 있는데, 런레벨에 따라 작동하는 서비스를 조정 가능하다.
- 런레벨은 0 에서 6까지 총 7가지이다.
- 7가지 런레벨 중 리눅스가 가동 시 특정 모드의 레벨을 디폴트로 할 경우 파일 `/etc/inittab` 에 설정
    - `/etc/inittab` 파일 형식은 ‘코드 런레벨:행동:명령어’ 이다.
- 현재 실행되는 런레벨을 확인하는 명령어는 `runlevel` 이다.

  

## Section 6. 사용자 생성 및 계정 관리

### 1. 환경변수 PATH

- `PATH` 는 ==**실행 파일들의 디렉터리 위치를 저장해 놓는 환경 변수**==
- 명령어 `echo $PATH` 는 지정된 PATH 값을 확인할 수 있다.
- 기존의 PATH 에 새로운 경로를 추가하는 방법
    - PATH 나 홈 디렉터리의 `.bash_profile` 에 PATH 추가
    - 기존 경로와 새 경로는 : 으로 구분

### 2. 리눅스 명령어

|**명령어**|**설명**|
|---|---|
|`which`|==**명령어의 경로**==를 확인  <br>위치 혹은 `alias` 를 보여준다|
|`alias`|==**자주사용하는 명령어를 특정 문자로 별칭**==을 정한다|
|`unalias`|`alias` 기능을 해제한다.|

### 3. 리눅스 도움말

|**명령어**|**설명**|
|---|---|
|`man`|==**리눅스에서 사용하는 명령어들의 메뉴얼**==을 제공|
|`info`|리눅스 명령어의 사용 방법, 옵션 등을 나타낸다.|
|`whatis`|==**명령어에 대한 기능을 간략하게 나타**==낸다.  <br>완전히 키워드가 일치해야만 해당 명령어의 기능을 확인|
|`manpath`|man 페이지의 위치 경로를 검색하여 표시|
|`whereis`|찾고자 하는 명령어의 ==**실행 파일 절대 경로와 소스코드, 설정 팡리 및 메뉴얼 페이지를 찾아 출력**==|
|`apropos`|man 페이지 설명에서 지정한 키워드를 포함하고 있는 명령어|

### 4. 사용자 생성 명령어

|**명령어**|**설명**|
|---|---|
|`useradd`|==**계정을 생성하는 명령어**==  <br>계정자의 홈 디렉터리는 `/home/계정명`  <br>생성된 계정자 정보 파일은 `/etc/passwd`, `/etc/shadow`, `/etc/group` 에 저장|
|`passwd`|==**생성된 계정자의 패스워드를 입력 및 변경**==하는 명령어  <br>계정자의 패스워드는 `/etc/shadow` 파일 안에 기록|
|`su`|`su` 는 `switch user` 의 줄임말  <br>현재의 사용자 계정에서 ==**로그아웃하지 않고 다른 사용자 계정으로 로그인하여 해당 사용자의 권한을 획득**==하는 명령어|

### 5. 사용자 관련 파일

|**명령어**|**설명**|
|---|---|
|`/etc/default/useradd`|명령어 useradd 로 사용자 계정을 추가할 때 사용되는 정보를 읽어옴|
|`/etc/passwd`|==**계정자의 정보를 가지고 있는 파일로 리눅스에 로그인할 때 사용**==|
|`/etc/shadow`|==**계정자의 패스워드 정보가 암호화**==되어 있는 파일  <br>패스워드 및 계정의 유효기간 등을 기록|
|`/etc/login.defs`|사용자 계정 설정과 관련된 기본값을 정의|

### 6. 사용자 계정 관리

|**명령어**|**설명**|
|---|---|
|`usermod`|디렉터리 `/home` 에 위치한 사용자들의 정보를 변경|
|`userdel`|==**기존 계정 정보를 삭제**==하는 명령어  <br>옵션 없이 userdel 을 사용하면 `/etc/passwd`, `/etc/shadow`, `/etc/group` 에서 삭제|
|`chage`|패스워드의 만료 정보를 변경하는 리눅스 명령어|

### 7. 그룹관리

|**명령어**|**설명**|
|---|---|
|`/etc/group`|사용자 그룹에 대해 정의되어 있는 파일|
|`/etc/gshadow`|그룹의 암호를 MD5로 하여 저장하며 그룹의 소유주, 구성원 설정이 가능|
|`groupadd`|새로운 그룹을 생성하는 명령어|
|`groupdel`|기존의 그룹을 삭제하는 명령어  <br>그룹 안에 소속되어 있는 계정명이 있을 경우 해당 그룹은 삭제되지 않는다|
|`groupmod`|그룹의 설정을 변경|
