## Section 1. 권한 및 그룹 설정

리눅스 시스템은 **==모든 파일과 디렉터리에 접근권한과 소유권이 부여==**된다.

### 1. 소유권 관련 명령어

- 소유권은 임의의 파일 또는 디렉터리에 대한 사용자와 그룹들의 소유 권한을 의미한다.
- 그룹은 사용자들의 시스템 운영 특성에 따라 묶어 놓은 것.

    |**명령어**|**설명**|
    |---|---|
    |`chown`|파일과 디렉터리의 ==**사용자 소유권과 그룹 소유권을 변경**==|
    |`chgrp`|파일이나 디렉터리의 ==**그룹 소유권을 변경**==|


### 2. 허가권 관련 명령어

- `ls -l` 명령어로 파일 유형과 허가권을 알 수 있다.
- 파일유형은 **일반 파일**, **디렉터리 파일**, **특수 파일**로 나뉜다.
- 파일권한은 **읽기(read)**, **쓰기(write)**, **실행(execute)**이 있다.

    |**명령어**|**설명**|
    |---|---|
    |`chmod`|파일이나 디렉터리의 접근 허가권을 변경하는 명령어|
    |`umask`|새로 생성되는 파일이나 디렉터리의 기본 허가권 값을 지정  <br>파일의 기본 권한은 666, 디렉터리의 기본 권한은 777 이다.|


### 3. 특수 권한

- `SetUID` 와 `SetGID` 가 있는데, 프로세스가 실행되는 동안 ==**해당 프로세스의 root 권한을 임시로 가져오는 기능**==
    - `SetUID` : 사용자가 사용할 때만 **==소유자 권한으로 파일을 실행==**
    - `SetGID` : 사용자가 사용할 때만 **==그룹 권한으로 파일을 실행==**


## Section 2. 파일 시스템의 관리

### 1. 파일 시스템의 개요

- ==**OS가 파일을 시스템의 디스크상에 구성**==하는 방식
- 컴퓨터에서 파일이나 자료를 쉽게 발견 및 접근할 수 있도록 보관 또는 조직하는 체제
- 하드디스크나 CD-ROM 과 같은 물리적 저장소를 관리한다.

### 2. 관련 명령어

|**명령어**|**설명**|
|---|---|
|`mount`/`umount`|마운트는 특정 디바이스를 특정 디렉터리처럼 사용하기 위해 장치와 디렉터리를 연결|
|`eject`|이동식 보조기억장치등과 같은 미디어를 해제하고 장치를 제거하는 명령어|
|`fdisk`|새로운 파티션의 생성, 기존 파티션의 삭제, 파티션의 타입 결정 등 작업을 수행하는 명령어  <br>한 번에 한 디스크에 대해서만 작업을 수행한다.|
|`mkfs`|리눅스 파일 시스템을 생성|
|`mke2fs`|ext2, ext3, ext4 타입의 리눅스 파일 시스템을 생성하는 명령어|
|`fsck`|파일 시스템의 무결성을 점검하고 대화식으로 복구하는 명령어|
|`e2fsck`|ext2, ext3, ext4 타입의 리눅스 파일 시스템을 복구하는 명령어|
|`du`|Disk Usage 의 약자, 디렉터리 별로 디스크 사용량을 확인|
|`df`|시스템에 마운트된 하드 디스크의 남은 용량을 확인할 때 사용하는 명령어  <br>기본적으로 1024 Byte 블록 단위로 출력|


## Section 3. Shell 개념 및 종류

### 1. 개념

- `Shell` 은 ==**명령어 해석기**==이다.
- 강력한 스크립트 언어이다.
- 입출력 방향 재지정과 파이프 기능을 제공

### 2. 종류

- `본 Sheel` 계열과 `C Sheel` 계열로 나뉜다.
- 사용자 프롬프트가 `$` 이면 `본 Shell 계열`, `%` 이면 `C Shell 계열`을 사용하고 있다는 것이다.
- **==대부분의 Shell 은 본 Shell 계열의 기능을 포함하여 확대 발전==**한 형태

### 3. Sheel 확인 및 변경

- `/etc/shell` 에서 사용할 수 있는 `Shell` 들을 확인할 수 있다.
- `echo $SHELL` 은 ==**현재 로그인한 사용자가 사용하고 있는 Shell 을 확인**==할 수 있다.
- `chsh` : 일반 사용자 환경에서 셀 변경 시 사용
- `usermod` : 관리자 환경에서 지정된 계정자의 정보를 변경할 때 사용

## Section 4. Shell 환경 설정

### 1. 환경 변수와 Shell 변수

- `환경 변수`란 ==**전체 Shell 에서 사용 가능한 전역 변수**==이다.
- 환경 변수를 확인하는 명령어는 `env` 이다.
- `Shell 변수`란 ==**현재 로그인 Shell 에서만 사용 가능한 지역 변수**==이다.
- Shell 변수를 확인하는 명령어는 `set` 이다.

### 2. 환경 설정 파일

- Shell 시작 시 자동으로 실행되는 고유의 시작 파일이다. 이 파일은 ==**사용자 운영환경을 설정**==한다.
- Bash Shell 의 시작 파일은 `/etc/profile` , `/etc/bashrc`, `~/.bash_profile`, `~/.bashrc` 이다.
- Shell 파일은 **전역적 파일**과 **지역적 파일**로 나뉜다.
- `/etc/profile.d` 는 몇몇 **==응용 프로그램들이 시작 시 자동 실행할 스크립트 파일 경로를 넣어둔다.==**

### 3. Bash Shell 의 주요기능

- **History 기능**
    - 일정 개수 이상 사용했던 명령어를 `.bash_history` 에 저장해 두고 다시 불러서 사용
    - 대부분의 Shell 은 이전에 입력한 명령어를 반복하거나 다시 변형하여 사용할 수 있다.
- **alias 기능**
    - ==**자주 사용하는 명령어를 특정 문자로 별칭**==을 정해 간편하게 사용할 수 있게 하는 기능

## Section 5. 프로세스 개념 및 유형

### 1. 개념

- `프로세스(Process)`란 ==**CPU 와 메모리를 할당받아 실행 중인 프로그램**==이다.
- 고유의 프로세스 ID를 할당받는다.

### 2. 프로세스의 유형

1. **포어그라운드 프로세스**
    - ==**사용자와 상호작용하는 프로세스**==
    - 명령 입력 후 수행 종료까지 기다려야 하는 프로세스
    - _==화면에서 실행==_되는 것이 보이는 프로세스
    - 예시) 응용프로그램, 명령어 등등
2. **백그라운드 프로세스**
    - ==**사용자와 직접적인 대화를 하지 않고 뒤에서 실행되는 프로세스**==
    - 사용자의 입력에 관계없이 실행되는 프로세스
    - _==실행은 되지만 화면에 나타나지 않고 실행==_되는 프로세스
    - 예시) 시스템 프로그램, 데몬 등등
3. **fork**
    - ==**새로운 프로세스를 위한 메모리를 할당받아 기존 프로세스를 복제**==하는 방식
    - 원본 프로세스를 부모 프로세스, 복제된 프로세스를 자식 프로세스라고 부른다.
4. **exec**
    - ==**호출하는 프로세스가 새로운 프로세스로 변경**==되는 방식이다.
    - _==새로운 프로세스를 위한 메모리를 할당하지 않는다.==_
    - 호출한 프로세스의 메모리에 새로운 프로세스의 코드를 덮어씌워 버린다.
5. **데몬**
    - ==**리눅스 시스템이 부팅 시 자동으로 실행되고 종료될 때 같이 종료되는 백그라운드 프로세스**==이다.
    - _==메모리에 상주하면서 사용자의 특정 요청이 오면 즉시 실행되는 대기 중인 서버 프로세스==_
    - 사용자들은 이 프로세스를 볼 수 있는 권한이 없다.


## Section 6. 프로세스 유틸리티

### 1. 프로세스 관련 명령어

|**명령어**|**설명**|
|---|---|
|`ps`|==**현재 실행중인 프로세스의 상태**==를 보여주는 명령어**==CPU 사용도가 낮은 순서로 출력==**|
|`pstree`|실행 중인 프로세스들을 트리구조로 나타낸다.|
|`jobs`|==**백그라운드로 실행중인 프로세스를 확인**==한다  <br>작업이 중지된 상태, 백그라운드로 진행 중인 상태, 변경되었지만 보고되지 않은 상태 등을 표시|
|`bg / fg`|포어그라운드에서 백그라운드로 전환 : `bg %작업번호` or `bg 작업번호` 백그라운드에서 포어그라운드로 전환 : `fg %작업번호` or `fg 작업번호`|
|`kill`|프로세스를 종료|
|`killall`|같은 데몬의 여러 프로세서를 한 번에 종료|
|`nice`|==**프로세스 사이의 우선순위를 확인하고 우선순위를 변경**==할 수 있는 명령어 → ==**증감**==  <br>조정할 수 있는 NI 값의 범위는 `-20 ~ 19` 이다.  <br>우선순위 0의 값을 가지며 ==값이 적을수록 우선순위가 높다==  <br>==**조정수치가 생략되면 디폴트 값은 10 증가**==이다.  <br>`nice [옵션] 프로세스명`|
|`renice`|이미 실행중인 프로세스의 우선순위를 변경 → ==**특정 값 변경**==  <br>renice 는 nice 와 다르게 특정 값을 증가하거나 감소하지않고 조정수치 값으로 바꾼다.  <br>`renice [옵션] NI값 PID`|
|`top`|==**리눅스 시스템의 운영 상태를 실시간으로 모니터링**==하거나 ==**프로세스 상태를 확인**==할 수 있다.|
|`nohup`|==**프로세스가 중단되지 않고 백그라운드로 작업을 수행**==할 수 있게 한다.  <br>사용자가 로그아웃하거나 작업 중인 터미널 창이 닫혀도 실행 중인 프로세스를 백그라운드 프로세스로 계속 작업할 수 있도록 한다.|
|`tail`|파일의 마지막 행을 기준으로 지정한 행까지 파일 내용의 일부를 출력|

### 2. 스케줄링과 cron

- 주기적으로 반복되는 일은 자동적으로 실행될 수 있도록 설정한다.
- 스케줄링 데몬은 crond 이며 관련 파일은 `/etc/crontab` 이다.

## Section 7. 에디터 종류

### 1. 개요

- 리눅스에서 지원하는 편집기로는 vi, emacs, pico, gedit, xedit 등이 있다.
- 편집기를 통해 파일을 수정한다.

### 2. 종류

1. `pico`
    - Aboli Kasar 가 개발한 ==**Unix 기반의 텍스트 에디터**==이다.
    - **윈도우 메모장과 유사**한 메뉴 선택 방식의 텍스트 편집기이다.
    - 자유 소프트웨어 라이선스가 아니기 때문에 ==**소스 수정이 불가능**==
    - GNU 프로젝트에서는 **pico 의 복제 버전 에디터**인 `nano`를 개발하였다.

        |**메뉴**|**기능**|
        |---|---|
        |`Ctrl + O`|파일 저장|
        |`Ctrl + X`|파일 종료, 종료 시 저장이 안되어 있으면 저장할 것인지 물어봄|
        |`Ctrl + A`|현재 커서 위치에 다른 파일을 불러옴|
        |`Ctrl + E`|현재 행의 맨 앞으로 이동|
        |`Ctrl + V`|현재 행의 맨 끝으로 이동|
        |`Ctrl + Y`|이전 페이지로 이동|
        |`Ctrl + C`|다음 페이지로 이동|
        |`Ctrl + T`|현재 커서의 위치를 표시|
        |`Ctrl + W`|영문자의 철자를 확인|
        |`Ctrl + O`|키를 누르고 문자열을 입력하면 원하는 문자열을 찾음|
        |`Ctrl + K`|현재 라인을 삭제|
        |`Ctrl + U`|마지막으로 삭제된 라인을 복구|

2. `emacs`
    - **리차드 스톨만**이 ==**매크로 기능이 있는 텍스트 교정 및 편집기로 개발**==
    - 최초의 개발자는 리차드 스톨만이며, 이후 제임스 고슬링이 LISP 언어를 기반하여 emacs에 다양한 기능을 개발하여 추가
    - LISP에 기반을 둔 환경 설정 언어를 가지고 있다.

        |**메뉴**|**기능**|
        |---|---|
        |`Ctrl + X Ctrl + S`|파일 저장|
        |`Ctrl + X Ctrl + C`|편집 종료|
        |`마크 설정 후 Ctrl + W`|잘라내기|
        |`Ctrl + K`|커서 뒤에 있는 한 줄이 모두 지워짐|
        |`Ctrl + A`|커서를 줄의 맨 앞으로 이동|
        |`Ctrl + E`|커서를 줄의 맨 뒤로 이동|
        |`Ctrl + N`|커서를 한 줄 아래로 이동|
        |`Ctrl + S 찾을 문자열`|커서의 아랫부분에서 찾을 문자열을 검색|
        |`Ctrl + R 찾을 문자열`|커서의 윗부분에서 찾을 문자열을 검색|
        |`Ctrl + G`|진행되고 있는 명령을 끔|

3. `vi`
    - 1976년 빌 조이가 초기 BSD 릴리즈에 포함될 편집기로 만들었다
    - ==**리눅스 배포판과 유닉스에 기본적으로 포함**==되어 있다.
    - `명령모드`, `입력모드`, `편집모드`로 구성되어 있는 ==**모드형 편집기**==이다.
4. `vim`
    - 브람 무레나르가 만든 편집기이다.
    - ==**vi 편집기와 호환**==되면서 독자적으로 다양한 기능을 추가하였다.
    - ==**편집 시 다양한 색상을 이용하여 가시성을 높일 수 있다.**==
    - 확장된 ==**정규 표현식 문법**==과 ==**강력한 문법 강조**== 기능을 갖는다.
5. `gedit`
    - GNOME 데스크탑 환경으로 개발된 자유 소프트웨어 텍스트 편집기이다.
    - `MS` , `Windows`, `Mac` 에서도 사용가능
    - ==**UTF-8 과 호환하며 텍스트 문서를 편집하는 용도**==에 중점을 두었다.

## Section 8. 소프트웨어 프로그램 설치 부터

### 1. 계열

- `Debian 계열`
    - 배포 업체 : `Debian` , `Ubuntu`, `Xandros`, `Linspire`
    - 패키지 툴 : `dpkg`, `apt-get`, `optitude`
- `Red Hat 계열`
    - 배포 업체 : `Fedora`, `CentOS`, `RHEL`, `openSUSE`, `Mandirva`
    - 패키지 툴 : `rpm`, `yum`
- `rpm`
    - ==**새로운 패키지를 설치하거나 업그레이드, 삭제**== 시 사용
    - `Windows`의 `setup.exe` 와 유사하게 만든 프로그램
- `yum`
    - 네트워크를 통해 기존 RPM 패키지 파일의 업데이트 자동 수행, 새로운 패키지 설치 및 제거를 수행한다.
    - ==**RPM의 의존성 문제를 해결하기 위한 유틸리티**==
    - 네트워크가 정상적으로 연결된 상태여야만 한다.
- `dpkg`
    - ==**데비안의 저레벨 패키지 관리 툴**==
    - 패키지 설치 및 제거 시 RPM과 같은 의존성 문제를 일으킨다.
    - `dpkg -s 패키지` 는 지정된 패키지에 대한 자세한 정보를 나타낸다.
- `apt-get`
    - 데비안 리눅스에서 소프트웨어 설치와 제거를 위한 패키지 관리 유틸리티이다.
- `aptitude`
    - 우분투 패키지 관리 유틸리티로 APT처럼 패키지 관리를 자동화


## Section 9. 소스 파일 설치

### 1. 파일 아카이브와 압축

- 파일 아카이브
    - 아카이브는 ==**다수 개의 파일이나 디렉터리를 하나의 파일로 묶는 것**==이다.
    - 아카이브 파일은 다른 시스템으로 다수 개의 파일을 한 번에 전송하거나 파일 백업용으로 사용한다.
- 파일압축과 해제
    - 대표적인 파일 압축 명은 `compress`, `gzip`, `bzip2`, `xz` 가 있다.
    - ==**일반적으로 많이 사용되는 압축 명령어**==는 `gzip` 과 `bzip2` 이다.
    - 압축률이 가장 낮은 것은 `compress` , 가장 높은 것은 `xz` 이다.

### 2. 소스 코드 설치

- 소스 코드를 압축 해제 후 컴파일 순서에 따라 프로그램을 설치한다.
- 컴파일 순서는 설치 파일의 환경설정, 컴파일, 파일 설치이다.
    - **1단계 환경설정** : `./configure` 프로그램 설치 과정에서 ==**필요로 하는 환경파일 makefile 생성**==
    - **2단계 컴파일** : `make makefile` 을 기반으로 소스 파일을 컴파일
    - **3단계 파일 설치** : `make install` 컴파일 된 실행파일을 지정된 속성으로 지정된 디렉터리에 설치
