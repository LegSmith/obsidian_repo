## 공통 인터페이스 적용

- JpaRepository<> 를 상속 받으면 된다.
- 제네릭에는 반환타입과 PK 타입 2개가 들어간다.

    ```java
    public interface MemberRepository extends JpaRepository<Member,Long> {}
    ```


## 공통 인터페이스 분석

- JpaRepository 인터페이스는 공통 CRUD 를 제공해준다.
- 정말 웬만한 공통 CRUD 는 다 구현이 되어있기 때문에 개발자는 편하게 가져다 쓰면 된다.

## 주요 메서드

> T : 엔티티  
> ID : 엔티티의 식별자(PK) 타입  
> S : 엔티티와 그 자식 타입

- `save(S)` : 새로운 엔티티는 저장하고 ==**이미 있는 엔티티는 병합(merge) 한다.**==
- `delete(T)` : 엔티티 하나를 삭제( 내부에서는 `EntityManager.remove()` 호출 )
- `findById(ID)` : 엔티티 하나를 조회( `EntityManager.find()` 호출 )
- `getOnd(ID)` : 엔티티를 프록시로 조회한다. (`EntityManager.getReference()` 호출 )
- ==findAll(…)== : 모든 엔티티를 조회한다. ==**정렬(Sort)이나 페이징(Pageable) 조건을 파라미터로 제공**==한다.
