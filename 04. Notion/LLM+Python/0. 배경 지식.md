## 1. Retrieval Augmented Generation(RAG)란 ?

- ==**LLM 어플리케이션 개발에서 가장 중요한 개념 !**==
- `LLM(Large Language Model)` 이 등장하기전 `NLP(Natural Language Processing)` 에서부터 사용됨.
- 직역하면 **==검색-증강 생성==**이다.
- 단어 하나하나 쪼개서 해석하면 다음과 같다.

### Retrieval

- 데이터를 가져오는 것, 구체적으로는 **=="컴퓨터 시스템에 저장된 자료를 취득하는 것" 이라는 뜻==**도 있음.
- ==**언어모델이 가지고있지 않은 정보를 가져오는 것.**==
    - 언어모델이 아웃풋을 만드는데 필요한 정보를 제공하는 것.
    - 언어모델은 답변 생성에 능숙하지만 모든 정보를 가지고 있지 않다.
    - 보안이 걸려있는 사내자료 혹은 GPT4에 경우 23년 10월까지의 정보만 가지고 있기 때문에 24년도 정보는 가지고 있지 않다.

### Augmented

- AR/VR(증강현실/가상현실)에서 자주사용하는 단어이다.
- **==마치 사실인 것 처럼을 뜻==**한다.
- Retrieval 된 데이터를 LLM에게 주면서, "마치 이 정보를 아는 것 처럼"

### Generation

- 생성
- ==**사용자가 Retrieval 된 데이터를 LLM에게 주면, 모델은 이 정보를 아는 것 처럼 답변을 생성**==한다.


## 2. Vector Database와 Embedding Model 성능 비교

- 사용자가 질문을 할 때 원하는 정보는 사용자의 질문과 관련있는 데이터이다.
- 그렇다면 ==**답변과 질문이 관련이 있는지 판단을 해야 하는데 이 때 Vector 를 사용하여 관련성 파악**==을 한다.

### Vector를 생성하는 방법

- ==**Embedding 모델을 활용해서 vector를 생성**==함.
- 문장에서 비슷한 단어가 자주 붙어있는 것을 학습
    - 왕은 왕자의 아버지다
    - 여왕은 왕자의 어머니다
- 위 예시를 보면 **=="왕자의" 라는 단어 앞에 등장하는 "왕"과 "여왕"은 유사할 가능성이 높다.==**

    > [!info] Embedding projector - visualization of high-dimensional data  
    > Visualize high dimensional data.  
    > [https://projector.tensorflow.org/](https://projector.tensorflow.org/)  

- 위와 같은 Embedding 모델을 활용해서 생성된 vector를 저장.
- **==단순히 vector만 저장하면 안되고 metadata도 같이 저장==**해야 한다.
    - 문서의 이름, 페이지 번호 등등을 같이 저장한다. → ==**LLM이 생성하는 답변의 퀄리티가 상승**==함.

### Vector를 대상으로 유사도 검색

- 사용자의 질문과 가장 비슷한 문서를 가져오는 것. → `Retrieval`
- 문서 전체를 활용하면 속도도 느리고, 토큰수 초과로 답변 생성이 안될 수 있다.
    - 이 때, 문서를 `chunking` 하는게 중요하다 !

        chunking 이란 ?

        - **==긴 문서나 대량의 정보를 작은 조각(chunks)으로 나누는 과정==**

- 그리고 가져온 문서를 prompt 를 통해 LLM에 제공한다. → `Augmented`
- LLM은 prompt를 활용해서 답변을 생성한다. → `Generation`

### Embedding 모델

- 영어를 이용한다면 OpenAI 의 Embedding 모델을 사용하면 되지만, 한글을 이용한다면 Upstage Embedding 모델을 사용하는게 좋다.
- 30불을 무료 크레딧으로 제공하기 때문에 테스트하기에는 좋다 !
