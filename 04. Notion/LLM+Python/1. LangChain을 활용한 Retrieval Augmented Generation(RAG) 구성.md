## 1. 환경 설정과 LangChain의 ChatOpenAI를 활용한 검증

### 환경설정 파일(.env)

- 주의할 점은 키값을 `OPENAI_API_KEY` 로 지정해야 한다. 추후 번거로운 세팅이 사라지기 때문

    ```bash
    OPENAI_API_KEY=사용자 OpenAPI 키
    ```


### 핵심코드(.ipynb)

- `LangChain` 을 이용하여 `OpenAPI` 를 사용 할려면 환경설정 파일을 불러왓어 OpenAPI 키를 등록하고, LangChain 모델을 불러와야 한다.

    ```python
    from dotenv import load_dotenv # 환경변수 불러오기
    from langchain_openai import ChatOpenAI # langchain 불러오기
    ```

- 설정한 환경변수를 불러오기 위해서는 `load_dotenv()` 함수를 호출하고, LangChain 모델을 쓸려면 `ChatOpenAI()` 를 호출하여 변수에 저장한다.

    ```python
    load_dotenv()

    llm = ChatOpenAI()
    ```

- 그리고 질문을 할 때는 `invoke()` 함수에 파라미터로 질문을 던져주고 `content` 파라미터를 이용하여 답변만 추출한다.

    ```python
    ai_message = llm.invoke("인프런에 어떤 강의가 있나요?")
    ai_message.content
    ```

## 2. LangChain과 Chroma를 활용한 RAG 구성

### 전체 흐름

1. 문서의 내용을 읽는다

2. 문서를 쪼갠다.

3. 임베딩 -> 벡터 DB에 저장

4. 질문이 있을 때, 벡터 DB에 유사도 검색

5. 유사도 검색으로 가져온 문서를 LLM에 질문과 같이 전달

- [https://law.go.kr/%EB%B2%95%EB%A0%B9/%EC%86%8C%EB%93%9D%EC%84%B8%EB%B2%95](https://law.go.kr/%EB%B2%95%EB%A0%B9/%EC%86%8C%EB%93%9D%EC%84%B8%EB%B2%95) → 우선 문서를 위해 소득세법을 가져온다.
- 우상단 저장 버튼 클릭 후 워드(DOC)파일 선택 후 저장을 한다.

    ![[image 1 5.png|image 1 5.png]]

    - 우선 **==우리말로 된 PDF는 Python 에서 읽을 때 단어 1개가 줄바꿈이 일어나게 되면 한 단어로 인식을 못한다 !==**
    - ==**워드를 사용하는 이유는 Load 할 때 문단 단위로 읽어 들이기 때문에 단어1개가 줄바꿈이 일어나더라도 단어1개로 인식**==한다 !
- 다운을 받았으면 파일 이름을 `tax.docs` 로 바꾼 후 Python 프로젝트로 가져온다 !

- 그리고 원드 문서를 읽을 때는 `Document Loader` 를 사용 → [https://python.langchain.com/v0.1/docs/integrations/document_loaders/microsoft_word/](https://python.langchain.com/v0.1/docs/integrations/document_loaders/microsoft_word/)

    ![[image 3 3.png|image 3 3.png]]

- 읽은 문서를 쪼갤 때는 `Text Splitters` 를 이용 → [https://python.langchain.com/v0.1/docs/modules/data_connection/document_transformers/recursive_text_splitter/](https://python.langchain.com/v0.1/docs/modules/data_connection/document_transformers/recursive_text_splitter/)

    ```python
    from langchain_community.document_loaders import Docx2txtLoader
    from langchain_text_splitters import RecursiveCharacterTextSplitter

    text_splitter = RecursiveCharacterTextSplitter(
        chunk_size=1500,    # 문서를 쪼갤 때 하나의 chunk 가 가지는 토큰 수
        chunk_overlap=200,  # 텍스트를 분할할 때, 각 chunk(조각) 사이에 중복되는 부분의 토큰 수
    )

    loader = Docx2txtLoader('./tax.docx')
    document_list = loader.load_and_split(text_splitter=text_splitter) # 문서가 쪼개지기 때문에 List 타입
    document = loader.load() \#단순히 문서를 읽을 때
    ```

- 임베딩을 하기 위해서는 `OpenAIEmbeddings` 를 사용한다. → [https://platform.openai.com/docs/guides/embeddings/](https://platform.openai.com/docs/guides/embeddings/)

    ```python
    from dotenv import load_dotenv                # OpenAI API Key 사용을 위해 환경변수 등록
    from langchain_openai import OpenAIEmbeddings # OpenAI 의 Embedding을 사용

    load_dotenv()

    embeddings = OpenAIEmbeddings(model='text-embedding-3-large') # 기본모델은 002인 예전모델이다. 신규 모델을 사용하기 위해 추가
    ```

- 임베딩한 데이터를 벡터 DB에 넣는데. 이 때 사용하는 DB 모델은 `Chroma` 이다. → [https://python.langchain.com/v0.1/docs/integrations/vectorstores/chroma/](https://python.langchain.com/v0.1/docs/integrations/vectorstores/chroma/)
- 파라미터에는 위에서 문서를 쪼갠 데이터와, 임베딩 모델을 넣어준다.
- 이 때, 단순히 데이터와 임베딩 모델만 넣어주면 DB가 저장이 되지 않는다. 그래서 `collection_name` 과 `persist_directory` 를 통해 폴더와 db테이블을 저장해준다.

    ```python
    from langchain_chroma import Chroma

    database = Chroma.from_documents(documents=document_list, embedding=embeddings, collection_name='chroma_tax',persist_directory="./chroma") 
    ```

- 유사도 검사를 할려면 `similarity_search()`를 호출하면 된다. 이 때 파라미터중 k 값은 결과 갯수이다. 디폴트 값은 4이다.

    ```python
    query = '연봉 3000만원인 직장인의 소득세는 얼마인가요?'
    retrieved_docs = database.similarity_search(query, 1); # similarity_search() 함수는 유사도 검색을 위한 함수이다.
    ```

- 이제 LLM 에게 질의를 하면 된다. 이 때 `ChatOpenAI` 를 사용한다.
- 질문을 하기 위해 프롬프트를 생성해야 한다.

    ```python
    from langchain_openai import ChatOpenAI

    llm = ChatOpenAI(model='gpt-4o') # 디폴트 모델은 gpt-4o 가 아니다.

    # 기존에는 질문만 날렸지만 이제는 질문+문서 를 질의한다. 즉, 프롬프트 작성
    prompt = f"""[Identity]
    - 당신은 현존하는 세계 최고의 한국 소득세 전문가 입니다
    - [Context]를 참고해서 사용자의 질문에 답변해주세요

    [Context]
    {retrieved_docs}

    Question: {query}
    """

    ai_message = llm.invoke(prompt)

    ai_message.content
    ```

- Retrieval 를 효과적으로 쓰기위해 `RetrievalQA Chain` 이라는 기술을 사용한다.
- 기존의 방법은 문서를 가져와서 쪼개고 임베딩한 결과를 크로마에 담아서 `LLM` 에 질문했다면 `RetrievalQA Chain` 은 아래와 같이 코딩해서 더 간편해졌다.

    ```python
    # QA Chain 만들기
    from langchain.chains import RetrievalQA

    qa_chain = RetrievalQA.from_chain_type(
        llm,
        retriever=database.as_retriever(),
        chain_type_kwargs={"prompt":prompt}
    )

    ai_message = qa_chain({"query" : query})

    ai_message
    ```

## 3. LangChain 없이 구성하는 RAG의 불편함

### 3.1 문서 읽기

- 기존 `LangChain` 을 안쓰기 때문에 `Document Loader` 는 사용하지 못하고 `Python Docx` 를 사용한다. → [https://python-docx.readthedocs.io/en/latest/user/install.html](https://python-docx.readthedocs.io/en/latest/user/install.html)
    - `$pip install python-docx` 명령어로 패키지를 추가한다.
- 문서를 문단별로 읽어서 특정 변수에 저장하는코드이다.

    ```python
    from docx import Document

    document = Document('./tax.docx');
    print(f'document == {document}')
    full_text = ''

    for index, paragraph in enumerate(document.paragraphs):
        print(f'paragraph == {paragraph.text}')
        full_text += f'{paragraph.text}\n'
    ```


### 3.2 문서 쪼개기

- 문서를 읽었으면 문서를 쪼개야한다. `Text Spliter` 대신에 `tiktoken` 을 사용한다. → [https://github.com/openai/tiktoken](https://github.com/openai/tiktoken)
- `tiktoken` 을 사용하여 텍스트 문서를 토큰화시켜 인코딩을 하면 gpt-4o 모델이 지원하는 최대 토큰 수를 넘어버린다. 그렇기 떄문에 chunk 를 잘 조절하여 토큰수를 줄여야한다.

    ![[image 5 3.png|image 5 3.png]]

    ```python
    import tiktoken

    def split_text(input_text, chunk_size):
        encoder = tiktoken.encoding_for_model("gpt-4o")
        total_encoding = encoder.encode(input_text) # 인코딩 : 텍스트 -> 숫자 리스트
        total_token_count = len(total_encoding);
        text_list = []
        for i in range(0, total_token_count, chunk_size):
            chunk = total_encoding[i: i+chunk_size]
            decoded = encoder.decode(chunk)   # 디코딩 : 숫자 리스트 -> 텍스트
            text_list.append(decoded)

        return text_list
    ```

- 위에서 읽은 문서를 `split_text()` 를 호출하여 쪼개서 `chunk_list` 라는 변수에 저장한다.

    ```python
    chunk_list = split_text(full_text, 1500)

    chunk_list
    ```


### 3.3 문서 임베딩

- 이제 문서 임베딩을 해야한다.
- 다행이 `Chroma` 에서 제공하는 임베딩 기술이 있다.
- 우선 `Chroma` 패키지를 설치받고 `LangChain` 에 `Chroma` 가 아닌 그냥 `Chroma` 를 불러온다.

    ```python
    import chromadb

    chroma_client = chromadb.Client()
    ```

- 그 다음 컬렉션을 만든다.
    - 컬렉션이란? RDB(관계형 DB)에서 테이블과 같은 것이다.
- `create_collection()` 함수를 호출하여 컬렉션을 만든다.

    ```python
    collection_name = 'tax_collection'
    tax_collection = chroma_client.create_collection(collection_name)
    ```

- 임베딩 기술은 `Chroma` 에서 제공하는 `embedding_functions` 을 사용한다.
- API 키와 사용할 임베딩 모델명을 입력하여 생성한다

    ```python
    import os
    from dotenv import load_dotenv
    from chromadb.utils.embedding_functions import OpenAIEmbeddingFunction

    load_dotenv()

    openai_api_key = os.getenv('OPENAI_API_KEY')
    openai_embedding = OpenAIEmbeddingFunction(api_key = openai_api_key,model_name='text-embedding-3-large')
    ```

- 위에서 만든 임베딩 모델을 DB에 넣기 위해 컬렉션명과 모델을 넣어준다.

    ```python
    tax_collection = chroma_client.get_or_create_collection(collection_name, embedding_function=openai_embedding)
    ```

- 이제 DB에 데이터를 넣어줘야 하는데. 이때 데이터와 인덱스가 같이 들어가기 떄문에 인덱스를 생성하는 for 문이 필요하다.

    ```python
    id_list = []
    for index in range(len(chunk_list)):
        id_list.append(f'{index}')

    tax_collection.add(documents=chunk_list, ids=id_list)
    ```


### 3.4 유사도 검색

- 이제 유사도 검색을 위해 질의를 만들고 유사도 검색을 한다.

    query = '연봉 5천만원인 직장인의 소득세는 얼마인가요?'

- 질의를 위해서 OpenAI 패지키를 설치한다 → `pip install openai`
- 공식문서 코드를 가져와서 문서와 쿼리만 수정한다.

## 4. LangChain을 활용한 Vector Database 변경(Chroma → Pinecone)

- `Chroma` 대신에 `Pinecone` 이라는 `Vector DB`를 사용해본다. → [https://python.langchain.com/docs/integrations/vectorstores/pinecone/](https://python.langchain.com/docs/integrations/vectorstores/pinecone/)
    - `Chroma` 는 In-Memory DB 인 반면 `Pinecone` 은 Cloud를 활용하는 DB이다.
- 필요 패키지를 다운받는다.
    - `%pip install -qU langchain-pinecone pinecone-notebooks`
- 인덱스를 직접 Python 코드로 설정할 수 있지만 보기 힘들기 때문에 Pinecone Console 을 이용한다. → [https://app.pinecone.io/organizations](https://app.pinecone.io/organizations)
- 이 때 pinecone-notebooks 패키지는 필요 없기 때문에 지워준다.
    - `%pip uninstall -y pinecone-notebooks`
- 그리고 공식문서에 따라 아래와 같이 코딩한다.

    ```python
    import os

    pinecone_api_key = os.environ.get('PINECONE_API_KEY')
    pinecone_api_key

    import time
    from pinecone import Pinecone

    pc = Pinecone(api_key=pinecone_api_key)
    ```

- 그리고 Index를 만드는 것은 코드가 아닌 `Pinecone Console` 에서 실행한다.
- 좌측메뉴 중 Database 를 클릭하고 Indexes 를 클릭한 후 Create Index 를 누른다.

    ![[image 6 3.png|image 6 3.png]]

- 이름을 정하고 Configuration 항목에서 각 옵션을 선택한다.
    - Dimensions 은 Embedding Model의 Deimensions 를 말한다. 벡터의 길이를 정하는 것이다.
    - Metric 은 유사도 검색의 방법을 정하는 옵션이다.
        1. `euclidean` : 점과 점 사이의 거리를 계산하여 거리가 가까울 수록 유사도가 높다고 측정
        2. `dotproduct` : 각 벡터의 같은 인덱스 값을 곱하여 다 더한 후 값이 클수록 유사도가 높다고 측정.
        3. `Cosine` : 두 벡터 사이의 각도를 정한다. 각도가 작을 수로 유사도가 높다고 측정.
- 체크된 옵션들만 입력하고 Create Index 를 클릭한다.

    ![[image 7 3.png|image 7 3.png]]

- 기존 코드를 다음과 같이 수정한다.

    - `.env` 파일에 `PINECONE_API_KEY` 를 넣어주고 기존의 쪼갠 문서와 임베딩 모델과 인덱스 네임을 넣어준다.

## 5. Retrieval 효율 개선을 위한 데이터 전처리
