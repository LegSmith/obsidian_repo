서비스 레이어를 테스트하기 위해 Repository 가 필요하다. 실제 Repository 를 메모리에 로딩하고 테스트하면 되지만 만약 ==_**테스트의 독립성을 위해서는 Mockito 환경에서 테스트**_==하는게 좋다.

  

## 가짜 Mock 환경 테스트 준비 & 책 등록하기 테스트

---

- Mock 환경에서 테스트를 하기 위해 `@ExtendWith(MockitoExtension.class)` 선언하여 Mock 환경구성
- 그리고 `@Mock` 어노테이션으로 가짜 환경에 `BookRepository` 와 `MailSender` 를 로딩한다.
- `@InjectMocks` 어노테이션은 _==**실제 메모리에 BookService 를 로딩하고 가짜 환경에 있는 BookRepository 와 MailSender 를 필드로 가져와 주입**==_시켜준다.
- 그리고 그냥 테스트를 하면 가짜 객체이기 때문에 NullPointerException 예외가 발생하기 때문에 when() 메서드로 가설을 세워야 한다.
    
    - when() : Mock 객체의 메소드 반환 값을 지정해준다.
    - `when(실행할 로직).thenReturn(기대하는 결과);`
    
    ```java
    @ExtendWith(MockitoExtension.class)
    public class BookServiceTest {
    
        // BookService 객체가 heap 메모리에 뜨고 BookService 가 의존하는 BookRepository , MailSender를
        // BookService 에 주입 시켜준다.
        @InjectMocks
        private BookService bookService;
    
        @Mock // 가짜 환경에 BookRepository , MailSender 를 로딩함.
        private BookRepository bookRepository;
        @Mock
        private MailSender mailSender;
    
        @Test
        public void 책등록하기_test() throws Exception {
            // given
            BookSaveReqDto dto = new BookSaveReqDto();
            dto.setTitle("Football");
            dto.setAuthor("Cristiano");
    
            // stub(가설)
            when(bookRepository.save(any())).thenReturn(dto.toEntity());
            when(mailSender.send()).thenReturn(true);
    
            // when
            BookRespDto bookRespDto = bookService.책등록하기(dto);
    
            // then
            assertThat(dto.getTitle()).isEqualTo(bookRespDto.getTitle());
            assertThat(dto.getAuthor()).isEqualTo(bookRespDto.getAuthor());
    
        }
    }
    ```
    
- 즉, Service 레이어 테스트를 할 때는 Repository 가 필요하기 때문에 ==**Mock 환경에 Repository 를 띄어두고 Service 클래스를 @InjectMock 어노테이션으로 지정하여 가짜환경에 있는 객체들을 주입받아서 테스트**== 하면 된다.

## 책 목록 보기 테스트

---

- 우선 임의의 Book 객체를 2개 가지고 있는 List<Book> 을 선언해서 정의해주고
- `when()` 메서드로 가설을 세운다 → _**BookRepository 의 findAll() 을 하면 2건의 Book 이 저장되있는 List<Book> 이 리턴된다.**_

    @ExtendWith(MockitoExtension.class)
    public class BookServiceTest {

        private BookService bookService;

        private BookRepository bookRepository;
        @Mock
        private MailSender mailSender;
    
    		@Test
        public void 책목록보기_test() throws Exception {

            // stub
            List<Book> books = Arrays.asList(

            when(bookRepository.findAll()).thenReturn(books);

## 메서드 참조 코드 리팩토링

---

- BookService.책목록보기() 메서드를 수정했었는데 그 이유를 정리했다.
- `map(new BookRespDto()::toDto)` → 이 구문이 메서드 참조인데 아래 로직에서는 stream() 에 2건의 Book 이 담기는데 map() 을 하면 2건 다 접근해야 하지만 `map(new BookRespDto()::toDto)` 이 구문은 1번밖에 실행을 안한다.
- 정확히는 _**t**_==_**oDto() 메서드는 stram()에 담긴 객체수만큼 2번을 실행**_==하지만 `new BookRespDto()`를 2번 실행하는 것이 아니다 !!!
    
    - _==**new BookRespDto() 객체 1개는 heap 메모리에 로딩이 대고 toDto()가 2번 실행이 된다 !!**==_
    
    ```java
    bookRepository.findAll().stream()
            .map(new BookRespDto()::toDto)
            .collect(Collectors.toList());
    ```
    
    ![[Untitled 2 6.png|Untitled 2 6.png]]
    
- 우선 BookRespDto 코드 수정을 한다.
    
    - 앞전에 Entity 를받아 DTO로 반환하는 코드를 지우고 생성자를 만들고 Builder 패턴으로 지정했다.
    
    ```java
    @Data
    public class BookRespDto {
        private Long id;
        private String title;
        private String author;
    
        @Builder
        public BookRespDto(Long id, String title, String author) {
            this.id = id;
            this.title = title;
            this.author = author;
        }
    }
    ```
    
- 그리고 ==**Book 엔티티에 dto로 변환하는 메서드를 넣는다.**==
    
    ```java
    @NoArgsConstructor
    @Getter
    @Entity
    public class Book {
    		
    		//생략...
    
        public BookRespDto toDto() {
            return BookRespDto.builder().id(id).title(title).author(author).build();
        }
    }
    ```
    
- 최종적으로 stream() 코드는 이렇게 수정된다.
    
    ```java
    public List<BookRespDto> 책목록보기() {
        return bookRepository.findAll().stream()
                .map(Book::toDto) // 코드가 더 간결해졌다.
                .collect(Collectors.toList());
    }
    ```
    

## 책 한건 보기 테스트

---

- `when()` 메서드로 가설을 세우기 위해 ==**1L 을 가진 id 변수와 1L 이 저장된 Book 객체를 만든다.**==
- `findById()` 를 호출하면 `Optional<Book>` 이 리턴되기 때문에 ==**book 에 Optional 을 감싸서 thenReturn()에 넣어준다.**==

```java
@ExtendWith(MockitoExtension.class)
public class BookServiceTest {

    // BookService 객체가 heap 메모리에 뜨고 BookService 가 의존하는 BookRepository , MailSender를
    // BookService 에 주입 시켜준다.
    @InjectMocks
    private BookService bookService;

    @Mock // 가짜 환경에 BookRepository , MailSender 를 로딩함.
    private BookRepository bookRepository;
    @Mock
    private MailSender mailSender;

		@Test
    public void 책한건보기_test() throws Exception {
        // given
        Long id = 1L;
        Book book = new Book(1L, "Junit", "MetaCoding");

        // stub
        when(bookRepository.findById(id)).thenReturn(Optional.of(book));

        // when
        BookRespDto dto = bookService.책한건보기(id);

        // then
        assertThat(dto.getTitle()).isEqualTo(book.getTitle());
        assertThat(dto.getAuthor()).isEqualTo(book.getAuthor());
    }
}
```

## 책 수정, 삭제 하기 테스트

---

삭제하기는 테스트 할 필요가 없다. _==**Service 코드에서 삭제하기() 는 DB에서 지우는 로직밖에 없기 때문에 !! Service 레이어에서 테스트할 필요가 없다.**==_

- 기존의 수정하기 본 코드는 리턴값이 void 라 테스트로 검증하기 힘들어서 `BookRespDto` 로 변경하였다.
    
    ```java
    // 5. 책 수정하기
    @Transactional(rollbackFor = RuntimeException.class)
    public BookRespDto 책수정하기(Long id, BookSaveReqDto dto) {
        Optional<Book> bookOP = bookRepository.findById(id);
        if (bookOP.isPresent()) {
            Book bookPS = bookOP.get();
            bookPS.update(dto.getTitle(), dto.getAuthor());
            return bookPS.toDto(); // Test 를 위해 return 값을 지정해줬다.(void ->BookRespDto)
        } else {
            throw new RuntimeException("해당 아이디를 찾을 수 없습니다.");
        }
    }
    ```
    
- 수정하기 테스트를 할 때는 id 값과 변경할 데이터인 DTO를 먼저 given 에 정의한다.
- 수정하기 로직에서는 `findById()`를 호출해서 데이터를 변경하기 때문에 `when()` 메서드로 가설을 정의
- 그리고 검증하면 끝 !
