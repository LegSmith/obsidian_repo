  

## 책 등록

---

- Entity를 반환하거나 직접받는 행위를 하지않고 DTO를 만들어서 파라미터로 받고 응답해주는게 안전하다.
    
    - 파라미터로 받는 DTO는 Entity로 변환하는 메서드를 하나 만들어주고, 반환해주는 DTO는 Entity를 받아 DTO로 변환하는 메서드를 만들어준다.( ==**요청<DTO> → Entity로 변환 → 응답<Entity→DTO>**== )
    
    ```java
    @Setter
    public class BookSaveReqDto {
        private String title;
        private String author;
    
        public Book toEntity() {
            return Book.builder()
                    .title(title)
                    .author(author)
                    .build();
        }
    }
    ---------------------------------------------ㅍ
    @Data
    public class BookRespDto {
        private Long id;
        private String title;
        private String author;
    
        public BookRespDto toDto(Book bookPS) {
            this.id = bookPS.getId();
            this.title = bookPS.getTitle();
            this.author = bookPS.getAuthor();
    
            return this;
        }
    }
    ```
    
- 클래스 단위에서 `@Transactional(readOnly = true)` 어노테이션은 ==**Select 하는 메서드에 성능을 향상시키기 위해 설정(기본값은 false)**==
- 직접적으로 Data변화가 있는 메서드에는 따로 `@Transactional` 을 걸어주면 된다.
    
    - 파라미터로 DTO를 받고 DTO를 Entity로 변환하여 save하고, save하고 반환된 엔티티를 다시 DTO로 변환하여 응답해주면 된다.
    
    ```jsx
    @Service
    @Transactional(readOnly = true)
    @RequiredArgsConstructor
    public class BookService {
    
        private final BookRepository bookRepository;
    
        // 1. 책 등록
        @Transactional(rollbackFor = RuntimeException.class)
        public BookRespDto 책등록하기(BookSaveReqDto dto) {
            Book bookPS = bookRepository.save(dto.toEntity());
    
            return new BookRespDto().toDto(bookPS);
        }
    }
    ```
    

## 책 목록보기

---

- 책 목록보기는 간단하다 스프링 데이터 JPA 가 지원하는 findAll() 을 이용하여 List타입의 Entity를 받을 수 있다.
- 하지만 ==**Entity를 직접반환하는 것 보다 DTO로 변환**==하는게 좋다. 이 때 Java 8의 stream() 을 이용하면 깔끔하게 변경할 수 있다.
    
    ```java
    @Service
    @Transactional(readOnly = true)
    @RequiredArgsConstructor
    public class BookService {
    
        private final BookRepository bookRepository;
    
    		// 2. 책 목록보기
        public List<BookRespDto> 책목록보기(){
            return bookRepository.findAll().stream()
                        .map(new BookRespDto()::toDto)
                        .collect(Collectors.toList());
        }
    }
    ```
    

## 책 한건보기

---

- findById() 는 반환값으로 Optional로 감싸져서 나온다.
- isPresent()로 null 인지 아닌지 체크후에 DTO로 바꿔서 반환해주면 된다.
    
    ```java
    @Service
    @Transactional(readOnly = true)
    @RequiredArgsConstructor
    public class BookService {
    
        private final BookRepository bookRepository;
    
    		// 3. 책 한건보기
        public BookRespDto 책한건보기(Long id) {
            Optional<Book> bookOP = bookRepository.findById(id);
            if (bookOP.isPresent()) {
                return new BookRespDto().toDto(bookOP.get());
            } else {
                throw new RuntimeException("해당 아이디를 찾을 수 없습니다.");
            }
        }
    }
    ```

## 책 수정하기

---

- JPA 에 더티체킹을 이용하여 간편해진다.
- 우선 Book 엔티티의 `update()` 메서드를 만들자 → ==_**Entity에서 Setter를 이용하는건 비추**_==
    
    ```java
    @NoArgsConstructor
    @Getter
    @Entity
    public class Book {
    
    		// 생략....
    
        public void update(String title, String author) {
            this.title = title;
            this.author = author;
        }
    }
    ```
    
- id값을 이용해 수정할 엔티티를 찾고 update()메서드를 호출하여 수정할 데이터를 넘겨준다. → 이렇게 되면 영속성 컨텍스트가 더티체킹을 하여 데이터가 update된다.
    
    ```java
    @Service
    @Transactional(readOnly = true)
    @RequiredArgsConstructor
    public class BookService {
    
        private final BookRepository bookRepository;
    
    		// 5. 책 수정하기
        @Transactional(rollbackFor = RuntimeException.class)
        public void 책수정하기(Long id, BookSaveReqDto dto) {
            Optional<Book> bookOP = bookRepository.findById(id);
            if (bookOP.isPresent()) {
                Book bookPS = bookOP.get();
                bookPS.update(dto.getTitle(), dto.getAuthor());
            } else {
                throw new RuntimeException("해당 아이디를 찾을 수 없습니다.");
            }
        }
    }
    ```
