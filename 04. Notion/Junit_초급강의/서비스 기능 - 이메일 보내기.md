시나리오  
- '홍길동'이 책 등록을 한다면 책이 등록되었다는 메일 전송

- 이메일 기능이 없다고 해서 책 등록 테스트를 못하는 것이 아니다.
    1. 책 등록
    2. true → 메일전송 → 실패(롤백)
    3. 끝
- 이럴 때는 `Mail` 클래스가 구현이 안되어있다고 해서 테스트를 안할 게 아니라 `MailSender` 라는 인터페이스를 만들고 `가짜메일 구현체`와 `MailAdapter` 를 만든다. 나중에 Mail 클래스가 만들어지면 메일 어댑터를 연결해주면 된다.
- 그리고 코드는 `가짜 메일 구현체`나, `MailAdapter`를 의존하지 않고 `MailSencder`로 의존하게 하고 추후에 `Mail` 클래스가 구현이 되면 교체만 해주면 된다.

    ![[Untitled 17.png|Untitled 17.png]]

- 우선 실제 서비스 코드에는 MailSender 인터페이스를 의존하게 하고 IOC 컨테이너에는 가짜 메일 구현체를 띄어놓아서 서비스코드에 연결하게 한다.

    ```java
    public interface MailSender {
        boolean send();
    }
    ```

- `MailSenderStub` 클래스가 가짜 Mail 클래스이다. `@Component` 어노테이션으로 가짜 Mail 클래스를 IOC 컨테이너에 띄운다.

    ```java
    // 가짜 
    @Component
    public class MailSenderStub implements MailSender {

        @Override
        public boolean send() {
            return true;
        }

    }
    ```

- 그리고 서비스코드에는 MailSender 인터페이스를 DI 하게 되면 IOC 컨테이너에 띄워져있는 MailSenderStub 이 연결된다 !

    ```java
    public class BookService {

        private final BookRepository bookRepository;
        private final MailSender mailSender;

        // 1. 책 등록
        @Transactional(rollbackFor = RuntimeException.class)
        public BookRespDto 책등록하기(BookSaveReqDto dto) {
            Book bookPS = bookRepository.save(dto.toEntity());

            if (bookPS != null) {
                if (!mailSender.send()) {
                    throw new RuntimeException("메일이 전송되지 않았습니다");
                }
            }
            return new BookRespDto().toDto(bookPS);
        }
    }
    ```

- 그리고 추후에 Mail 클래스가 완성이 된다면 아래와 같이 클래스가 생성될 것이고

    ```java
    // 아직 구현되지 않음
     public class Mail {
    	 public boolean sendMail() {
    		  // 기능
    		 return true;
    	 }
     }
    ```

- 아래와 같이 Mail 클래스를 필드로 받아 생성자에 연결해주고 `@Component` 어노테이션을 붙혀준다(**==MailSenderStub 클래스에는 어노테이션을 지워줘야 한다, 같은 타입이 IOC 컨테이너에 띄워지면 안되기 떄문 → 싱글톤 위반==**)

    ```java
    @Component
    public class MailSenderAdapter implements MailSender {

    		private Mail mail;

    		public MailSenderAdapter(){
    				this.mail = new Mail();
    		}

        @Override
        public boolean send() {
            return true;
        }
    }
    ```


### 정리

1. `MailSender` 라는 인터페이스를 생성하여 `MailSenderStub`(가짜 Mail 클래스) , `MailSenderAdapter`(추후에 진짜 Mail 을 연결할 클래스) 를 구현체를 만든다.
2. `MailSenderStub` 을 `IOC 컨테이너`에 띄어놓는다. → `@Component`
3. Mail 클래스가 만들어지기 전 `MailSenderStub` 이라는 ==**가짜 클래스를 Service 코드에 주입**==시킨다.(OCP 원칙을 지키기 위해)
4. Mail 클래스가 완성이 되면 `MailSenderAdapter` 에 연결한다.

    ```java
    public class MailSenderAdapter implements MailSender{
    		private Mail mail;

    		public MailSenderAdapter(){
    				this.mail = new Mail();
    		}

    		@Override
    		public boolean sned(){
    				return mail.sendMail();
    		}
    }
    ```

5. Mail 클래스가 완성이 되면 ==**MailSenderStub 클래스를 IOC 컨테이너에 빼고**== Mail 클래스를 연결한 `MailSenderAdapter` 클래스를 IOC 컨테이너에 띄워준다.

이렇게 되면 추후에 Mail 클래스가 완성이 되도 Service 코드를 수정할 필요가 없다 !!! OCP원칙을 지키게 된다 !
