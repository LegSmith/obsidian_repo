  

## 책 저장하기 테스트

---

- 통합테스트기 때문에 SpringBoot 를 띄어놓고 테스트해야한다.
    
    `@SpringBootTest(webEnvironment = WebEnvironment.``_RANDOM_PORT_``)` : 실제 내장톰캣을 실행시키는 테스트
    
- 생성자 주입은 안되고 필드주입으로 받아야한다 → 왜 ???????(추후에 작성)
- ==**JSON 데이터가 필요**==하기 때문에 `ObjectMapper` 를 static으로 선언 ==**HTTP 헤더의 MediaType**== 을 정해야 하기 때문에 `HttpHeaders` 를 static 으로 선언한다.
- @BeforeAll 어노테이션으로 테스트 전 한번만 변수들을 heap메모리에 띄우고 MediaType 을 정해준다.
- 우선 given 절에서는 실제 클라이언트로부터 받는 DTO를 세팅해주고 DTO를 JSON 데이터로 변환한다 !!
- when 절에서는 클라이언트가 요청했다고 가정하기위해 HttpRequest 를 선언해주고 DTO가 담긴 body와 헤더정보가 담긴 header 를 넣어준다.
- 그리고 응답데이터인 response 는 TestRestTemplate 클래스의 exchange() 메서드를 통해 테스트할 URL , HTTP Method, 요청내용인 request, 그리고 반환타입을 적는다.
- 그리고 응답해오는 데이터는 실제론 JSON 인데 JSON 은 테스트 검증하기가 힘들기 때문에 DocumentContext 클래스를 이용하여 JSONPath 규칙에 맞게 찾을 데이터를 갖고 온뒤 검증하면 된다.
    
    ```java
    @SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)
    public class BookApiControllerTest {
    
        @Autowired
        private BookService bookService;
        @Autowired
        private TestRestTemplate rt;
    
        private static ObjectMapper om;
        private static HttpHeaders headers;
        @BeforeAll
        public static void init(){
            om = new ObjectMapper();
            headers = new HttpHeaders();
            headers.setContentType(MediaType.APPLICATION_JSON);
        }
    
        @Test
        public void saveBookTest() throws Exception {
            //given
            BookSaveReqDto dto = new BookSaveReqDto();
            dto.setTitle("This is Football");
            dto.setAuthor("Cristiano");
    
            String body = om.writeValueAsString(dto);
    
            //when
            HttpEntity<String> request = new HttpEntity<>(body, headers);
            ResponseEntity<String> response = rt.exchange("/api/v1/book", HttpMethod.POST, request, String.class);
    
            //then
            DocumentContext documentContext = JsonPath.parse(response.getBody()); //JSON 은 검증하기 힘들기 때문에 DocumentContext 이용
            String title = documentContext.read("$.body.title");
            String author = documentContext.read("$.body.author");
    
            assertThat(title).isEqualTo(dto.getTitle());
            assertThat(author).isEqualTo(dto.getAuthor());
        }
    }
    ```
    

  

궁금한 것  
1. 생성자 주입은 안대고 필드 주입은 왜 되는가 ? → Junit5 가 스스로 DI(의존성주입)을 해주기 떄문에 @Autowired 사용해야함  
2. TestRestTemplate 라는 클래스는 무슨 용도 ? → HTTP 통신을 위해 ?  
3. exchange() 메서드는 무엇 ? → URL에 대해 HTTP 메서드를 실행해주고 요청 엔티티를 요청하여 ResponseEntity 로 응답을 받아온다.

## 책 목록보기 테스트

---

- 책 목록보기는 _==**GET 방식으로 하기 때문에 HTTP 바디가 필요없다.**==_
- 그리고 데이터가 하나 들어가야 검증이 편하므로 `@BeforeEach` 로 데이터 1건을 미리 넣어두고 테스트를 한다.
- Body에 있는 code값과 데이터가 들어있는 body키 값에 실제 List데이터중 1번째 값의 title과 author 를 비교하면 검증하면 끝이다.
    
    - CMRespDto 에서 code는 Integer 로 했기 때문에 Integer 로 선언.
    
    ```java
    @SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)
    public class BookApiControllerTest {
    
        @Autowired
        private BookService bookService;
        @Autowired
        private TestRestTemplate rt;
    
        private static ObjectMapper om;
        private static HttpHeaders headers;
        @BeforeAll
        public static void init(){
            om = new ObjectMapper();
            headers = new HttpHeaders();
            headers.setContentType(MediaType.APPLICATION_JSON);
        }
    		@BeforeEach
        public void dataInit() {
            String title = "This is Football";
            String author = "Cristiano Ronaldo";
            Book book = Book.builder().title(title).author(author).build();
            bookRepository.save(book);
        }
    
        @Test
        public void getBookListTest() throws Exception {
            //given
    
            //when
            HttpEntity<String> request = new HttpEntity<>(null, headers); //GET 은 HTTP Body가 없다.
            ResponseEntity<String> response = rt.exchange("/api/v1/book", HttpMethod.GET, request, String.class);
    
            //then
            DocumentContext documentContext = JsonPath.parse(response.getBody());
            Integer code = documentContext.read("$.code");
            String title = documentContext.read("$.body.data[0].title");
            String author = documentContext.read("$.body.data[0].author");
    
            assertThat(code).isEqualTo(1);
            assertThat(title).isEqualTo("This is Football");
            assertThat(author).isEqualTo("Cristiano Ronaldo");
    
        }
    }
    ```
    

## 책 한건보기 테스트

---

- given 절에 id값을 주고, 이 테스트는 ID값을 건든느 테스트이기 때문에 ==**테스트 실행전에 PK값을 초기화하는 SQL구절이 필요**==하기 때문에 `@Sql("classpath:db/tableInit.sql")` 선언하여 테이블의 PK값을 초기화한다 !!
    
    ```java
    @SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)
    public class BookApiControllerTest {
    
        @Autowired
        private BookService bookService;
        @Autowired
        private TestRestTemplate rt;
    
        private static ObjectMapper om;
        private static HttpHeaders headers;
        @BeforeAll
        public static void init(){
            om = new ObjectMapper();
            headers = new HttpHeaders();
            headers.setContentType(MediaType.APPLICATION_JSON);
        }
    		@BeforeEach
        public void dataInit() {
            String title = "This is Football";
            String author = "Cristiano Ronaldo";
            Book book = Book.builder().title(title).author(author).build();
            bookRepository.save(book);
        }
    
        @Sql("classpath:db/tableInit.sql")
        @Test
        public void getBookTest() throws Exception {
            //given
            Long id = 1L;
    
            //when
            HttpEntity<String> request = new HttpEntity<>(null, headers); //GET 은 HTTP Body가 없다.
            ResponseEntity<String> response = rt.exchange("/api/v1/book/"+ id, HttpMethod.GET, request, String.class);
    
            System.out.println(response.getBody());
    
            //then
            DocumentContext documentContext = JsonPath.parse(response.getBody());
            Integer code = documentContext.read("$.code");
            Integer bodyId = documentContext.read("$.body.id");
            String title = documentContext.read("$.body.title");
            String author = documentContext.read("$.body.author");
    
            assertThat(code).isEqualTo(1);
            assertThat(bodyId).isEqualTo(1);
            assertThat(title).isEqualTo("This is Football");
            assertThat(author).isEqualTo("Cristiano Ronaldo");
    
        }
    }
    ```
    

## 책 삭제하기 테스트

---

- DELETE 통신 시에는 HTTP Body도 없고 응답으로 오는 body 도 null 이기 때문에 응답으로 오는 code나 HTTP 상태코드값을 검증하면 된다.
    
    - 이 테스트도 ID 를 이용하기 때문에 `@SQL` 로 테이블을 초기화 시킨다
    
    ```java
    @SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)
    public class BookApiControllerTest {
    
        @Autowired
        private BookService bookService;
        @Autowired
        private TestRestTemplate rt;
    
        private static ObjectMapper om;
        private static HttpHeaders headers;
        @BeforeAll
        public static void init(){
            om = new ObjectMapper();
            headers = new HttpHeaders();
            headers.setContentType(MediaType.APPLICATION_JSON);
        }
    		@BeforeEach
        public void dataInit() {
            String title = "This is Football";
            String author = "Cristiano Ronaldo";
            Book book = Book.builder().title(title).author(author).build();
            bookRepository.save(book);
        }
    
    		@Sql("classpath:db/tableInit.sql")
        @Test
        public void deleteBook() throws Exception {
            //given
            Long id = 1L;
    
            //when
            HttpEntity<String> request = new HttpEntity<>(null, headers); //GET 은 HTTP Body가 없다.
            ResponseEntity<String> response = rt.exchange("/api/v1/book/"+ id, HttpMethod.DELETE, request, String.class);
    
            //then
            DocumentContext documentContext = JsonPath.parse(response.getBody());
            Integer code = documentContext.read("$.code");
    
            assertThat(code).isEqualTo(1);
            assertThat(response.getStatusCode()).isEqualTo(HttpStatus.OK);
        }
    }
    ```
