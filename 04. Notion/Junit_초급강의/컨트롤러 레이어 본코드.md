  

## 책 저장하기 & 유효성 검사

---

- 컨트롤러 단에서는 ==_**클라이언트가 잘못된 데이터를 넘기지 않았는지 API 스펙에 맞게 요청을 했는지 유효성 검사가 필요**_==하다 !!! → `Validation` 라이브러리를 추가해서 어노테이션 기반으로 체크할 수 있다.
- `@RequestBody` 어노테이션을 붙히면 ==**HTTP 메시지 바디에 데이터를 직접 가져와서 스프링이 메시지 컨버터를 이용해 알맞게 바인딩**==을 해준다.
- 그리고 유효성 검사를 위해 파라미터 앞에 `@Valid` 를 붙히고 ==**바로 뒤에 오는 파라미터에 BindingResult 객체를 받아와야 한다.(규칙)**==
    - 만약 유효성검사를 실패하면 오류내용이 `BindingResult` 에 담긴다.
- hasErrors() 메서드로 에러가 있는지를 체크하고 있다면 임의의 Map 을 만들어 `RuntimeException()`에 담아 throw 해준다.
    
    - `@RestControllerAdvice` 어노테이션을 붙히면 예외가 터졌을 때 제어권을 개발자가 직접 가져올 수 있다.(@ControllerAdvice 에 @ResponseBody 가 붙은 어노테이션이 @RestControllerAdvice 이다.
    - `@ExceptionHandler(Exception.class)` 에 예외타입을 적으면 해다 예외가 터졌을때 아래 메서드가 제일 먼저 호출된다.
    
    ```java
    @RestControllerAdvice
    public class GlobalExceptionHandler {
    
        @ExceptionHandler(RuntimeException.class)
        public ResponseEntity<?>  apiException(RuntimeException e){
    
            return new ResponseEntity<>(CMRespDto.builder().code(-1).msg(e.getMessage()).body(null).build(), HttpStatus.BAD_REQUEST);
        }
    }
    ```
    
- 에러가 없다면 BookService 를 이용하여 save하고 클라이언트에게 JSON 형식으로 응답해주면 된다.(`CMRespDto` 는 공통 응답을 위한 클래스)
    
    ```java
    @RequiredArgsConstructor
    @RestController
    public class BookApiController {
    
        private final BookService bookService;
    
        // 1. 책 등록
        @PostMapping("/api/v1/book")
        public ResponseEntity<?> saveBook(@RequestBody @Valid BookSaveReqDto dto, BindingResult bindingResult) {
            if (bindingResult.hasErrors()) {
                Map<String, String> errorMap = new HashMap<>();
                for (FieldError error : bindingResult.getFieldErrors()) {
                    errorMap.put(error.getField(), error.getDefaultMessage());
                }
                throw new RuntimeException(errorMap.toString());
            }
            BookRespDto result = bookService.책등록하기(dto);
            return new ResponseEntity<>(CMRespDto.builder().code(1).msg("글 저장 성공!").body(result).build(), HttpStatus.CREATED);
        }
    
    }
    ```
    

## 책 목록보기

---

- List 타입을 그대로 JSON 데이터에 담아서 보내면 좋지 않다. → API 확장성
- List를 가지는 DTO를 생서
    
    ```java
    @Data
    public class BookListRespDto {
        List<BookRespDto> data;
    
        @Builder
        public BookListRespDto(List<BookRespDto> bookList) {
            this.data = bookList;
        }
    }
    ```
    
- Service 코드도 수정이 필요하다.
    
    ```java
    public BookListRespDto 책목록보기() {
        return BookListRespDto.builder().
                bookList(
                        bookRepository.findAll().stream()
                                .map(Book::toDto)
                                .collect(Collectors.toList())
                ).build();
    }
    ```
    
- JSON 에 응답하는 Body 에는 List 타입을 가지는 DTO 를 응답해주면 된다.
    
    ```java
    
    @RequiredArgsConstructor
    @RestController
    public class BookApiController {
    
        private final BookService bookService;
    
    		// 2. 책 목록보기
        @GetMapping("/api/v1/book")
        public ResponseEntity<?> getBookList(){
            BookListRespDto dto = bookService.책목록보기();
    
            return new ResponseEntity<>(CMRespDto.builder().code(1).msg("글 목록보기 성공").body(dto).build(),
                    HttpStatus.OK);
        }
    }
    ```
