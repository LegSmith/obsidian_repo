스프링 시큐리티는 로그인 요청이 들어오면 권한을 가로채서 `UserDetailsService` 인터페이스의 `loadUserByUsername()` 메서드를 호출시켜 로그인 로직을 처리한다.  
이 때 DB에서 로그인정보를 확인하고 올바른 사용자면 `UserDetails` 타입의 객체에 로그인정보를 담고 `Authentication` 객체로 감싸서 Session 에 저장한다.

### UserDetails 구현

- User 엔티티를 필드로 갖고 생성자를 만든다.
    
    ```java
    public class PrincipalDetails implements UserDetails {
    
        private User user;
    
        public PrincipalDetails(User user) {
            this.user = user;
        }
    
        /**
         * 해당 User 의 권한을 리턴하는 메서드
         */
        @Override
        public Collection<? extends GrantedAuthority> getAuthorities() {
            Collection<GrantedAuthority> collect = new ArrayList<>();
            collect.add(new GrantedAuthority() {
                @Override
                public String getAuthority() {
                    return user.getRole();
                }
            });
            return collect;
        }
    
        @Override
        public String getPassword() {
            return user.getPassword();
        }
    
        @Override
        public String getUsername() {
            return user.getUsername();
        }
    
        /**
         * 계정이 만료 되었는지
         */
        @Override
        public boolean isAccountNonExpired() {
            return true;
        }
    
        /**
         * 계정이 잠겨있는지 ?
         */
        @Override
        public boolean isAccountNonLocked() {
            return true;
        }
    
        /**
         * 게정 비밀번호가 오래사용됐는지 ?
         */
        @Override
        public boolean isCredentialsNonExpired() {
            return true;
        }
    
        /**
         * 계정이 비활성화 되어있는지 ?
         */
        @Override
        public boolean isEnabled() {
            return true;
        }
    }
    ```
    

### UserDetailsService 구현

- `UserDetailsService` 는 인터페이스이기 때문에 구현체를 만들어 사용해야 한다.
- 로그인정보를 통해 User 테이블에서 조회해야 하기 때문에 UserRepository 를 주입받는다.
- `loadUserByUsername()` 메서드는 username을 파라미터로 갖고오기 때문에 username을 조회한 후 DB 에 있는 사용자면 `PrincipalDetails()` 에 user 정보를 넣어서 session 에 저장한다.
    
    ==_**PrincipalDetails 은 UserDetails 를 구현한 구현체이다.**_==
    
    ```java
    @RequiredArgsConstructor
    @Slf4j
    @Service
    public class PrincipalDetailsService implements UserDetailsService {
    
        private final UserRepository userRepository;
    
        /**
         * 파라미터명과 뷰템플릿에서 username 이 입력되는 input 태그의 name 속성값과 일치해야 한다.
         */
        @Override
        public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
            User userEntity = userRepository.findByUsername(username);
            if(userEntity != null){
                return new PrincipalDetails(userEntity);
            }
            return null;
        }
    }
    ```
