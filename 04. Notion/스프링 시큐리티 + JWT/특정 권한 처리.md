  

## 시큐리티 설정 파일에서 권한 처리

---

- 시큐리티 설정 파일에서 특정 URL 로 들어올 때 권한에 따라 접근을 가능하게 하는 설정이 가능하다.
    
    - `antMatchers()` 에 경로를 적고 `access()` 메서드에 ==**hasRole 에 권한을 적어서 필터링**==을 할 수 있다.
    
    ```java
    @Configuration
    @EnableWebSecurity // 스프링 시큐리티 필터가 스프링 필터체인에 등록이 된다.
    public class SecurityConfig {
    
        @Bean
        public BCryptPasswordEncoder encodePwd() {
            return new BCryptPasswordEncoder();
        }
    
        @Bean
        SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
            http.csrf().disable();
            http.authorizeRequests()
                .antMatchers("/user/**").authenticated() // 인증만 되면 들어갈 수 있다.
                .antMatchers("/manager/**").access("hasRole('ROLE_ADMIN') or hasRole('ROLE_MANAGER')")
                .antMatchers("/admin/**").access("hasRole('ROLE_ADMIN')")
                .anyRequest().permitAll()
                .and()
                .formLogin()
                .loginPage("/loginForm")
                .loginProcessingUrl("/login") // /login 경로가 호출되면 시큐리티가 권한을 가져 로그인 진행
                .defaultSuccessUrl("/"); // login 성공시 메인페이지로 이동
    
            return http.build();
        }
    }
    ```
    

## 특정 메서드에 권한 처리

---

- 먼저 시큐리티 설정파일에 `@EnableGlobalMethodSecurity` 어노테이션을 클래스 위에 적어준다.
    
    `securedEnabled` : 특정 메서드에서 `@Secured` 어노테이션이 사용 가능하다.
    
    `prePostEnabled` : 특정 메서드에서 `@preAuthorize` , `@postAuthorize` 사용 가능
    
    ```java
    @Configuration
    @EnableWebSecurity // 스프링 시큐리티 필터가 스프링 필터체인에 등록이 된다.
    @EnableGlobalMethodSecurity(securedEnabled = true, prePostEnabled = true) // secured 어노테이션 활성화, preAuthorize,postAuthorize 어노테이션 활성화
    public class SecurityConfig {
    		
    		//생략...
    
    }
    ```
    
- `@Secured` 어노테이션은 권한명을 바로 적어주면 되지만 @PreAuthorize 에는 hasRole 안에 권한명을 적어줘야 한다.
- `@preAuthorize` 는 메서드 실행 전에 동작하고 , `@postAuthorize` 는 메서드가 끝난 후에 동작한다.
- `@postAuthorize` 는 특정 권한만 접근
    
    ```java
    @Secured("ROLE_ADMIN") // 특정 권한만 접근가능
    @GetMapping("/info")
    public @ResponseBody String info(){
        return "개인정보";
    }
    
    @PreAuthorize("hasRole('ROLE_MANAGER') or hasRole('ROLE_ADMIN')") // 해당 메서드 실행 전에 실행된다.
    @GetMapping("/data")
    public @ResponseBody String data(){
        return "데이터 정보";
    }
    ```
